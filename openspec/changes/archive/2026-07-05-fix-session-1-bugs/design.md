## Context

В результате первой сессии ручного тестирования (Session 1) выявлено 4 дефекта, два из которых Critical. Все дефекты относятся к базовому UX: создание/выделение текста, отображение маркеров при повороте, форматирование StatusBar, поведение Shift при рисовании линии.

Кодовая база — .NET 10 WPF MVVM-приложение с CommunityToolkit.Mvvm, фиксированными координатами в микронах. Текущий спринт — Sprint 57 (цвета, half-форматы, MultiLine, библиотека, настройки). 1796 unit-тестов, 1 pre-existing skip.

Существующий spec `session1-defects` уже содержит требования по 45° диагонали (с допуском 1 микрон), которые нужно скорректировать.

## Goals / Non-Goals

**Goals:**
- Увеличить `DefaultFontSizeMicrons` с 3500 до 14000 (14 мм) — текст видим и кликабелен при 100% zoom
- Исправить `Coordinate.FormatMm()` — убрать лишние `.000` для целых мм
- Сделать 45° диагональ при Shift достижимой — увеличить `diagonalTolerance`
- Разместить визуальные маркеры повёрнутого текста на реальных углах (4 отдельных свойства-пары)
- Исправить `GetTextHandle()` для поиска хендлов на повёрнутых углах
- Скорректировать `ComputeTextResize()` для учёта поворота текста

**Non-Goals:**
- Не добавлять mid-side маркеры для текста (остаются только 4 угловых)
- Не менять поведение Rectangle или Line при повороте (они не вращаются)
- Не менять сериализацию/десериализацию `.tdel`
- Не затрагивать Print Preview

## Decisions

### D1: DefaultFontSizeMicrons = 14000 (14 мм)

Выбран размер 14 мм — стандартный ГОСТ 2.304-81 размер шрифта. Альтернативы:
- 7000 (7 мм) — всё ещё мелковат для клика при 100% zoom (7 px)
- 20000 (20 мм) — слишком крупно для стандартного текста на чертеже
- 10000 (10 мм) — возможен, но 14 мм даёт запас для кликабельности

14 мм при 100% zoom: ~14 px font size, текст "Текст" (5 символов, ГОСТ А) ≈ 35 px ширина — уверенно кликабельно.

### D2: FormatMm — "0.###" вместо "F3"

`"F3"` всегда выдаёт 3 знака: `420.000`. `"0.###"` подавляет trailing zeros и точку для целых чисел: `420`. Для дробных мм (например `42.5`) показывает `42.5`. Обратно совместим — все существующие парсеры (`ParseMm`) принимают оба формата.

### D3: 4 отдельные пары свойств RotatedCorner вместо массива

WPF DataBinding не поддерживает индексаторы `{Binding Corners[0].X}` в текущей архитектуре (MarkerPosition привязан к XPropertyPath/YPropertyPath как строка). Решение — 4 пары плоских свойств:

```
RotatedCorner0X, RotatedCorner0Y  // центр вращения (top-left WPF)
RotatedCorner1X, RotatedCorner1Y  // top-right WPF
RotatedCorner2X, RotatedCorner2Y  // bottom-left WPF
RotatedCorner3X, RotatedCorner3Y  // bottom-right WPF
```

Каждая пара вычисляется через матрицу поворота вокруг центра `(MicronsX, MicronsY + HeightMicrons)`.

### D4: Проекция dx/dy в локальную СК для ResizeMath

Текущий код применяет мировые `dx`/`dy` напрямую. Для повёрнутого текста это даёт неверный scale factor.

Решение: перед вычислением scale, спроецировать `(dx, dy)` в локальную систему координат текста через обратную матрицу поворота (угол `-RotationAngle`):

```
dxLocal =  dx * cosA + dy * sinA
dyLocal = -dx * sinA + dy * cosA
```

Затем использовать `dxLocal`/`dyLocal` в существующей логике scale. После вычисления `newFontSize`, позиция `newX`/`newY` остаётся в мировых координатах (сдвиг базовой линии не зависит от поворота).

### D5: diagonalTolerance = 5000 микрон (5 мм)

Текущее значение 1 микрон делает 45° недостижимым. 5000 микрон (5 мм) при типичном drag в 100-200 мм даёт угловой допуск ~1.5-3°, что комфортно для пользователя. Значение выбрано эмпирически — достаточно широкое для попадания, достаточно узкое чтобы не захватывать явно не-диагональные линии.

## Risks / Trade-offs

- **[R1] Тесты на DefaultFontSizeMicrons**: если есть тесты с хардкодом `3500`, они упадут. Проверить `EditorSettingsTests` и тесты `TextTool`. Mitigation: grep по `DefaultFontSizeMicrons|3500` в тестах.
- **[R2] RotatedCorner свойства — INPC**: при изменении `RotationAngle`, `WidthMicrons`, `HeightMicrons` или позиции текста все 4 пары `RotatedCornerNX/NY` должны оповестить `PropertyChanged`. Иначе маркеры не перерисуются. Mitigation: `OnPropertyChanged(nameof(RotatedCorner0X))` и т.д. при изменении любого из зависимых свойств.
- **[R3] ResizeMath — регрессия для неповёрнутого текста**: при RotationAngle=0, cosA=1, sinA=0 → dxLocal=dx, dyLocal=dy — поведение идентично текущему.
- **[R4] diagonalTolerance в пикселях vs микронах**: 5000 микрон = 5 мм. При 100% zoom = 5 px. При 50% = 2.5 px. Может быть слишком узко при низком zoom. Альтернатива — вычислять tolerance в пикселях через zoom. Но это требует доступа к ZoomPanManager из DrawingLineTool, что усложняет архитектуру. Принято: 5000 микрон фиксированно.
