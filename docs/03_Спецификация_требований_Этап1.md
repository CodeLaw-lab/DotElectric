#### 2.3.3. Правила для будущего разделения

Чтобы разделение прошло безболезненно, **уже на Этапе 1** соблюдаются правила:

| Правило | Описание |
|---------|----------|
| **Models не зависят от WPF** | `Models/` НЕ импортируют `System.Windows.*`. Исключение: `Coordinate.cs` может использовать `CultureInfo` |
| **Services (интерфейсы) не зависят от UI** | `I*Service.cs` НЕ импортируют WPF-типы |
| **Чистые зависимости** | `Models` ← `Services` ← `ViewModels` ← `Views` (строго в одну сторону) |
| **Без циклических зависимостей** | View → ViewModel → Service → Model. Обратных зависимостей нет. |
| **ITool в отдельной папке** | `ITool.cs` — в `Tools/`, реализации — тоже в `Tools/`. При разделении интерфейс уйдёт в Core |
| **Commands без WPF** | `ICommand` (для Undo/Redo) НЕ зависит от `System.Windows.Input.ICommand` |
| **Helpers без UI** | `HitTestHelper`, `SnapHelper` работают только с моделями |

### 3.1. Модель данных

#### 3.1.1. Template (Шаблон)

| Свойство | Тип | Описание |
|----------|-----|----------|
| Metadata | Metadata | Метаданные шаблона |
| Sheet | Sheet | Параметры листа |
| DefaultGrid | Grid | Сетка по умолчанию (фиксированная, 5 мм, readonly) |
| Objects | List\<ITemplateObject\> | Коллекция объектов |
| Version | string | Версия формата |

**Примечание:** `DefaultGrid` — константа (шаг 5 мм = 5000 микрон), не настраивается и не сериализуется. Рабочие настройки сетки хранятся в `EditorViewModel.GridSettings` (см. раздел 3.1.F.4).

#### 3.1.2. Metadata (Метаданные)

| Свойство | Тип | Описание |
|----------|-----|----------|
| Name | string | Наименование шаблона |
| Description | string | Описание |
| Author | string | Автор |
| CreatedDate | DateTime | Дата создания |
| ModifiedDate | DateTime | Дата последнего изменения |

#### 3.1.3. Sheet (Лист)

| Свойство | Тип | Описание |
|----------|-----|----------|
| Format | string | Формат (A0, A1, A2, A3, A4, A4×2, A3×2, A2×2, A1×2, A0×2, Custom) |
| WidthMicrons | long | Ширина в микронах (1 мм = 1000 микрон) |
| HeightMicrons | long | Высота в микронах |
| Unit | string | Единица измерения (mm) |

**Примечание:** Внутреннее хранение координат и размеров осуществляется в микронах (Fixed-Point) для исключения погрешностей вычислений. **Сериализация в XML также выполняется в микронах** (`xs:long`). Конвертация в мм происходит только при отображении на UI через `Coordinate.ToMm()`.

#### 3.1.4. Grid (Сетка по умолчанию)

| Свойство | Тип | Описание |
|----------|-----|----------|
| StepMicrons | long | Шаг сетки = 5000 микрон (5 мм, фиксированный) |

**Примечание:** `DefaultGrid` в `Template` — константа. Рабочие настройки сетки (`Enabled`, `SnapEnabled`, `StepMicrons`, `Visible`) хранятся в `EditorViewModel.GridSettings` и НЕ сериализуются в `.tdel`.

**Доступные шаги сетки в редакторе:**
- 0.5 мм = 500 микрон
- 1 мм = 1000 микрон
- 5 мм = 5000 микрон (по умолчанию, совпадает с DefaultGrid)
- 10 мм = 10000 микрон

| Свойство | Тип | Описание |
|----------|-----|----------|
| Id | string | Уникальный идентификатор (Guid) |

| MicronsY | long | Координата Y в микронах |

**Свойства-обёртки (для удобства):**
| Свойство | Тип | Описание |

#### 3.1.6. Line (Линия)

| Свойство | Тип | Описание |
|----------|-----|----------|
| Id | string | Уникальный идентификатор (Guid) |
| StartMicronsX | long | X начальной точки в микронах |
| StartMicronsY | long | Y начальной точки в микронах |
| EndMicronsX | long | X конечной точки в микронах |
| EndMicronsY | long | Y конечной точки в микронах |
| LineType | LineType | Тип линии (Solid, Dashed, DashDot, DashDotDotDot) |
| StrokeColor | string | Цвет линии в HEX (#RRGGBB или #AARRGGBB) |

**Опорная точка:** `MicronsX/Y` = `StartMicronsX/Y` (начало линии).

**Метод `Move(micronsX, micronsY)`:** Сдвигает **обе** точки на дельту (разница между старой и новой опорной точкой).

**Метод `Clone()`:** Создаёт копию с новым `Id`.

**Значения по умолчанию:** `LineType = LineType.Solid`.

| Свойство | Тип | Описание |
|----------|-----|----------|
| Id | string | Уникальный идентификатор (Guid) |

| MicronsY | long | Y левого нижнего угла в микронах |

| HeightMicrons | long | Высота в микронах |

**Метод `Clone()`:** Создаёт копию с новым `Id`.

| Свойство | Тип | Описание |
|----------|-----|----------|
| Id | string | Уникальный идентификатор (Guid) |
| MicronsX | long | X базовой линии в микронах (опорная точка) |

| FontSizeMicrons | long | Размер шрифта в микронах (высота символа) |

#### 3.1.10. PointMicrons (Точка в микронах)

Вспомогательная структура для работы с координатами в микронах.

```csharp
public readonly struct PointMicrons
{
    public long MicronsX { get; }
    public long MicronsY { get; }
    
    public PointMicrons(long micronsX, long micronsY)
    {
        MicronsX = micronsX;
        MicronsY = micronsY;
    }
    
    // Конвертация из мм (с округлением)
    public static PointMicrons FromMm(double mmX, double mmY)
        => new((long)Math.Round(mmX * 1000), (long)Math.Round(mmY * 1000));
    
    // Конвертация в мм
    public double X => MicronsX / 1000.0;
    public double Y => MicronsY / 1000.0;
}
```

| Direction | enum | Направление рисования (LeftToRight, RightToLeft) |
| StartMicrons | PointMicrons | Начальная точка рамки (в микронах) |
| EndMicrons | PointMicrons | Конечная точка рамки (в микронах) |
| VisualStyle | Style | Стиль отрисовки (синий/зелёный) |

**Логика работы:**
- LeftToRight: Выделяются только объекты, целиком попавшие в рамку (синий контур + полупрозрачная синяя заливка)
- RightToLeft: Выделяются все задетые объекты (зелёный пунктирный контур + зелёная заливка)

### 3.1.A. Coordinate Helper (Fixed-Point)

Класс `Coordinate` предоставляет методы для работы с Fixed-Point координатами.

**Назначение:** Исключение погрешностей вычислений с плавающей точкой.

**Принцип работы:**
- 1 мм = 1000 микрон (единиц)
- Все вычисления производятся в `long` (целочисленная арифметика)
- **Сериализация в XML — в микронах** (`xs:long`). Конвертация в мм только для UI.
- Round-trip (сохранить → загрузить) не теряет точность

**Основные методы:**

```csharp
public static class Coordinate
{
    // Константы
    public const long MicronsPerMm = 1000;
    public const long MicronsPerCm = 10000;

    // Конвертация мм → микроны
    public static long ToMicrons(double mm)
        => (long)Math.Round(mm * MicronsPerMm);

    // Конвертация микроны → мм
    public static double ToMm(long microns)
        => microns / (double)MicronsPerMm;

    // Snap к сетке
    public static long SnapToGrid(long microns, long stepMicrons)
        => ((microns + stepMicrons / 2) / stepMicrons) * stepMicrons;

    // Форматирование для отображения (мм, 3 знака)
    public static string FormatMm(long microns)
        => (microns / 1000.0).ToString("F3");

    // Парсинг из строки (мм → микроны)
    public static long ParseMm(string value)
        => ToMicrons(double.Parse(value, CultureInfo.InvariantCulture));

    // === Сериализация XML ===

    // Сериализация: микроны → строка XML (целое число)
    public static string SerializeMicrons(long microns)
        => microns.ToString(CultureInfo.InvariantCulture);

    // Десериализация: строка XML → микроны (целое число)
    public static long DeserializeMicrons(string value)
        => long.Parse(value, CultureInfo.InvariantCulture);
}
```

**Преимущества Fixed-Point:**
- ✅ Полное отсутствие погрешности накопления
- ✅ Быстрые целочисленные операции
- ✅ Точная привязка к сетке
- ✅ Предсказуемое сравнение координат
- ✅ **Round-trip сериализация без потерь** (XML хранит `long`)

**Конвертация для WPF:**
```csharp
// WPF использует double, конвертация только для отрисовки
var wpfPoint = new System.Windows.Point(
    Coordinate.ToMm(micronsX),
    Coordinate.ToMm(micronsY)
);
```

**Поток данных:**
```
Модель (long) ──→ UI: Coordinate.ToMm() ──→ WPF (double) ──→ отрисовка
Модель (long) ──→ XML: SerializeMicrons() ──→ xs:long ──→ файл .tdel
Файл .tdel ──→ xs:long ──→ DeserializeMicrons() ──→ Модель (long)
```

### 3.1.B. Система координат и конвертация

#### 3.1.B.1. Описание проблемы

Требование ТЗ (раздел 3.2):
> Система координат: Декартова, точка **(0, 0) — левый нижний угол** листа.

WPF по умолчанию использует экранную систему координат:
> **(0, 0) — левый верхний угол**, ось Y направлена **вниз**.

Это фундаментальное расхождение требует стратегии конвертации.

#### 3.1.B.2. Принятый подход

**Декартова в модели, конвертер на уровне `EditorCanvas`.**

| Слой | Система координат | Ось Y |
|------|-------------------|-------|
| **Model** (Template, Objects) | Декартова | ↑ Вверх |
| **Services** (TemplateService, сериализация) | Декартова | ↑ Вверх |
| **ViewModel** | Декартова | ↑ Вверх |
| **EditorCanvas** (View) | WPF (экранная) | ↓ Вниз |
| **Другие View** (PropertiesPanel, etc.) | Декартова (через модель) | ↑ Вверх |

**Принципы:**
- Модель **всегда** хранит Декартовы координаты (0,0 = левый нижний угол)
- Конвертация происходит **только** на уровне `EditorCanvas`
- ViewModel, Model, Services — ничего не знают про WPF-координаты
- Сериализация/десериализация — прямая из модели (без дополнительных преобразований)
- Привязка к сетке (`SnapToGrid`) работает с микронными координатами **до** конвертации

#### 3.1.B.3. Редактор координат (EditorCanvas Coordinate Mapper)

`EditorCanvas` предоставляет два метода-конвертера:

```csharp
public partial class EditorCanvas : Canvas
{
    private Sheet? _currentSheet;

    /// <summary>
    /// Из WPF-координат мыши → в модель (Декартовы, микроны).
    /// Вызывается при MouseDown, MouseMove, MouseUp.
    /// </summary>
    public PointMicrons FromWpfPoint(Point wpfPoint)
    {
        var mmX = wpfPoint.X / _zoom;
        var mmY = _currentSheet.HeightMm - (wpfPoint.Y / _zoom);
        return PointMicrons.FromMm(mmX, mmY);
    }

    /// <summary>
    /// Из модели (Декартовы, микроны) → в WPF-координаты для отрисовки.
    /// Вызывается при рендеринге каждого объекта.
    /// </summary>
    public Point ToWpfPoint(PointMicrons microns)
    {
        var x = Coordinate.ToMm(microns.MicronsX) * _zoom;
        var y = (_currentSheet.HeightMm - Coordinate.ToMm(microns.MicronsY)) * _zoom;
        return new Point(x, y);
    }

    /// <summary>
    /// Инверсия Y для заданного значения в мм (упрощённый вариант).
    /// </summary>
    public double InvertY(double mmY) => _currentSheet.HeightMm - mmY;
}
```

### 3.1.C. EditorCanvas и State Pattern

#### 3.1.C.1. Архитектура обработки событий

`EditorCanvas` использует **State Pattern** для обработки событий мыши. Каждый инструмент рисования/взаимодействия — отдельное состояние.

```csharp
/// <summary>
/// Интерфейс инструмента рисования/взаимодействия.
/// Реализует State Pattern — каждый инструмент является состоянием.
/// </summary>
public interface ITool
{
    /// <summary>
    /// Вызывается при нажатии кнопки мыши.
    /// modelPoint — уже в Декартовых координатах (микроны).
    /// </summary>
    void OnMouseDown(PointMicrons modelPoint, MouseButton button, ModifierKeys modifiers);

    /// <summary>
    /// Вызывается при перемещении мыши.
    /// </summary>
    void OnMouseMove(PointMicrons modelPoint, ModifierKeys modifiers);

    /// <summary>
    /// Вызывается при отпускании кнопки мыши.
    /// </summary>
    void OnMouseUp(PointMicrons modelPoint, MouseButton button, ModifierKeys modifiers);

    /// <summary>
    /// Вызывается при двойном клике.
    /// </summary>
    void OnDoubleClick(PointMicrons modelPoint);

    /// <summary>
    /// Вызывается при прокрутке колёсика мыши.
    /// zoomDelta — направление и величина (положительное = zoom in).
    /// </summary>
    void OnMouseWheel(PointMicrons modelPoint, double zoomDelta);

    /// <summary>
    /// Курсор для данного инструмента.
    /// </summary>
    Cursor GetCursor();
}
```

#### 3.1.C.2. Реализации инструментов

| Инструмент | Класс | Описание |
|-----------|-------|----------|
| **Выделение** | `SelectTool` | Клик → выделить, Shift+клик → добавить, рамка → selection box, перетаскивание → move |
| **Линия** | `DrawingLineTool` | Нажатие → начать линию, drag → preview, отпускание → создать Line |

#### 3.1.C.3. Переключение инструментов (Tool Stack)

`EditorCanvas` использует **стек инструментов** для временной смены активного инструмента:

```csharp
public partial class EditorCanvas : Canvas
{
    private readonly Stack<ITool> _toolStack = new();
    private ITool _baseTool; // «Базовый» инструмент (Select, Line, Rect, Text)

    /// <summary>
    /// Текущий активный инструмент.
    /// Если стек пуст — возвращается _baseTool.
    /// </summary>
    public ITool ActiveTool => _toolStack.Count > 0 ? _toolStack.Peek() : _baseTool;

    /// <summary>
    /// Установить базовый инструмент (выбор в тулбаре).
    /// </summary>
    public void SetActiveTool(ToolType type)
    {
        _baseTool = type switch
        {
            ToolType.Select      => _selectTool,
            ToolType.DrawLine    => _drawingLineTool,
            ToolType.DrawRect    => _drawingRectangleTool,
            ToolType.Text        => _textTool,
            _ => throw new ArgumentException($"Unknown tool type: {type}")
        };
        Cursor = ActiveTool.GetCursor();
    }

    /// <summary>
    /// Временная смена инструмента (Pan, Resize) — поверх базового.
    /// </summary>
    public void PushTool(ITool tempTool)
    {
        _toolStack.Push(tempTool);
        Cursor = tempTool.GetCursor();
    }

    /// <summary>
    /// Вернуться к предыдущему инструменту.
    /// </summary>
    public void PopTool()
    {
        if (_toolStack.Count == 0) return;
        _toolStack.Pop();
        Cursor = ActiveTool.GetCursor();
    }
}
```

**Типичные сценарии использования стека:**

| Сценарий | Действие | Стек |
|----------|----------|------|
| Пользователь выбрал «Линия» | `SetActiveTool(DrawLine)` | `[DrawLineTool]` |
| Зажал среднюю кнопку | `PushTool(PanTool)` | `[PanTool, DrawLineTool]` |
| Отпустил среднюю кнопку | `PopTool()` | `[DrawLineTool]` |
| Тянет маркер ресайза | `PushTool(ResizeTool)` | `[ResizeTool, SelectTool]` |
| Отпустил маркер | `PopTool()` | `[SelectTool]` |

#### 3.1.C.4. Делегирование событий

`EditorCanvas` делегирует все mouse-события активному инструменту:

```csharp
public partial class EditorCanvas : Canvas
{
    protected override void OnMouseDown(MouseButtonEventArgs e)
    {
        var wpfPoint = e.GetPosition(this);
        var modelPoint = FromWpfPoint(wpfPoint);    // конвертация координат

        // Привязка к сетке (если включена)
        if (_grid.SnapEnabled)
            modelPoint = SnapHelper.SnapToGrid(modelPoint, _grid.StepMicrons);

        ActiveTool.OnMouseDown(modelPoint, e.ChangedButton, Keyboard.Modifiers);
        e.Handled = true;
    }

    protected override void OnMouseMove(MouseEventArgs e)
    {
        var wpfPoint = e.GetPosition(this);
        var modelPoint = FromWpfPoint(wpfPoint);

        if (_grid.SnapEnabled)
            modelPoint = SnapHelper.SnapToGrid(modelPoint, _grid.StepMicrons);

        ActiveTool.OnMouseMove(modelPoint, Keyboard.Modifiers);
        UpdatePreview();
        e.Handled = true;
    }

    protected override void OnMouseUp(MouseButtonEventArgs e)
    {
        var wpfPoint = e.GetPosition(this);
        var modelPoint = FromWpfPoint(wpfPoint);

        if (_grid.SnapEnabled)
            modelPoint = SnapHelper.SnapToGrid(modelPoint, _grid.StepMicrons);

        ActiveTool.OnMouseUp(modelPoint, e.ChangedButton, Keyboard.Modifiers);
        ClearPreview();
        e.Handled = true;
    }

    protected override void OnMouseWheel(MouseWheelEventArgs e)
    {
        var wpfPoint = e.GetPosition(this);
        var modelPoint = FromWpfPoint(wpfPoint);

        // Zoom обрабатывается глобально (не через инструмент)
        var zoomDelta = e.Delta / 1200.0; // ±0.1 за клик
        ApplyZoom(modelPoint, zoomDelta);

        // Уведомить инструмент об изменении масштаба
        ActiveTool.OnMouseWheel(modelPoint, zoomDelta);
        e.Handled = true;
    }
}
```

#### 3.1.C.5. Preview-отрисовка

При рисовании линии/прямоугольника показывается «призрак» объекта (preview).

**Принцип:** `EditorCanvas` владеёт визуалом, инструмент отдаёт данные.

**Ключевой принцип производительности:**
- Preview-элементы создаются **один раз** при первом использовании
- При каждом `MouseMove` **обновляются свойства** (X1, Y1, X2, Y2), а не создаётся новый Shape
- Это исключает аллокации и GC-давление

```csharp
public partial class EditorCanvas : Canvas
{
    // Переиспользуемые preview-элементы (НЕ создаются каждый кадр)
    private System.Windows.Shapes.Line? _previewLine;
    private System.Windows.Shapes.Rectangle? _previewRectangle;

    public void UpdatePreview()
    {
        ClearPreview();

        if (_currentTool is DrawingLineTool lineTool && lineTool.HasPreview)
        {
            var points = lineTool.GetPreviewPoints();
            var start = ToWpfPoint(points.Start);
            var end = ToWpfPoint(points.End);

            _previewLine ??= CreatePreviewLine();
            _previewLine.X1 = start.X;
            _previewLine.Y1 = start.Y;
            _previewLine.X2 = end.X;
            _previewLine.Y2 = end.Y;
            _previewLine.Visibility = Visibility.Visible;
        }
        else if (_currentTool is DrawingRectangleTool rectTool && rectTool.HasPreview)
        {
            var points = rectTool.GetPreviewPoints();
            var topLeft = ToWpfPoint(points.TopLeft);
            var bottomRight = ToWpfPoint(points.BottomRight);

            _previewRectangle ??= CreatePreviewRectangle();
            Canvas.SetLeft(_previewRectangle, Math.Min(topLeft.X, bottomRight.X));
            Canvas.SetTop(_previewRectangle, Math.Min(topLeft.Y, bottomRight.Y));
            _previewRectangle.Width = Math.Abs(bottomRight.X - topLeft.X);
            _previewRectangle.Height = Math.Abs(bottomRight.Y - topLeft.Y);
            _previewRectangle.Visibility = Visibility.Visible;
        }
    }

    public void ClearPreview()
    {
        if (_previewLine != null) _previewLine.Visibility = Visibility.Collapsed;
        if (_previewRectangle != null) _previewRectangle.Visibility = Visibility.Collapsed;
    }
}
```

### 3.1.D. Hit-testing

#### 3.1.D.1. Описание задачи

При клике на холсте необходимо определить, **по какому объекту** кликнул пользователь. Это нужно для:
- Выделения объектов (SelectTool)
- Перемещения объектов
- Inline-редактирования текста
- Контекстного меню

#### 3.1.D.2. Алгоритм поиска

**Линейный поиск с конца списка** (Z-order: верхний объект имеет приоритет):

```csharp
public static class HitTestHelper
{
    /// <summary>
    /// Найти верхний объект в указанной точке.
    /// Поиск с конца списка — последний нарисованный = верхний.
    /// </summary>
    public static ITemplateObject? HitTest(
        PointMicrons point,
        IList<ITemplateObject> objects)
    {
        for (int i = objects.Count - 1; i >= 0; i--)
        {
            var obj = objects[i];
            if (obj is Line line && HitTestLine(line, point))
                return line;
            if (obj is Rectangle rect && HitTestRectangle(rect, point))
                return rect;
            if (obj is Text text && HitTestText(text, point))
                return text;
        }
        return null;
    }
}
```

#### 3.1.D.3. Hit-test для каждого типа

**Прямоугольник** — проверка попадания в bounding box:

```csharp
public static bool HitTestRectangle(Rectangle rect, PointMicrons point)
{
    return point.MicronsX >= rect.MicronsX &&
           point.MicronsX <= rect.MicronsX + rect.WidthMicrons &&
           point.MicronsY >= rect.MicronsY &&
           point.MicronsY <= rect.MicronsY + rect.HeightMicrons;
}
```

**Текст** — аналогично, по bounding box:

```csharp
public static bool HitTestText(Text text, PointMicrons point)
{
    return point.MicronsX >= text.MicronsX &&
           point.MicronsX <= text.MicronsX + text.WidthMicrons &&
           point.MicronsY >= text.MicronsY &&
           point.MicronsY <= text.MicronsY + text.HeightMicrons;
}
```

**Линия** — расстояние от точки до отрезка с допуском (tolerance):

```csharp
public static class HitTestHelper
{
    /// <summary>
    /// Зона захвата для линий (в микронах).
    /// 5000 микрон = 5 мм — комфортно для мыши.
    /// </summary>
    public const long LineHitTolerance = 5000;

    public static bool HitTestLine(Line line, PointMicrons point)
    {
        return DistanceFromPointToLine(
            point, line.StartMicrons, line.EndMicrons) <= LineHitTolerance;
    }

    /// <summary>
    /// Расстояние от точки до отрезка (в микронах).
    /// Использует проекцию точки на прямую.
    /// </summary>
    public static long DistanceFromPointToLine(
        PointMicrons point,
        PointMicrons lineStart,
        PointMicrons lineEnd)
    {
        long dx = lineEnd.MicronsX - lineStart.MicronsX;
        long dy = lineEnd.MicronsY - lineStart.MicronsY;
        long lengthSquared = dx * dx + dy * dy;

        if (lengthSquared == 0)
            return Distance(point, lineStart);

        // Проекция точки на прямую (параметр t)
        long t = (point.MicronsX - lineStart.MicronsX) * dx +
                 (point.MicronsY - lineStart.MicronsY) * dy;
        t = Math.Max(0, Math.Min(lengthSquared, t));

        // Ближайшая точка на отрезке
        long closestX = lineStart.MicronsX + (t * dx) / lengthSquared;
        long closestY = lineStart.MicronsY + (t * dy) / lengthSquared;

        return Distance(point, new PointMicrons(closestX, closestY));
    }

    private static long Distance(PointMicrons a, PointMicrons b)
    {
        long dx = a.MicronsX - b.MicronsX;
        long dy = a.MicronsY - b.MicronsY;
        return (long)Math.Sqrt(dx * dx + dy * dy);
    }
}
```

#### 3.1.D.4. Нюансы и пограничные случаи

| Проблема | Решение |
|----------|---------|
| **Тонкие линии** | Tolerance 5 мм (настраиваемый через константу `LineHitTolerance`) |
| **Пересекающиеся объекты** | Поиск с конца списка — верхний объект приоритетнее |
| **Повёрнутые объекты** | Нужно трансформировать точку в локальные координаты объекта (с учётом `AngleDegrees`) |
| **Горизонтальные/вертикальные линии** | Алгоритм проекции работает корректно для любых углов |
| **Нулевая длина линии** | Проверка `lengthSquared == 0` → расстояние до начальной точки |

#### 3.1.D.5. Производительность

| Параметр | Значение |
|----------|----------|
| **Сложность** | O(N) — линейный поиск |
| **1000 объектов** | ~5-10 мкс на один HitTest |
| **Частота вызовов** | Только при MouseDown (~1 раз/клик), НЕ при MouseMove |
| **Для N > 10 000** | Рекомендуется QuadTree или Spatial Hash |

**Для Этапа 1 (N ≤ 1000)** — линейный поиск абсолютно достаточен.

#### 3.1.D.6. Selection Box (рамка выделения)

```csharp
public static class SelectionBoxHelper
{
    /// <summary>
    /// LeftToRight: выделить только объекты, целиком попавшие в рамку.
    /// </summary>
    public static List<ITemplateObject> GetFullyContained(
        RectMicrons box, IList<ITemplateObject> objects)
    {
        var result = new List<ITemplateObject>();
        foreach (var obj in objects)
        {
            if (IsFullyInside(box, obj))
                result.Add(obj);
        }
        return result;
    }

    /// <summary>
    /// RightToLeft: выделить все задетые объекты.
    /// </summary>
    public static List<ITemplateObject> GetIntersecting(
        RectMicrons box, IList<ITemplateObject> objects)
    {
        var result = new List<ITemplateObject>();
        foreach (var obj in objects)
        {
            if (Intersects(box, obj))
                result.Add(obj);
        }
        return result;
    }
```

```csharp
public partial class MainViewModel : ObservableObject
{
    private readonly ITemplateService _templateService;
    private readonly IFileService _fileService;
    private readonly IDialogService _dialogService;

    /// <summary>
    /// Коллекция открытых вкладок.
    /// </summary>
    [ObservableProperty]
    private ObservableCollection<EditorViewModel> _openedTabs = new();

    /// <summary>
    /// Активная вкладка.
    /// </summary>
    [ObservableProperty]
    private EditorViewModel? _selectedTab;

    /// <summary>
    /// Создать новую вкладку (пустой шаблон).
    /// </summary>
    [RelayCommand]
    private void NewTab(string format)
    {
        var template = _templateService.CreateNew(format);
        var editorVm = new EditorViewModel(template);
        OpenedTabs.Add(editorVm);
        SelectedTab = editorVm;
    }

    /// <summary>
    /// Открыть файл в новой вкладке.
    /// </summary>
    [RelayCommand]
    private async Task OpenFileAsync()
    {
        var filePath = _fileService.OpenFileDialog("DotElectric Template|*.tdel");
        if (string.IsNullOrEmpty(filePath)) return;

        var template = await Task.Run(() => _templateService.Load(filePath));
        var editorVm = new EditorViewModel(template)
        {
            FilePath = filePath,
            IsDirty = false
        };
        OpenedTabs.Add(editorVm);
        SelectedTab = editorVm;
    }

    /// <summary>
    /// Закрыть вкладку (с проверкой несохранённых изменений).
    /// </summary>
    [RelayCommand]
    private async Task CloseTabAsync(EditorViewModel tab)
    {
        if (tab == null) return;

        if (tab.IsDirty)
        {
            var result = await _dialogService.ShowUnsavedChangesDialog(tab.DisplayName);
            if (result == UnsavedChangesResult.Cancel) return;
            if (result == UnsavedChangesResult.Save)
                await SaveTabAsync(tab);
        }

        OpenedTabs.Remove(tab);
        tab.Dispose();

        // Если закрыли активную — выбрать соседнюю
        if (SelectedTab == null && OpenedTabs.Count > 0)
            SelectedTab = OpenedTabs.Last();
    }

    private async Task SaveTabAsync(EditorViewModel tab)
    {
        if (string.IsNullOrEmpty(tab.FilePath))
        {
            tab.FilePath = _fileService.SaveFileDialog("DotElectric Template|*.tdel", "template.tdel");
            if (string.IsNullOrEmpty(tab.FilePath)) return;
        }

        _templateService.Save(tab.Template, tab.FilePath);
        tab.IsDirty = false;
    }
}
```

#### 3.1.F.4. EditorViewModel

Каждая вкладка — отдельный `EditorViewModel` с собственной историей Undo/Redo.

```csharp
public partial class EditorViewModel : ObservableObject, IDisposable
{
    private readonly Template _template;
    private bool _suppressDirty;

    public EditorViewModel(Template template)
    {
        _template = template;
        CommandHistory = new CommandHistory(maxLevels: 50);

    /// Модель шаблона.

    /// <summary>
    /// Путь к файлу (null если новый несохранённый).
    /// </summary>
    [ObservableProperty]
    private string? _filePath;

    /// <summary>
    /// Флаг несохранённых изменений.
    /// Устанавливается через Undo-команды, НЕ вручную.
    /// </summary>
    [ObservableProperty]
    private bool _isDirty;

    /// <summary>
    /// Выделенные объекты.
    /// </summary>
    [ObservableProperty]
    private List<ITemplateObject> _selectedObjects = new();

    /// <summary>
    /// Текущий масштаб (1.0 = 100%).
    /// </summary>
    [ObservableProperty]
    private double _zoom = 1.0;

    /// <summary>
    /// Настройки сетки.
    /// </summary>
    [ObservableProperty]
    private GridSettings _gridSettings = new();

    /// <summary>
    /// История Undo/Redo (своя для каждой вкладки).
    /// </summary>
    public CommandHistory CommandHistory { get; }

    /// <summary>
    /// Пометить как «грязный». Вызывается автоматически при Execute команды.
    /// </summary>
    public void MarkDirty()
    {
        if (!_suppressDirty)
            IsDirty = true;
    }

    /// <summary>
    /// Подавляет автоматическую установку IsDirty.
    /// Возвращает IDisposable для использования в using.
    /// Используется при загрузке из файла для предотвращения
    /// ложной установки IsDirty.
    /// </summary>
    internal LoadSuspension SuspendDirty()
    {
        _suppressDirty = true;
        return new LoadSuspension(this);
    }

    /// <summary>
    /// Scope подавления IsDirty (IDisposable).
    /// При Dispose автоматически сбрасывает _suppressDirty.
    /// </summary>
    internal readonly struct LoadSuspension : IDisposable
    {
        private readonly EditorViewModel _vm;
        public LoadSuspension(EditorViewModel vm) => _vm = vm;
        public void Dispose() => _vm._suppressDirty = false;
    }

    /// <summary>
    /// Подписка на изменения шаблона (для зума, сетки — не через Undo).
    /// </summary>
    private void SubscribeToChanges()
    {
        _template.Objects.CollectionChanged += (s, e) =>
        {
            if (!_suppressDirty) IsDirty = true;
        };
    }

    public void Dispose()
    {
        // Отписка от событий
    }
}
```

#### 3.1.F.5. IsDirty — механизм установки

| Тип изменения | Как устанавливается IsDirty |
|---------------|----------------------------|
| Добавление объекта | Через `AddObjectCommand.Execute()` → `editor.MarkDirty()` |
| Удаление объекта | Через `DeleteObjectCommand.Execute()` → `editor.MarkDirty()` |
| Перемещение объекта | Через `MoveObjectCommand.Execute()` → `editor.MarkDirty()` |
| Изменение свойств | Через `ChangePropertyCommand.Execute()` → `editor.MarkDirty()` |
| Загрузка из файла | Через `using (SuspendDirty())` — `IsDirty` остаётся `false` |
| Масштаб, сетка | НЕ устанавливают IsDirty |

**Ключевой принцип:** все изменения модели идут через Undo-команды. Каждая команда автоматически вызывает `editor.MarkDirty()`. Это гарантирует, что `IsDirty` никогда не забудут установить.

```csharp
class AddObjectCommand : ICommand
{
    private readonly EditorViewModel _editor;
    private readonly ITemplateObject _object;

    public void Execute()
    {
        _editor.Template.Objects.Add(_object);
        _editor.MarkDirty(); // ← автоматически
    }

    public void Undo()
    {
        _editor.Template.Objects.Remove(_object);
        _editor.MarkDirty(); // ← undo тоже делает «грязным»
    }
}
```

#### 3.1.G.3. AutosaveService

```csharp
/// <summary>
/// Сервис автосохранения открытых шаблонов.
/// Сохраняет все «грязные» вкладки каждые N минут.
/// </summary>
public class AutosaveService : IDisposable
{
    private readonly ITemplateService _templateService;
    private readonly ISettingsService _settingsService;
    private readonly ILogger<AutosaveService>? _logger;
    private Timer? _timer;
    private readonly string _autosaveFolder;
    private readonly SemaphoreSlim _saveLock = new(1, 1);
    private readonly string _sessionFile;

    public AutosaveService(
        ITemplateService templateService,
        ISettingsService settingsService,
        ILogger<AutosaveService>? logger = null)
    {
        _templateService = templateService;
        _settingsService = settingsService;
        _logger = logger;

        _autosaveFolder = Path.Combine(
            Environment.GetFolderPath(Environment.SpecialFolder.ApplicationData),
            "DotElectric", "autosave");

        _sessionFile = Path.Combine(_autosaveFolder, "session.json");
        Directory.CreateDirectory(_autosaveFolder);
    }

    /// <summary>
    /// Запустить таймер автосохранения.
    /// </summary>
    public void Start()
    {
        var intervalMinutes = _settingsService.Get("AutosaveIntervalMinutes", 5);
        if (intervalMinutes < 1) intervalMinutes = 1;
        if (intervalMinutes > 60) intervalMinutes = 60;

        _timer = new Timer(
            OnAutosaveTick,
            null,
            TimeSpan.FromMinutes(intervalMinutes),
            TimeSpan.FromMinutes(intervalMinutes));
    }

    /// <summary>
    /// Остановить таймер.
    /// </summary>
    public void Stop()
    {
        _timer?.Dispose();
        _timer = null;
    }

    private void OnAutosaveTick(object? state)
    {
        AutosaveTick?.Invoke();
    }

    /// <summary>
    /// Событие тика таймера. Подписывается MainViewModel.
    /// </summary>
    public event Action? AutosaveTick;

    /// <summary>
    /// Сохранить все «грязные» вкладки (вызывается из MainViewModel по событию).
    /// </summary>
    public async Task AutosaveAllTabsAsync(
        IEnumerable<EditorViewModel> openedTabs,
        CancellationToken ct = default)
    {
        await _saveLock.WaitAsync(ct);
        try
        {
            var session = new AutosaveSession
            {
                SessionStart = session.LastAutosave = DateTime.Now,
                Tabs = new List<AutosaveTabInfo>()
            };

            foreach (var tab in openedTabs.Where(t => t.IsDirty))
            {
                try
                {
                    var tabId = GetTabAutosaveId(tab);
                    var fileName = $"autosave_{tabId}_{DateTime.Now:yyyyMMdd_HHmmss}.tdel";
                    var filePath = Path.Combine(_autosaveFolder, fileName);

                    _templateService.Save(tab.Template, filePath);

                    // Удаляем предыдущий файл автосохранения для этой вкладки
                    CleanupOldAutosaveForTab(tabId);

                    session.Tabs.Add(new AutosaveTabInfo
                    {
                        TabId = tabId,
                        DisplayName = tab.DisplayName,
                        OriginalFilePath = tab.FilePath,
                        AutosaveFile = fileName,
                        WasDirty = true
                    });
                }
                catch (Exception ex)
                {
                    _logger?.LogError(ex,
                        "Autosave failed for tab: {TabName}", tab.DisplayName);
                }
            }

            // Сохраняем session.json
            if (session.Tabs.Count > 0)
            {
                var json = JsonSerializer.Serialize(session,
                    new JsonSerializerOptions { WriteIndented = true });
                await File.WriteAllTextAsync(_sessionFile, json, ct);
            }
            else
            {
                // Нет «грязных» вкладок — удаляем session.json
                if (File.Exists(_sessionFile))
                    File.Delete(_sessionFile);
            }

            // Очистка старых файлов (старше 7 дней)
            CleanupOldAutosaveFiles();
        }
        finally
        {
            _saveLock.Release();
        }
    }

    /// <summary>
    /// Получить уникальный ID автосохранения для вкладки.
    /// Использует TabId или создаёт новый на основе FilePath.
    /// </summary>
    private string GetTabAutosaveId(EditorViewModel tab)
    {
        return tab.TabId ?? 
               (string.IsNullOrEmpty(tab.FilePath)
                   ? Guid.NewGuid().ToString("N")[..8]
                   : Path.GetFileNameWithoutExtension(tab.FilePath));
    }

    /// <summary>
    /// Удалить предыдущие файлы автосохранения для данной вкладки.
    /// Хранится только последняя версия.
    /// </summary>
    private void CleanupOldAutosaveForTab(string tabId)
    {
        var oldFiles = Directory.GetFiles(_autosaveFolder, $"autosave_{tabId}_*.tdel");
        if (oldFiles.Length > 1)
        {
            // Удаляем все, кроме самого нового
            var newest = oldFiles.OrderByDescending(File.GetCreationTime).First();
            foreach (var file in oldFiles.Where(f => f != newest))
                File.Delete(file);
        }
    }

    /// <summary>
    /// Удалить файлы автосохранения старше 7 дней.
    /// </summary>
    private void CleanupOldAutosaveFiles()
    {
        var retentionDays = _settingsService.Get("AutosaveRetentionDays", 7);
        var cutoff = DateTime.Now.AddDays(-retentionDays);

        foreach (var file in Directory.GetFiles(_autosaveFolder, "*.tdel"))
        {
            if (File.GetCreationTime(file) < cutoff)
                File.Delete(file);
        }
    }

    /// <summary>
    /// Обновить session.json после ручного сохранения (файл больше не «грязный»).
    /// </summary>
    public async Task UpdateSessionAfterManualSaveAsync(string tabId)
    {
        if (!File.Exists(_sessionFile)) return;

        var session = JsonSerializer.Deserialize<AutosaveSession>(
            await File.ReadAllTextAsync(_sessionFile));

        session?.Tabs.RemoveAll(t => t.TabId == tabId);

        if (session?.Tabs.Count > 0)
        {
            var json = JsonSerializer.Serialize(session,
                new JsonSerializerOptions { WriteIndented = true });
            await File.WriteAllTextAsync(_sessionFile, json);
        }
        else
        {
            File.Delete(_sessionFile);
        }
    }

    /// <summary>
    /// Загрузить сессию автосохранения (для диалога восстановления).
    /// </summary>
    public AutosaveSession? LoadSession()
    {
        if (!File.Exists(_sessionFile)) return null;

        return JsonSerializer.Deserialize<AutosaveSession>(
            File.ReadAllText(_sessionFile));
    }

    /// <summary>
    /// Получить полный путь к файлу автосохранения.
    /// </summary>
    public string GetAutosaveFilePath(string autosaveFileName) =>
        Path.Combine(_autosaveFolder, autosaveFileName);

    /// <summary>
    /// Очистить всю папку автосохранения (после восстановления или вручную).
    /// </summary>
    public void ClearAutosaveFolder()
    {
        Stop();
        if (Directory.Exists(_autosaveFolder))
        {
            foreach (var file in Directory.GetFiles(_autosaveFolder))
                File.Delete(file);
        }
    }

    public void Dispose()
    {
        Stop();
        _saveLock.Dispose();
    }
}
```

#### 3.1.G.5. Восстановление после сбоя (App.xaml.cs)

```csharp
protected override async void OnStartup(StartupEventArgs e)
{
    base.OnStartup(e);

    // Блокировка второго экземпляра
    _mutex = new Mutex(true, "DotElectric_TemplateEditor", out bool createdNew);
    if (!createdNew)
    {
        _dialogService.ShowError("Приложение уже запущено.");
        Shutdown();
        return;
    }

    var mainVm = _container.GetRequiredService<MainViewModel>();
    var autosaveService = _container.GetRequiredService<AutosaveService>();

    // Проверяем наличие автосохранённой сессии
    var session = autosaveService.LoadSession();
    if (session?.Tabs.Count > 0)
    {
        var result = await _dialogService.ShowRecoveryDialog(session.Tabs.Count);

        if (result == RecoveryResult.Restore)
        {
            foreach (var tabInfo in session.Tabs)
            {
                var autosavePath = autosaveService.GetAutosaveFilePath(tabInfo.AutosaveFile);

                if (File.Exists(autosavePath))
                {
                    var template = _templateService.Load(autosavePath);
                    var editorVm = new EditorViewModel(template)
                    {
                        TabId = tabInfo.TabId,
                        FilePath = tabInfo.OriginalFilePath,
                        IsDirty = true
                    };
                    mainVm.OpenedTabs.Add(editorVm);
                }
            }

            if (mainVm.OpenedTabs.Count > 0)
                mainVm.SelectedTab = mainVm.OpenedTabs.First();

            // Очищаем папку автосохранения после восстановления
            autosaveService.ClearAutosaveFolder();
        }
        else
        {
            // Пользователь отказался — тоже очищаем
            autosaveService.ClearAutosaveFolder();
        }
    }

    // Подписываемся на тик автосохранения
    autosaveService.AutosaveTick += async () =>
    {
        await autosaveService.AutosaveAllTabsAsync(mainVm.OpenedTabs);
    };

    // Запускаем таймер
    autosaveService.Start();

    ShowMainWindow(mainVm);
}
```

#### 3.1.G.9. Edge cases

| Ситуация | Обработка |
|----------|-----------|
| **Ошибка сохранения** (диск полный) | Логируем ошибку, НЕ прерываем остальные вкладки |
| **Папка автосохранения недоступна** | Создаём заново, логируем предупреждение |
| **Автосохранение во время ручного сохранения** | `SemaphoreSlim` обеспечивает последовательный доступ |
| **Вкладка закрыта до автосохранения** | При закрытии вкладка исключается из `session.json` |
| **Автосохранение пустой коллекции** | `session.json` удаляется (нет «грязных» вкладок) |

### 3.1.H. Логирование

#### 3.1.H.1. Общие принципы

| Параметр | Значение |
|----------|----------|
| **Библиотека** | **Serilog** (структурное логирование) |
| **Абстракция** | `Microsoft.Extensions.Logging.ILogger<T>` (Serilog как провайдер) |
| **Минимальный уровень** | **Debug** (Debug, Warning, Error, Fatal). **Information не логируется** |
| **Sink: Файл** | Rolling file, ежедневная ротация, 30 файлов |
| **Sink: Консоль** | **Не используется** |
| **Папка логов** | `%APPDATA%\DotElectric\logs\` |
| **Имя файла** | `dotelectric-{yyyyMMdd}.txt` |

#### 3.1.H.2. NuGet-пакеты


#### 3.1.H.3. Уровни логирования

| Уровень | Когда используется | Пример |
|---------|-------------------|--------|
| **Debug** | Детали работы приложения (для разработчиков, включается при отладке) | `Загружен шаблон из файла {FilePath}, объектов: {Count}` |
| **Warning** | Потенциальные проблемы, но приложение работает | `Файл автосохранения не найден, создаём новый` |
| **Error** | Ошибки, но приложение продолжает работу | `Ошибка автосохранения вкладки {TabName}: {Message}` |
| **Fatal** | Критическая ошибка, приложение не может продолжать | `Не удалось создать папку автосохранения` |

**Information не логируется** — пропускаем информационные сообщения, чтобы логи содержали только важные события и проблемы.

#### 3.1.H.8. Где логируем

| Компонент | Что логируем | Уровень |
|-----------|-------------|---------|
| **App.xaml.cs** | Запуск, завершение, глобальные ошибки | Debug / Fatal |
| **AutosaveService** | Тик таймера, сохранение, ошибки | Debug / Error |
| **TemplateService** | Загрузка/сохранение файлов, ошибки валидации | Debug / Error |
| **FileService** | Ошибки открытия/сохранения файлов | Debug / Warning / Error |
| **EditorViewModel** | Создание/закрытие вкладок, ошибки восстановления | Debug / Warning |
| **CommandHistory** | Только ошибки (ошибка undo/redo) | Error |

#### 3.1.H.9. Что НЕ логируем

| Не логируем | Причина |
|-------------|---------|
| Пароли, токены, секреты | Безопасность (SEC-002) |
| Полное содержимое XML-файлов | Производительность, приватность |
| Координаты каждого объекта при каждом изменении | Спам логами |
| Mouse events (click, move, и т.д.) | ~120 событий/сек, спам |
| Zoom/Pan события | Спам |
| Information-сообщения | По решению — только Debug/Warning/Error/Fatal |

#### 3.1.H.10. Ротация и очистка

| Параметр | Значение |
|----------|----------|
| **Ротация** | Ежедневно (`RollingInterval.Day`) |
| **Хранение** | Последние 30 файлов |
| **Имя файла** | `dotelectric-20260403.txt`, `dotelectric-20260404.txt`, ... |
| **Размер** | Не ограничен (Serilog не режет по размеру при daily ротации) |
| **Общий объём** | ~30 файлов × ~1-5 МБ/день ≈ 30-150 МБ максимум |

#### 3.1.H.11. Структурированное логирование

Serilog поддерживает структурное логирование — параметры передаются отдельно от сообщения:

```csharp
// ✅ Правильно (структурное логирование)
_logger.LogError(ex, "Ошибка загрузки файла {FilePath}", filePath);

// ❌ Неправильно (конкатенация строк)
_logger.LogError("Ошибка загрузки файла " + filePath + ": " + ex.Message);
```

**Преимущества:**
- Параметры можно фильтровать/группировать при анализе
- Нет проблем с форматированием
- Легче парсить автоматически (если позже подключить Seq)

#### 3.1.H.12. Включение/отключение логирования

Логирование **всегда включено** (нельзя отключить пользователем). Однако:

| Сценарий | Поведение |
|----------|-----------|
| **Ошибка записи лога** | Serilog проглатывает ошибку silently (SelfLog можно включить для дебага) |
| **Диск переполнен** | Логи не пишутся, приложение продолжает работу |
| **Нет доступа к папке** | Логи не пишутся, предупреждение в UI при запуске |

### 3.1.I. Undo/Redo — детальный дизайн команд

#### 3.1.I.1. Общие принципы

| Параметр | Значение |
|----------|----------|
| **Интерфейс** | `ICommand` (не зависит от `System.Windows.Input.ICommand`) |
| **Менеджер истории** | `CommandHistory` (до 50+ уровней) |
| **Составные операции** | `BatchCommand` (группировка нескольких команд) |
| **IsDirty** | Устанавливается автоматически командами, НЕ вручную |
| **Checkpoint** | История НЕ очищается при сохранении |
| **Обработка ошибок** | Rollback при failed undo, логирование через Serilog |

**Ключевые принципы:**
1. Все изменения модели идут **только** через Undo-команды
2. Каждая команда автоматически вызывает `editor.MarkDirty()` после выполнения
3. Redo-стек очищается при выполнении новой команды (новая ветка истории)
4. При превышении лимита (50) — обрезаются oldest команды из undo-стека
5. Undo/Redo работают только из UI-потока (потокобезопасность не требуется)

#### 3.1.I.2. Базовый интерфейс команды

```csharp
namespace DotElectric.TemplateEditor.Commands;

/// <summary>
/// Базовый интерфейс для Undo/Redo команд.
/// НЕ зависит от System.Windows.Input.ICommand (это для UI binding).
/// </summary>
public interface ICommand
{
    /// <summary>
    /// Выполнить действие команды.
    /// Вызывается при Execute из CommandHistory.
    /// </summary>
    void Execute();

    /// <summary>
    /// Отменить действие команды.
    /// Вызывается при Undo из CommandHistory.
    /// Должен восстановить состояние точно таким, каким оно было до Execute().
    /// </summary>
    void Undo();

    /// <summary>
    /// Читаемое имя команды для отображения в UI (например, в меню Undo).
    /// Пример: "Добавить линию", "Удалить прямоугольник", "Переместить текст".
    /// </summary>
    string Name { get; }
}
```

#### 3.1.I.3. CommandHistory — менеджер истории

```csharp
namespace DotElectric.TemplateEditor.Commands;

/// <summary>
/// Менеджер истории Undo/Redo с ограничением по уровням.
/// Потокобезопасность не требуется (вызовы только из UI-потока).
/// </summary>
public class CommandHistory
{
    private readonly List<ICommand> _undoStack = new();
    private readonly List<ICommand> _redoStack = new();
    private readonly int _maxLevels;

    /// <summary>
    /// Конструктор с ограничением по уровням.
    /// </summary>
    /// <param name="maxLevels">Максимальное количество уровней (рекомендуется 50+).</param>
    public CommandHistory(int maxLevels = 50)
    {
        _maxLevels = maxLevels;
    }

    /// <summary>
    /// Количество доступных уровней Undo.
    /// </summary>
    public int UndoCount => _undoStack.Count;

    /// <summary>
    /// Количество доступных уровней Redo.
    /// </summary>
    public int RedoCount => _redoStack.Count;

    /// <summary>
    /// Может ли выполнить Undo.
    /// </summary>
    public bool CanUndo => _undoStack.Count > 0;

    /// <summary>
    /// Может ли выполнить Redo.
    /// </summary>
    public bool CanRedo => _redoStack.Count > 0;

    /// <summary>
    /// Имя последней команды для отображения в меню Undo.
    /// </summary>
    public string? LastUndoName => _undoStack.Count > 0 ? _undoStack[^1].Name : null;

    /// <summary>
    /// Имя последней команды для отображения в меню Redo.
    /// </summary>
    public string? LastRedoName => _redoStack.Count > 0 ? _redoStack[^1].Name : null;

    /// <summary>
    /// Выполнить команду и добавить в историю.
    /// Redo-стек очищается автоматически (стандартное поведение).
    /// </summary>
    public void ExecuteCommand(ICommand command)
    {
        command.Execute();
        _undoStack.Add(command);
        _redoStack.Clear(); // Новая ветка истории

        // Обрезка если превышен лимит
        TrimHistory();
    }

    /// <summary>
    /// Отменить последнюю команду.
    /// Если стек пуст — nothing to do, исключение НЕ выбрасывается.
    /// </summary>
    public void Undo()
    {
        if (!CanUndo) return;

        var command = _undoStack[^1];
        _undoStack.RemoveAt(_undoStack.Count - 1);

        try
        {
            command.Undo();
            _redoStack.Add(command);
        }
        catch (Exception ex)
        {
            // Rollback: команда возвращается в undo-стек
            _undoStack.Add(command);
            throw new InvalidOperationException($"Ошибка при отмене команды '{command.Name}': {ex.Message}", ex);
        }
    }

    /// <summary>
    /// Повторить последнюю отменённую команду.
    /// Если стек пуст — nothing to do, исключение НЕ выбрасывается.
    /// </summary>
    public void Redo()
    {
        if (!CanRedo) return;

        var command = _redoStack[^1];
        _redoStack.RemoveAt(_redoStack.Count - 1);

        try
        {
            command.Execute();
            _undoStack.Add(command);
            TrimHistory();
        }
        catch (Exception ex)
        {
            // Rollback: команда возвращается в redo-стек
            _redoStack.Add(command);
            throw new InvalidOperationException($"Ошибка при повторе команды '{command.Name}': {ex.Message}", ex);
        }
    }
        catch (Exception ex)
        {
            // Восстановить команду обратно в undo-стек при ошибке
            _undoStack.Add(command);
            throw new InvalidOperationException($"Undo failed for '{command.Name}': {ex.Message}", ex);
        }
    }

    /// <summary>
    /// Повторить последнюю отменённую команду.
    /// </summary>
    public void Redo()
    {
        if (!CanRedo)
            throw new InvalidOperationException("Redo stack is empty.");

        var command = _redoStack[^1];
        _redoStack.RemoveAt(_redoStack.Count - 1);
        
        try
        {
            command.Execute();
            _undoStack.Add(command);
        }
        catch (Exception ex)
        {
            _redoStack.Add(command);
            throw new InvalidOperationException($"Redo failed for '{command.Name}': {ex.Message}", ex);
        }
    }

    /// <summary>
    /// Обрезать историю если превышен лимит.
    /// Удаляются самые старые команды из undo-стека.
    /// </summary>
    private void TrimHistory()
    {
        while (_undoStack.Count > _maxLevels)
        {
            _undoStack.RemoveAt(0); // Удаляем oldest (index 0)
        }
    }

    /// <summary>
    /// Очистить всю историю.
    /// Вызывается при загрузке нового файла или создании нового шаблона.
    /// </summary>
    public void Clear()
    {
        _undoStack.Clear();
        _redoStack.Clear();
    }
}
```

**BatchCommand (составная операция):**
```csharp
public class BatchCommand : ICommand
{
    private readonly List<ICommand> _subCommands = new();
    public string Name => "Пакетная операция";

    /// <summary>
    /// Добавить подкоманду. Вызывается ДО Execute().
    /// </summary>
    public void Add(ICommand cmd) => _subCommands.Add(cmd);

    public void Execute()
    {
        foreach (var cmd in _subCommands)
            cmd.Execute();
    }

    public void Undo()
    {
        // Отменяем в обратном порядке (LIFO)
        for (int i = _subCommands.Count - 1; i >= 0; i--)
            _subCommands[i].Undo();
    }
}
```

#### 3.1.I.6. Интеграция с EditorViewModel

```csharp
public partial class EditorViewModel : ObservableObject
{
    private readonly CommandHistory _commandHistory;
    private bool _isDirty;
    private bool _suppressDirty;

    public CommandHistory CommandHistory => _commandHistory;

    [ObservableProperty]
    private string? _displayName;

    public bool IsDirty
    {
        get => _isDirty;
        private set => SetProperty(ref _isDirty, value);
    }

    public EditorViewModel(Template template)
    {
        Template = template ?? throw new ArgumentNullException(nameof(template));
        _commandHistory = new CommandHistory(maxLevels: 50);
        DisplayName = template.Metadata.Name;
        IsDirty = false;
    }

    /// <summary>
    /// Установить флаг "грязный". Вызывается автоматически командами.
    /// </summary>
    internal void MarkDirty()
    {
        if (!_suppressDirty)
        {
            IsDirty = true;
        }
    }

    /// <summary>
    /// Подавляет автоматическую установку IsDirty.
    /// Возвращает IDisposable для использования в using.
    /// </summary>
    internal LoadSuspension SuspendDirty()
    {
        _suppressDirty = true;
        return new LoadSuspension(this);
    }

    internal readonly struct LoadSuspension : IDisposable
    {
        private readonly EditorViewModel _vm;
        public LoadSuspension(EditorViewModel vm) => _vm = vm;
        public void Dispose() => _vm._suppressDirty = false;
    }

    // === Команды для UI (RelayCommand для XAML binding) ===

    [RelayCommand(CanExecute = nameof(CanUndo))]
    private void Undo()
    {
        _commandHistory.Undo();
        MarkDirty(); // Undo тоже делает грязным
        OnPropertyChanged(nameof(CanUndo));
        OnPropertyChanged(nameof(CanRedo));
    }

    [RelayCommand(CanExecute = nameof(CanRedo))]
    private void Redo()
    {
        _commandHistory.Redo();
        MarkDirty();
        OnPropertyChanged(nameof(CanUndo));
        OnPropertyChanged(nameof(CanRedo));
    }

    private bool CanUndo() => _commandHistory.CanUndo;
    private bool CanRedo() => _commandHistory.CanRedo;

    /// <summary>
    /// Получить имя для меню Undo (например: "↶ Переместить объект").
    /// </summary>
    public string UndoDisplayName => 
        _commandHistory.CanUndo 
            ? $"↶ {_commandHistory.LastUndoName}" 
            : "↶ Отменить";

    /// <summary>
    /// Получить имя для меню Redo (например: "↷ Переместить объект").
    /// </summary>
    public string RedoDisplayName => 
        _commandHistory.CanRedo 
            ? $"↷ {_commandHistory.LastRedoName}" 
            : "↷ Повторить";
}
```

```csharp
public class SelectTool : ITool
{
    private readonly EditorViewModel _editor;
    private ITemplateObject? _draggedObject;
    private long _dragStartX, _dragStartY;

    public void OnMouseDown(PointMicrons modelPoint, MouseButton button, ModifierKeys modifiers)
    {
        var hit = HitTestHelper.HitTest(_editor.Template.Objects, modelPoint);
        
        if (hit != null)
        {
            _draggedObject = hit;
            _dragStartX = modelPoint.MicronsX;
            _dragStartY = modelPoint.MicronsY;
            _editor.Selection.Clear();
            _editor.Selection.Add(hit);
        }
    }

    public void OnMouseUp(PointMicrons modelPoint, MouseButton button, ModifierKeys modifiers)
    {
        if (_draggedObject != null)
        {
            var dx = modelPoint.MicronsX - _dragStartX;
            var dy = modelPoint.MicronsY - _dragStartY;

            if (Math.Abs(dx) > 0 || Math.Abs(dy) > 0)
            {
                var newX = _draggedObject.MicronsX + dx;
                var newY = _draggedObject.MicronsY + dy;
                
                var cmd = new MoveObjectCommand(_draggedObject, newX, newY);
                _editor.CommandHistory.ExecuteCommand(cmd);
                _editor.MarkDirty();
            }
            
            _draggedObject = null;
        }
    }
}
```

#### 3.1.I.8. Checkpoint при сохранении

| Тип операции | IsDirty | История Undo/Redo |
|--------------|---------|-------------------|
| Добавление объекта | Устанавливается | Добавляется в undo-стек |
| Удаление объекта | Устанавливается | Добавляется в undo-стек |
| Перемещение объекта | Устанавливается | Добавляется в undo-стек |
| Изменение свойств | Устанавливается | Добавляется в undo-стек |
| Загрузка из файла | Сбрасывается (`using SuspendDirty()`) | Очищается (`CommandHistory.Clear()`) |
| Сохранение в файл | Сбрасывается | **НЕ очищается** (checkpoint) |

```csharp
public class FileService : IFileService
{
    private readonly ITemplateService _templateService;

    public void Save(EditorViewModel editor, string filePath)
    {
        // Сохраняем в файл
        _templateService.Save(editor.Template, filePath);
        
        // Флаг IsDirty сбрасывается, НО история НЕ очищается
        // Это позволяет Undo после сохранения (checkpoint-подход)
        editor.IsDirty = false;
        
        // Путь к файлу обновляется
        editor.FilePath = filePath;
        editor.DisplayName = Path.GetFileName(filePath);
    }
}
```

> **Принцип:** история **НЕ** очищается при сохранении. Пользователь может сохранить файл, затем отменить изменения и сохранить снова (более ранняя версия). Это стандартный checkpoint-подход.

#### 3.1.I.10. Обработка ошибок

| Сценарий | Поведение |
|----------|-----------|
| Undo при пустом стеке | `InvalidOperationException` (не должно произойти — CanUndo disabled) |
| Ошибка в команде.Undo() | Восстановить стек, логировать через Serilog, показать MessageBox |
| Ошибка в команде.Execute() | НЕ добавлять в историю, логировать, показать MessageBox |
| Переполнение истории (>50) | Автоматическая обрезка oldest (TrimHistory) |
| Память | 50 команд × ~100 bytes каждая = ~5 KB (пренебрежимо мало) |

### 3.1.J. Маркеры выделения (Selection Adorners)

#### 3.1.J.1. Общие принципы

| Параметр | Значение |
|----------|----------|
| **Отрисовка** | Canvas Children (Rectangle/Ellipse элементы) |
| **Размер маркеров** | 6×6px фиксированный (не масштабируется с zoom) |
| **Hit-test зона** | 12×12px (в 2 раза больше визуала для удобства) |
| **Стиль маркеров** | Белый квадрат/круг, синяя обводка `#0078D4` 1px |
| **Подсветка** | Пунктирная синяя обводка (`DashArray="4,2"`, 1.5px) |
| **Multi-selection** | Только подсветка, маркеры НЕ отображаются |
| **Вращение** | R = +90°, Shift+R = -90°, панель свойств — ручной ввод |
| **Ресайз модификаторы** | Shift = пропорции, Ctrl = от центра |
| **Мин. размер** | 1 мм (1000 микрон) |
| **Привязка при ресайзе** | SnapToGrid если включена |

**Ключевые принципы:**
1. Маркеры отображаются **только при single-selection** (один выделенный объект)
2. При multi-selection — только пунктирная подсветка вокруг каждого объекта
3. Hit-testing маркеров имеет **приоритет** над hit-testing объектов (проверяются первыми)
4. Все изменения размеров/углов идут через **Undo-команды** (`ResizeObjectCommand`, `RotateObjectCommand`)
5. При ресайзе используется **ToolStack**: `PushTool(ResizeTool)` → `PopTool()`

#### 3.1.J.5. ResizeTool — логика работы

**Поток событий:**

```
MouseDown на маркере → EditorCanvas.PushTool(ResizeTool)
                       ↓
               ResizeTool.OnMouseDown()
               - Сохраняет начальные координаты мыши
               - Сохраняет начальные размеры объекта
               - Определяет направление ресайза (какой маркер)
                       ↓
               MouseMove → ResizeTool.OnMouseMove()
               - Вычисляет delta (текущая - начальная)
               - Если Shift → сохраняет пропорции
               - Если Ctrl → ресайз от центра
               - Проверяет минимальный размер (1 мм)
               - Обновляет preview-отрисовку
                       ↓
               MouseUp → ResizeTool.OnMouseUp()
               - Создаёт ResizeObjectCommand
               - CommandHistory.ExecuteCommand(cmd)
               - EditorViewModel.MarkDirty()
               - EditorCanvas.PopTool()
```

**Базовый ресайз (без модификаторов):**

```csharp
// Тянем SE маркер (нижний правый)
newWidth = oldWidth + deltaX;
newHeight = oldHeight + deltaY;
// X, Y не меняются (верхний левый угол фиксирован)
```

**Shift — сохранение пропорций:**

```csharp
// Вычисляем aspect ratio
aspectRatio = oldWidth / (double)oldHeight;

// Применяем по большей оси
delta = Math.Max(Math.Abs(deltaX), Math.Abs(deltaY));

newWidth = oldWidth + delta * Math.Sign(deltaX);
newHeight = (long)(newWidth / aspectRatio); // Сохраняем пропорции
```

**Ctrl — ресайз от центра:**

```csharp
// Обе стороны двигаются симметрично
newX = oldX - deltaX / 2;
newY = oldY - deltaY / 2;
newWidth = oldWidth + deltaX;
newHeight = oldHeight + deltaY;
```

**Минимальный размер:**

```csharp
private const long MinSizeMicrons = 1000; // 1 мм

newWidth = Math.Max(newWidth, MinSizeMicrons);
newHeight = Math.Max(newHeight, MinSizeMicrons);
```

**Привязка к сетке при ресайзе:**

```csharp
if (_grid.SnapEnabled)
{
    newWidth = SnapHelper.SnapToGrid(newWidth, _grid.StepMicrons);
    newHeight = SnapHelper.SnapToGrid(newHeight, _grid.StepMicrons);
}
```

#### 3.1.J.7. Hit-testing маркеров

**Проблема:** маркеры фиксированного размера (6px), но при zoom 10% они становятся очень маленькими на холсте.

**Решение:** hit-testing зона больше визуального маркера (12×12px).

```csharp
public static class MarkerHitTest
{
    private const double HitTestRadiusPx = 6; // 12×12px зона

    /// <summary>
    /// Проверить попадание мыши в маркер.
    /// wpfMousePoint — WPF-координаты мыши.
    /// Возвращает null если промах.
    /// </summary>
    public static SelectionHandle? HitTest(
        Point wpfMousePoint,
        IReadOnlyList<SelectionHandle> handles)
    {
        // Проверяем с конца (Z-order: последние = сверху)
        for (int i = handles.Count - 1; i >= 0; i--)
        {
            var handle = handles[i];
            var dist = Distance(wpfMousePoint, handle.WpfPosition);

            if (dist <= HitTestRadiusPx)
                return handle;
        }
        return null;
    }

    private static double Distance(Point a, Point b)
        => Math.Sqrt(Math.Pow(a.X - b.X, 2) + Math.Pow(a.Y - b.Y, 2));
}
```

**Приоритет:** Hit-testing маркеров выполняется **до** hit-testing объектов. Если попали в маркер — начинается ресайз, а не выделение/перемещение.

#### 3.1.J.10. ResizeObjectCommand

```csharp
public class ResizeObjectCommand : Commands.ICommand
{
    private readonly ITemplateObject _object;
    private readonly long _oldX, _oldY;
    private readonly long _oldWidth, _oldHeight;
    private readonly long _newX, _newY;
    private readonly long _newWidth, _newHeight;

    public string Name => "Изменить размер";

    public ResizeObjectCommand(
        ITemplateObject obj,
        long newX, long newY,
        long newWidth, long newHeight)
    {
        _object = obj ?? throw new ArgumentNullException(nameof(obj));
        _oldX = obj.MicronsX;
        _oldY = obj.MicronsY;
        _oldWidth = GetObjectWidth(obj);
        _oldHeight = GetObjectHeight(obj);
        _newX = newX;
        _newY = newY;
        _newWidth = newWidth;
        _newHeight = newHeight;
    }

    public void Execute() => Apply(_newX, _newY, _newWidth, _newHeight);
    public void Undo() => Apply(_oldX, _oldY, _oldWidth, _oldHeight);

    private void Apply(long x, long y, long width, long height)
    {
        switch (_object)
        {
            case Rectangle rect:
                rect.MicronsX = x;
                rect.MicronsY = y;
                rect.WidthMicrons = width;
                rect.HeightMicrons = height;
                break;

            case Text text:
                text.MicronsX = x;
                text.MicronsY = y;
                text.WidthMicrons = width;
                text.HeightMicrons = height;
                break;

            case Line line:
                ApplyLineResize(line, x, y, width, height);
                break;
        }
    }

    private static void ApplyLineResize(
        Line line, long x, long y, long width, long height)
    {
        // При ресайзе линии один конец фиксирован (противоположный маркеру).
        // Если ресайз за Start — Start меняется, End фиксирован.
        // Если ресайз за End — End меняется, Start фиксирован.
        // ResizeTool определяет какой маркер тянется и передаёт
        // соответствующие координаты. Здесь просто присваиваем.
        line.StartMicrons = new PointMicrons(x, y);
        line.EndMicrons = new PointMicrons(x + width, y + height);
    }

    private static long GetObjectWidth(ITemplateObject obj)
    {
        return obj switch
        {
            Rectangle r => r.WidthMicrons,
            Text t => t.WidthMicrons,
            Line l => Math.Abs(l.EndMicrons.MicronsX - l.StartMicrons.MicronsX),
            _ => 0
        };
    }

    private static long GetObjectHeight(ITemplateObject obj)
    {
        return obj switch
        {
            Rectangle r => r.HeightMicrons,
            Text t => t.HeightMicrons,
            Line l => Math.Abs(l.EndMicrons.MicronsY - l.StartMicrons.MicronsY),
            _ => 0
        };
    }
}
```

#### 3.1.J.14. Ограничения и edge cases

| Сценарий | Поведение |
|----------|-----------|
| Ресайз до минимального размера | Объект не может стать < 1 мм. Delta игнорируется. |
| Multi-selection + R | Вращаются все выделенные объекты (BatchCommand) |
| Ресайз за границы листа | Запрещено (валидация в ResizeObjectCommand) |
| Ресайз текста | Bounding box меняется, размер шрифта НЕ меняется |

#### 3.1.K.1. Общие принципы

| Параметр | Значение |
|----------|----------|
| **Composition Root** | `App.xaml.cs` |
| **DI-пакет** | `Microsoft.Extensions.DependencyInjection` |

**Ключевые принципы:**
1. Все сервисы — **Singleton** (один экземпляр на всё приложение)

4. `App.xaml.cs` — **Composition Root**, конфигурирует и запускает всё
5. Глобальные обработчики — в `App.xaml.cs` (AppDomain, Dispatcher, Task)
6. Автосохранение — проверка при запуске **до** показа окна

```csharp
        var templateService = _serviceProvider.GetRequiredService<ITemplateService>();
        var resolvedPrintService = printService ?? _serviceProvider.GetService<IPrintService>();

        return new EditorViewModel(
            template,
            templateService,
            gridSettings,
            resolvedPrintService);

        var templateService = _serviceProvider.GetRequiredService<ITemplateService>();
        var resolvedPrintService = printService ?? _serviceProvider.GetService<IPrintService>();

        return new EditorViewModel(
            template,
            filePath,
            templateService,
            gridSettings,
            resolvedPrintService);
    }
```

```csharp
    /// <summary>
    /// Проверить папку автосохранения и предложить восстановление.

    private async Task CheckAutosaveRecoveryAsync(MainViewModel mainVm)
    {
        var autosaveFolder = Path.Combine(
            Environment.GetFolderPath(
                Environment.SpecialFolder.ApplicationData),
            "DotElectric", "autosave");

        if (!Directory.Exists(autosaveFolder))
            return;

        var autosaveFiles = Directory.GetFiles(autosaveFolder, "*.tdel");
        if (autosaveFiles.Length == 0)
            return;

        var dialogService = _serviceProvider
            .GetRequiredService<IDialogService>();
        var templateService = _serviceProvider
            .GetRequiredService<ITemplateService>();

        if (await dialogService.ShowRecoveryDialogAsync())
        {
            foreach (var file in autosaveFiles)
            {
                try
                {
                    var template = templateService.Load(file);
                    var factory = _serviceProvider
                        .GetRequiredService<EditorViewModelFactory>();
                    var editorVm = factory.Create(template);
                    // Всегда грязный после восстановления
                    editorVm.IsDirty = true;
                    mainVm.OpenedTabs.Add(editorVm);
                }
                catch (Exception ex)
                {
                    Log.Error(ex,
                        "Ошибка восстановления файла {FilePath}", file);
                }
            }

            // Очистка папки автосохранения
            try
            {
                Directory.Delete(autosaveFolder, recursive: true);
                Log.Debug("Папка автосохранения очищена");
            }
            catch (Exception ex)
            {
                Log.Warning(ex,
                    "Не удалось очистить папку автосохранения");
            }
        }
        else
        {
            // Пользователь отказался — очистить папку
            try
            {
                Directory.Delete(autosaveFolder, recursive: true);
            }
            catch (Exception ex)
            {
                Log.Warning(ex,
                    "Не удалось очистить папку автосохранения");
            }
        }
    }

                "DotElectric",
                MessageBoxButton.OK,
                MessageBoxImage.Error);
        };

        {
            Log.Error(e.Exception,
                "Незамеченное исключение задачи");
            e.SetObserved();
        };

{
```

```csharp
    private ObservableCollection<EditorViewModel>

    private EditorViewModel? _selectedTab;

    public MainViewModel(
        ITemplateService templateService,
        IFileService fileService,
        IDialogService dialogService,
        EditorViewModelFactory editorFactory)
    {
        _templateService = templateService
            ?? throw new ArgumentNullException(nameof(templateService));
        _fileService = fileService
            ?? throw new ArgumentNullException(nameof(fileService));
        _dialogService = dialogService
            ?? throw new ArgumentNullException(nameof(dialogService));
        _editorFactory = editorFactory
            ?? throw new ArgumentNullException(nameof(editorFactory));
    }

    /// <summary>
    /// Создать новую вкладку (пустой шаблон).
    /// </summary>
    [RelayCommand]
    private void NewTab(string format = "A3")
    {
        var editorVm = _editorFactory.CreateNew(format);
        OpenedTabs.Add(editorVm);
        SelectedTab = editorVm;

        Log.Debug("Создана новая вкладка: {Format}", format);
    }

    /// <summary>
    /// Открыть файл в новой вкладке.
    /// </summary>
    [RelayCommand]
    private async Task OpenFileAsync()
    {
        var filePath = _fileService.OpenFileDialog(
            "DotElectric Template|*.tdel");
        if (string.IsNullOrEmpty(filePath)) return;

        try
        {
            var editorVm = _editorFactory.CreateFromFile(filePath);
            OpenedTabs.Add(editorVm);
            SelectedTab = editorVm;

            Log.Information("Открыт файл: {FilePath}", filePath);
        }
        catch (Exception ex)
        {
            Log.Error(ex,
                "Ошибка открытия файла {FilePath}", filePath);
            _dialogService.ShowError(
                $"Не удалось открыть файл: {ex.Message}");
        }
    }

    /// <summary>
    /// Закрыть вкладку.
    /// </summary>
    [RelayCommand]
    private async Task CloseTabAsync(EditorViewModel tab)
    {
        if (tab == null) return;

        if (tab.IsDirty)
        {
            var result =
                await _dialogService.ShowUnsavedChangesDialog(
                    tab.DisplayName);
            if (result == UnsavedChangesResult.Cancel) return;
            if (result == UnsavedChangesResult.Save)
                await SaveTabAsync(tab);
        }

        OpenedTabs.Remove(tab);
        tab.Dispose();

        // Если закрыли активную вкладку — выбрать другую
        if (SelectedTab == tab)
            SelectedTab = OpenedTabs.LastOrDefault();

        Log.Debug("Закрыта вкладка: {TabName}", tab.DisplayName);
    }
```

- **Версионирование DTO**: `TemplateDto.Version` (по умолчанию "1.0") + метод `MigrateDto()` для миграции будущих версий (вызывается при `Load` перед маппингом).
- Формат `.tdel`: XML в ZIP архиве,micron-значения сохраняются как `xs:long`.

### 3.4. Валидация

#### 3.4.1. Правила валидации XML

| Правило | Описание |
|---------|----------|
| V-001 | Все ID объектов должны быть уникальными |
| V-002 | Все ключи (Key) текстовых полей должны быть уникальными в пределах шаблона | Проверяет дубликаты Text.Key при сохранении; дубликаты отвергаются |
| V-003 | Координаты объектов должны быть в пределах листа | |
| V-004 | Размеры объектов должны быть положительными |
| V-005 | Цвета должны быть в формате HEX (#RRGGBB, #AARRGGBB или Transparent) | V-005 активирован — ValidateHexColor() проверяет формат при сохранении |
| V-006 | Формат листа должен быть корректным |
| V-007 | Тип линии должен быть корректным (Solid, Dash, DashDot, DashDotDotDot) |

## 6. ТРЕБОВАНИЯ К ФОРМАТУ ФАЙЛА

### 6.1. XML Schema (XSD)

> **Важно:** Все координаты и размеры хранятся в **микронах** (целые числа, `xs:long`). 1 мм = 1000 микрон. Исключение: `FontSize` (пункты, `xs:double`).

```xml
<?xml version="1.0" encoding="utf-8"?>
<!-- Все координаты и размеры — в микронах (1 мм = 1000 микрон).
     Fixed-Point: целочисленные значения для исключения погрешностей. -->
<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema"
           targetNamespace="http://dotelectric.local/schema/template"
           xmlns:tns="http://dotelectric.local/schema/template">

  <xs:element name="Template" type="tns:TemplateType"/>

  <xs:complexType name="TemplateType">
    <xs:sequence>
      <xs:element name="Metadata" type="tns:MetadataType"/>
      <xs:element name="Sheet" type="tns:SheetType"/>
      <xs:element name="Grid" type="tns:GridType"/>
      <xs:element name="Objects" type="tns:ObjectsType"/>
    </xs:sequence>
    <xs:attribute name="version" type="xs:string" use="required"/>
  </xs:complexType>

  <xs:complexType name="MetadataType">
    <xs:sequence>
      <xs:element name="Name" type="xs:string"/>
      <xs:element name="Description" type="xs:string" minOccurs="0"/>
      <xs:element name="Author" type="xs:string" minOccurs="0"/>
      <xs:element name="CreatedDate" type="xs:dateTime"/>
      <xs:element name="ModifiedDate" type="xs:dateTime"/>
    </xs:sequence>
  </xs:complexType>

  <xs:complexType name="SheetType">
    <xs:sequence>
      <xs:element name="Format" type="xs:string"/>
      <xs:element name="Width" type="xs:long"/>
      <xs:element name="Height" type="xs:long"/>
      <xs:element name="Unit" type="xs:string" fixed="mm"/>
    </xs:sequence>
  </xs:complexType>

  <xs:complexType name="GridType">
    <xs:sequence>
      <xs:element name="Enabled" type="xs:boolean"/>
      <xs:element name="SnapEnabled" type="xs:boolean"/>
      <xs:element name="Step" type="xs:long"/>
      <xs:element name="Visible" type="xs:boolean"/>
    </xs:sequence>
  </xs:complexType>

  <xs:complexType name="ObjectsType">
    <xs:choice maxOccurs="unbounded">
      <xs:element name="Line" type="tns:LineType"/>
      <xs:element name="Rectangle" type="tns:RectangleType"/>
      <xs:element name="Text" type="tns:TextType"/>
    </xs:choice>
  </xs:complexType>

  <xs:complexType name="LineType">
    <xs:sequence>
      <xs:element name="Start" type="tns:PointType"/>
      <xs:element name="End" type="tns:PointType"/>
      <xs:element name="StrokeThickness" type="xs:long"/>
      <xs:element name="StrokeColor" type="xs:string"/>
      <xs:element name="LineType" type="tns:LineTypeEnum" minOccurs="0" default="Solid"/>
      <xs:element name="AngleDegrees" type="xs:double" minOccurs="0" default="0"/>
    </xs:sequence>
    <xs:attribute name="id" type="xs:string" use="required"/>
  </xs:complexType>

  <xs:complexType name="RectangleType">
    <xs:sequence>
      <xs:element name="Bounds" type="tns:BoundsType"/>
      <xs:element name="StrokeThickness" type="xs:long"/>
      <xs:element name="StrokeColor" type="xs:string"/>
      <xs:element name="FillColor" type="xs:string"/>
      <xs:element name="AngleDegrees" type="xs:double" minOccurs="0" default="0"/>
    </xs:sequence>
    <xs:attribute name="id" type="xs:string" use="required"/>
  </xs:complexType>

  <xs:complexType name="TextType">
    <xs:sequence>
      <xs:element name="Bounds" type="tns:BoundsType"/>
      <xs:element name="Content" type="xs:string"/>
      <xs:element name="FontFamily" type="xs:string"/>
      <xs:element name="FontSize" type="xs:double"/>
      <xs:element name="Foreground" type="xs:string"/>
      <xs:element name="TextAlignment" type="xs:string" minOccurs="0"/>
      <xs:element name="TextWrapping" type="xs:boolean" minOccurs="0"/>
      <xs:element name="AngleDegrees" type="xs:double" minOccurs="0" default="0"/>
      <xs:element name="TextType" type="tns:TextTypeEnum" minOccurs="0" default="SingleLine"/>
    </xs:sequence>
    <xs:attribute name="id" type="xs:string" use="required"/>
    <xs:attribute name="isEditable" type="xs:boolean" default="false"/>
    <xs:attribute name="key" type="xs:string"/>
  </xs:complexType>

  <xs:complexType name="PointType">
    <xs:attribute name="X" type="xs:long" use="required"/>
    <xs:attribute name="Y" type="xs:long" use="required"/>
  </xs:complexType>

  <xs:complexType name="BoundsType">
    <xs:attribute name="X" type="xs:long" use="required"/>
    <xs:attribute name="Y" type="xs:long" use="required"/>
    <xs:attribute name="Width" type="xs:long" use="required"/>
    <xs:attribute name="Height" type="xs:long" use="required"/>
  </xs:complexType>

  <!-- Перечисления -->
  <xs:simpleType name="LineTypeEnum">
    <xs:restriction base="xs:string">
      <xs:enumeration value="Solid"/>
      <xs:enumeration value="Dash"/>
      <xs:enumeration value="DashDot"/>
      <xs:enumeration value="DashDotDotDot"/>
    </xs:restriction>
  </xs:simpleType>

  <xs:simpleType name="TextTypeEnum">
    <xs:restriction base="xs:string">
      <xs:enumeration value="SingleLine"/>
      <xs:enumeration value="MultiLine"/>
    </xs:restriction>
  </xs:simpleType>

</xs:schema>
```
