# USER STORIES
## Редактор шаблонов листов (Этап 1)

**Проект:** DotElectric
**Версия:** 1.0
**Дата:** 02 июля 2026 г.
**Статус:** УТВЕРЖДЕНО

---

## ЭПИК 1: Управление шаблонами

### US-1.1: Создание нового шаблона

**Как** инженер-конструктор,
**Я хочу** создавать новый шаблон с выбором формата листа,
**Чтобы** начать разработку конструкторского документа.

**Критерии приемки:**
- [x] Пользователь может выбрать формат листа из списка (A0, A1, A2, A3, A4)
- [x] Пользователь может задать пользовательские размеры листа
- [x] После создания отображается пустой лист с заданными размерами
- [x] Границы листа визуально обозначены
- [x] Рисование за пределами листа заблокировано
- [x] Новый шаблон открывается в новой вкладке холста

---

### US-1.2: Открытие существующего шаблона

**Как** инженер-конструктор,
**Я хочу** открывать существующие файлы .tdel,
**Чтобы** редактировать ранее созданные шаблоны.

**Критерии приемки:**
- [x] Пользователь может выбрать файл .tdel через диалог открытия файлов
- [x] Шаблон корректно загружается и отображается на холсте
- [x] Все объекты шаблона сохраняют свои свойства
- [x] При ошибке загрузки отображается понятное сообщение
- [x] Шаблон открывается в новой вкладке холста
- [x] Система координат: Декартова, точка (0, 0) — левый нижний угол, единицы измерения — мм

---

### US-1.3: Сохранение шаблона

**Как** инженер-конструктор,
**Я хочу** сохранять шаблон в файл .tdel,
**Чтобы** использовать его в будущем и передавать коллегам.

**Критерии приемки:**
- [x] Пользователь может сохранить шаблон через диалог сохранения файлов
- [x] Файл сохраняется с расширением .tdel
- [x] Файл представляет собой ZIP-архив с template.xml внутри
- [x] XML соответствует заданной схеме
- [x] При сохранении перезаписывается существующий файл без подтверждения (если файл открыт)
- [x] Система выполняет автосохранение каждые 5 минут для восстановления после сбоев

---

### US-1.4: Отмена и повтор действий

**Как** инженер-конструктор,  
**Я хочу** отменять и повторять свои действия,  
**Чтобы** исправить случайные ошибки при редактировании.

**Критерии приемки:**
- [x] Комбинация Ctrl+Z отменяет последнее действие
- [x] Комбинация Ctrl+Y повторяет отменённое действие
- [x] Минимум 50 уровней отмены
- [x] Отмена работает для всех операций (создание, перемещение, удаление, изменение свойств)

---

### US-1.5: Библиотека шаблонов

**Как** инженер-конструктор,
**Я хочу** видеть библиотеку доступных шаблонов,
**Чтобы** быстро выбрать подходящий для работы.

**Критерии приемки:**
- [x] В приложении отображается панель «Библиотека шаблонов» (левая панель)
- [x] Шаблоны загружаются из папки `%APPDATA%\DotElectric\Templates\`
- [x] Каждый шаблон отображается с именем (имя файла без `.tdel`)
- [x] Шаблоны отсортированы в алфавитном порядке (по имени, без учёта регистра)
- [x] Если шаблонов нет — панель пустая (без элементов списка, только заголовок)
- [x] Резервная копия шаблонов хранится в общедоступном месте
- [x] Необходимые шрифты (ГОСТ) поставляются в комплекте с приложением

---

### US-1.6: Работа с вкладками (Multi-tab)

**Как** инженер-конструктор,
**Я хочу** открывать несколько шаблонов в вкладках,
**Чтобы** работать с несколькими документами одновременно.

**Критерии приемки:**
- [x] Каждый открытый шаблон отображается в отдельной вкладке
- [x] Пользователь может переключаться между вкладками кликом
- [x] Пользователь может закрыть вкладку (крестик на вкладке)
- [x] При закрытии несохранённого шаблона предлагается сохранить
- [x] Вкладки отображаются в верхней части холста редактора
- [x] Есть кнопка [+] для быстрого создания нового шаблона

---

## ЭПИК 2: Графические примитивы

### US-2.1: Рисование линий

**Как** инженер-конструктор,
**Я хочу** рисовать линии на шаблоне,
**Чтобы** создавать рамку и элементы штампа.

**Критерии приемки:**
- [x] Пользователь может выбрать инструмент «Линия»
- [x] Линия рисуется перетаскиванием мыши на холсте
- [x] Концы линии привязываются к сетке (если привязка включена)
- [x] Можно изменить толщину линии (Sprint 42)
- [x] Можно изменить цвет линии
- [x] Можно изменить тип линии (сплошная, пунктирная, штрихпунктирная, штрихпунктирная с двумя точками)
- [x] Можно изменить угол линии
- [x] Линию можно выделить, переместить и удалить

---

### US-2.2: Рисование прямоугольников

**Как** инженер-конструктор,
**Я хочу** рисовать прямоугольники на шаблоне,
**Чтобы** создавать рамки и области штампа.

**Критерии приемки:**
- [x] Пользователь может выбрать инструмент «Прямоугольник»
- [x] Прямоугольник рисуется перетаскиванием мыши на холсте
- [x] Углы привязываются к сетке (если привязка включена)
- [x] Можно изменить толщину линии (Sprint 42)
- [x] Можно изменить тип линии (сплошная, пунктирная, штрихпунктирная, штрихпунктирная с двумя точками)
- [x] Можно изменить угол поворота прямоугольника
- [x] Можно изменить цвет фигуры
- [x] Прямоугольник можно выделить, переместить, изменить размер и удалить

---

### US-2.3: Добавление текста

**Как** инженер-конструктор,
**Я хочу** добавлять текстовые объекты на шаблон,
**Чтобы** создавать надписи в штампе и пояснения.

**Критерии приемки:**
- [x] Пользователь может выбрать инструмент «Текст»
- [x] Текстовое поле создаётся кликом на холсте
- [x] Можно выбрать тип текста: однострочный или многострочный
- [x] Можно редактировать содержимое текста
- [x] Можно изменить шрифт, размер, цвет, выравнивание
- [x] Можно изменить угол поворота текста
- [x] Можно задать свойство «Изменяемый» (да/нет)
- [x] Если текст помечен как изменяемый, его можно изменить из свойств проекта или вручную
- [x] Если текст помечен как неизменяемый, его можно изменять только в редакторе шаблонов
- [x] Текст можно выделить, переместить и удалить

---

### US-2.4: Выделение и манипуляции с объектами

**Как** инженер-конструктор,
**Я хочу** выделять и перемещать объекты,
**Чтобы** корректировать их положение на шаблоне.

**Критерии приемки:**
- [x] Клик по объекту выделяет его
- [x] Выделенный объект отображается с маркерами выделения
- [x] Перетаскивание перемещает объект
- [x] При включённой привязке объект привязывается к сетке при перемещении
- [x] Delete удаляет выделенный объект
- [x] Ctrl+C / Ctrl+V копируют и вставляют объект
- [x] Shift + Клик добавляет объект к выделению
- [x] Ctrl + Клик снимает выделение с объекта
- [x] Выделение рамкой слева направо выделяет только целиком попавшие объекты (синяя рамка)
- [x] Выделение рамкой справа налево выделяет все задетые объекты (зелёная пунктирная рамка)
- [x] Зажатая клавиша Shift при рисовании фиксирует горизонтальные/вертикальные линии или пропорции квадрата
- [x] Стрелки перемещают объект на шаг сетки (привязка включена) или на 0.1 мм (привязка выключена)
- [x] Shift + Стрелки перемещают объект на большой шаг (10 мм) или на текущий шаг сетки
- [x] Колесо мыши масштабирует вид
- [x] Зажатое среднее колесо мыши перемещает вид (панорамирование)

---

### US-2.5: Изменение свойств объектов

**Как** инженер-конструктор,  
**Я хочу** изменять свойства выделенных объектов,  
**Чтобы** настроить их внешний вид.

**Критерии приемки:**
- [x] Панель свойств отображает параметры выделенного объекта
- [x] Можно изменить координаты (X, Y) объекта
- [x] Можно изменить размеры (для прямоугольника и текста)
- [x] Можно изменить цвет обводки/заливки
- [x] Можно изменить толщину линий
- [x] Можно изменить шрифт (для текста)

---

## ЭПИК 3: Сетка и привязка

### US-3.1: Отображение сетки

**Как** инженер-конструктор,  
**Я хочу** включать и отключать отображение сетки,  
**Чтобы** точно позиционировать объекты.

**Критерии приемки:**
- [x] Кнопка/меню для включения/отключения сетки
- [x] Сетка отображается в виде точек в узлах
- [x] Отображение сетки не влияет на печать
- [x] Сетка не мешает выделению объектов

---

### US-3.2: Выбор шага сетки

**Как** инженер-конструктор,  
**Я хочу** выбирать шаг сетки,  
**Чтобы** работать с разной точностью.

**Критерии приемки:**
- [x] Доступные шаги: 0.5 мм, 1 мм, 5 мм, 10 мм
- [x] Выбор шага через выпадающий список или меню
- [x] При изменении шага сетка перерисовывается
- [x] Текущий шаг отображается в строке состояния

---

### US-3.3: Привязка к сетке

**Как** инженер-конструктор,  
**Я хочу** включать и отключать привязку к сетке,  
**Чтобы** контролировать точность позиционирования.

**Критерии приемки:**
- [x] Кнопка/меню для включения/отключения привязки
- [x] При включённой привязке объекты «прилипают» к узлам сетки
- [x] Привязка работает при рисовании и перемещении
- [x] Привязка не работает при отключённой сетке (или работает независимо)

---

## ЭПИК 4: Текстовые поля

### US-4.1: Создание текстового поля с ключом

**Как** инженер-конструктор,  
**Я хочу** создавать текстовые поля с уникальным ключом,  
**Чтобы** программно заполнять их при использовании шаблона.

**Критерии приемки:**
- [x] У каждого текстового поля есть уникальный ключ (ID)
- [x] Ключ задаётся в панели свойств
- [x] Ключ должен быть уникальным в пределах шаблона
- [x] Ключ сохраняется в XML шаблона

---

### US-4.2: Пометка поля как изменяемого

**Как** инженер-конструктор,  
**Я хочу** помечать текстовые поля как «изменяемые»,  
**Чтобы** пользователь мог заполнять их при использовании шаблона.

**Критерии приемки:**
- [x] У текстового поля есть флаг «Изменяемое» (isEditable)
- [x] Флаг задаётся в панели свойств (чекбокс)
- [x] Флаг сохраняется в XML шаблона
- [x] Изменяемые поля визуально не отличаются (или отличаются по желанию)

---

### US-4.3: Редактирование свойств текста

**Как** инженер-конструктор,  
**Я хочу** настраивать внешний вид текста,  
**Чтобы** соответствовать требованиям ГОСТ.

**Критерии приемки:**
- [x] Можно выбрать шрифт из списка
- [x] Можно задать размер шрифта
- [x] Можно выбрать цвет текста
- [x] Можно выбрать выравнивание (левое, центр, правое)
- [x] Можно задать начальный контент

---

## ЭПИК 5: Печать

### US-5.1: Печать шаблона

**Как** инженер-конструктор,  
**Я хочу** печатать шаблон,  
**Чтобы** проверить его внешний вид на бумаге.

**Критерии приемки:**
- [x] Кнопка/меню «Печать» открывает стандартный диалог печати Windows
- [x] Можно выбрать принтер
- [x] Можно задать количество копий
- [x] Шаблон печатается в масштабе 1:1
- [x] Печать работает корректно для всех форматов листов

---

### US-5.2: Предварительный просмотр печати

**Как** инженер-конструктор,  
**Я хочу** видеть предварительный просмотр перед печатью,  
**Чтобы** убедиться в корректности размещения.

**Критерии приемки:**
- [x] Предпросмотр печати открывается через меню `Файл > Предпросмотр печати` или `Ctrl+Shift+P`
- [x] Предпросмотр отображает шаблон в масштабе страницы (FitToWidth в DocumentViewer)
- [x] Можно вернуться к редактированию, закрыв окно предпросмотра (X / Close)

## ЭПИК 6: Навигация и масштабирование

### US-6.1: Масштабирование вида

**Как** инженер-конструктор,  
**Я хочу** изменять масштаб отображения,  
**Чтобы** работать с мелкими деталями и видеть весь лист.

**Критерии приемки:**
- [x] Масштаб меняется от 10% до 1000%
- [x] Кнопки «+» и «-» на панели инструментов
- [x] Колёсико мыши с Ctrl меняет масштаб
- [x] Текущий масштаб отображается в строке состояния
- [x] Кнопка «Fit to Screen» масштабирует по окну

---

### US-6.2: Прокрутка холста

**Как** инженер-конструктор,  
**Я хочу** прокручивать содержимое холста,  
**Чтобы** перемещаться по большим шаблонам.

**Критерии приемки:**
- [x] Полосы прокрутки отображаются при необходимости
- [x] Прокрутка колёсиком мыши работает
- [x] Перетаскивание средней кнопкой мыши перемещает вид
- [x] Прокрутка плавная, без задержек

## ЭПИК 7: Интерфейс и настройки

### US-7.1: Классическое меню

**Как** инженер-конструктор,  
**Я хочу** иметь классическое меню,  
**Чтобы** быстро находить нужные команды.

**Критерии приемки:**
- [x] Меню содержит пункты: Файл, Правка, Вид, Вставка, Инструменты, Справка
- [x] Каждый пункт имеет выпадающий список команд
- [x] Горячие клавиши отображаются рядом с командами
- [x] Меню соответствует стандартам Windows

---

### US-7.2: Тёмная тема

**Как** инженер-конструктор,  
**Я хочу** переключаться между светлой и тёмной темой,  
**Чтобы** работать комфортно в разных условиях освещения.

**Критерии приемки:**
- [x] Переключатель темы в меню «Вид» или «Инструменты»
- [x] Тёмная тема меняет цвета всех элементов интерфейса
- [x] Холст остаётся белым (или светло-серым) в любой теме
- [x] Тема сохраняется между запусками приложения

---

### US-7.3: Панель свойств

**Как** инженер-конструктор,  
**Я хочу** иметь панель свойств выделенного объекта,  
**Чтобы** быстро редактировать параметры.

**Критерии приемки:**
- [x] Панель свойств отображается справа (или слева)
- [x] При выделении объекта панель показывает его свойства
- [x] При выделении нескольких объектов показывает общие свойства
- [x] При отсутствии выделения панель пуста или показывает свойства холста
