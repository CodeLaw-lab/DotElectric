# ТЕХНИЧЕСКОЕ ЗАДАНИЕ
## Редактор шаблонов листов для CAD-системы DotElectric
### Этап 1 разработки

**Версия документа:** 1.0
**Дата:** 26 марта 2026 г.
**Статус:** УТВЕРЖДЕНО

---

## 1. ОБЩИЕ СВЕДЕНИЯ

### 1.1. Наименование проекта
**DotElectric** — CAD-система для проектирования электрических схем и документации.

### 1.2. Наименование программы
**Редактор шаблонов** — инструмент для создания и редактирования шаблонов конструкторских документов (рамки, штампы, текстовые поля).

### 1.3. Область применения
Система предназначена для использования сотрудниками инженерно-конструкторских подразделений при разработке конструкторской документации в соответствии с ГОСТ ЕСКД.

### 1.4. Термины и определения

| Термин | Определение |
|--------|-------------|
| **Шаблон** | Заготовка листа с рамкой, штампом и предопределёнными полями |
| **УГО** | Условное графическое обозначение (электрические компоненты на схемах) |
| **Штамп** | Основная надпись конструкторского документа по ГОСТ 2.104 |
| **Сетка** | Вспомогательная координатная сетка для точного позиционирования |
| **Привязка** | Автоматическое выравнивание объектов по узлам сетки |
| **.tdel** | Фирменный формат файла шаблона (XML в ZIP-архиве) |

---

## 2. НАЗНАЧЕНИЕ И ЦЕЛИ

### 2.1. Назначение системы
Создание инструмента для разработки шаблонов конструкторских документов с возможностью:
- Рисования графических примитивов (линии, прямоугольники, текст)
- Создания штампов произвольной формы
- Определения изменяемых текстовых полей
- Сохранения в собственный формат .tdel
- Печати готовых шаблонов

### 2.2. Цели разработки
- Предоставить пользователям возможность создания собственных шаблонов по ГОСТ
- Обеспечить точное позиционирование объектов с привязкой к сетке
- Реализовать хранение шаблонов в компактном переносимом формате
- Подготовить основу для интеграции с главным редактором схем

**Детализация FR-005:**
- **Интерфейс команд:** `ICommand` с методами `Execute()`, `Undo()`, свойством `Name` (не зависит от `System.Windows.Input.ICommand`)
- **Менеджер истории:** `CommandHistory` с отдельными стеками Undo/Redo, автоматической обрезкой oldest команд при превышении 50 уровней
- **Составные операции:** `BatchCommand` для группировки нескольких команд (например, удаление нескольких объектов), Undo выполняется в обратном порядке (LIFO)
- **IsDirty:** Устанавливается автоматически командами через `editor.MarkDirty()`, НЕ вручную
- **Checkpoint:** История НЕ очищается при сохранении файла (пользователь может Undo после сохранения)
- **Горячие клавиши:** Ctrl+Z (Undo), Ctrl+Y (Redo)
- **UI:** Динамические названия в меню ("↶ Переместить объект", "↷ Добавить линию")
- **Обработка ошибок:** Rollback при failed Undo, логирование через Serilog, показ MessageBox пользователю
- **Покрытие тестами:** CommandHistory 100%, конкретные команды 100%

## 5. ТРЕБОВАНИЯ К ФОРМАТУ ФАЙЛА

### 5.1. Структура файла .tdel

Файл `.tdel` представляет собой ZIP-архив со следующей структурой:

```
template.tdel
└── template.xml
```

### 5.2. Структура XML

```xml
<?xml version="1.0" encoding="utf-8"?>
<!-- Все размеры указаны в микронах (1 мм = 1000 микрон).
     Координаты: Fixed-Point, целочисленные значения. -->
<Template xmlns="http://dotelectric.local/schema/template" version="1.0">
  <Metadata>
    <Name>Наименование шаблона</Name>
    <Description>Описание</Description>
    <Author>Автор</Author>
    <CreatedDate>2026-03-25T10:00:00</CreatedDate>
    <ModifiedDate>2026-03-25T12:00:00</ModifiedDate>
  </Metadata>

  <Sheet>
    <Format>A3</Format>
    <Width>420000</Width>
    <Height>297000</Height>
    <Unit>mm</Unit>
  </Sheet>

  <Objects>
    <Line id="line1">
      <Start X="10000" Y="10000" />
      <End X="410000" Y="10000" />
      <StrokeThickness>500</StrokeThickness>
      <StrokeColor>#000000</StrokeColor>
    </Line>

    <Rectangle id="rect1">
      <Bounds X="0" Y="0" Width="420000" Height="297000" />
      <StrokeThickness>500</StrokeThickness>
      <Fill>#FFFFFF</Fill>
    </Rectangle>

    <Text id="text1" isEditable="true" key="DOCUMENT_NAME">
      <Bounds X="50000" Y="50000" Width="100000" Height="10000" />
      <Content>Наименование документа</Content>
      <FontFamily>Arial</FontFamily>
      <FontSize>10</FontSize>
      <Foreground>#000000</Foreground>
    </Text>
  </Objects>
</Template>
```

### 5.3. Требования к XML

- Кодировка: UTF-8
- Пространство имён: `http://dotelectric.local/schema/template`
- Все координаты и размеры — в **микронах** (целые числа, `long`). 1 мм = 1000 микрон.
- Все цвета в формате HEX (#RRGGBB)
- ID объектов должны быть уникальными в пределах шаблона
- Ключи изменяемых полей должны быть уникальными
- Типы значений в XML: `xs:long` для координат/размеров, `xs:string` для текста/цветов, `xs:boolean` для флагов, `xs:dateTime` для дат
- Шрифт (FontSize) указывается в пунктах (`double`), не конвертируется в микроны

> **Примечание:** Fixed-Point подход (целочисленные микроны) исключает погрешности floating-point при round-trip сериализации. Конвертация в мм для отображения происходит только на уровне UI через `Coordinate.ToMm()`.

**Визуальная логика выделения (Selection Box):**

| Направление | Визуализация | Логика выделения |
|-------------|--------------|------------------|
| Слева направо | Синий контур + полупрозрачная синяя заливка | Выделяются только объекты, целиком попавшие в рамку |
| Справа налево | Зелёный пунктирный контур + зелёная заливка | Выделяются все объекты, задетые рамкой |

**Система координат:** Декартова, точка (0, 0) — левый нижний угол листа. Единицы измерения — миллиметры (мм).

> **Примечание реализации:** Конвертация в WPF-координаты (Y↓) происходит на уровне `EditorCanvas` через `FromWpfPoint()` / `ToWpfPoint()`. См. Спецификацию, раздел 3.1.B.

### 8.2. Логика управления и навигация

**Выделение:**
- Одиночный клик — выбор объекта и отображение свойств
- Shift + Клик — последовательное добавление объектов в выборку
- Ctrl + Клик — снятие выделения с объекта
- Рамка слева направо — выделяются только объекты, целиком попавшие в рамку
- Рамка справа налево — выделяются все задетые объекты

**Клавиатура и мышь:**
- Shift — при рисовании фиксирует горизонтальные/вертикальные линии или пропорции квадрата
- Стрелки (привязка включена, F9) — перемещение на текущий шаг сетки (0.5/1/5/10 мм)
- Shift + Стрелки (привязка включена) — перемещение на большой шаг (10 мм)
- Стрелки (привязка выключена) — микро-смещение на 0.1 мм
- Shift + Стрелки (привязка выключена) — перемещение на текущий шаг сетки
- Zoom — колесо мыши
- Pan — зажатие среднего колеса мыши
