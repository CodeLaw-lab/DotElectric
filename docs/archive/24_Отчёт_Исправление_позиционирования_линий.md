# Отчёт — Исправление позиционирования линий

**Дата:** 07.04.2026
**Задача:** Исправить рисование линий — линии отображаются не в тех местах, где рисуются

---

## ПРОБЛЕМА

После исправления прямоугольников (Sprint 9), линии стали отображаться в неправильных позициях.

**Причина:** Координаты `Canvas.Left/Top` были перенесены в `ItemContainerStyle` (ContentPresenter), но `X1/Y1/X2/Y2` в DataTemplate для Line по-прежнему использовали **абсолютные** модельные координаты через `ModelXToCanvasLeftConverter`.

В результате:
- ContentPresenter размещался в `(LeftEdge, TopEdge)`
- Линия внутри ContentPresenter рисовалась от `X1 = ToCanvas(StartMicronsX)`, т.е. от **абсолютной** позиции, а не от позиции относительно ContentPresenter
- Итоговая позиция = `Canvas.Left + X1 = LeftEdge + absoluteX` — **двойное смещение**

---

## РЕШЕНИЕ

`X1/Y1/X2/Y2` должны задаваться **относительно** ContentPresenter:

| Свойство | Было | Стало |
|----------|------|-------|
| `Canvas.Left` | `ToCanvas(MicronsX)` | `ToCanvas(LeftEdgeMicrons)` = `ToCanvas(min(StartX, EndX))` |
| `Canvas.Top` | `ToCanvas(MicronsY)` | `ToCanvas(TopEdgeMicrons)` = `ToCanvas(max(StartY, EndY))` |
| `X1` | `ToCanvas(StartMicronsX)` | `(StartX - minX) * zoom` |
| `Y1` | `ToCanvas(StartMicronsY)` | `(maxY - StartY) * zoom` |
| `X2` | `ToCanvas(EndMicronsX)` | `(EndX - minX) * zoom` |
| `Y2` | `ToCanvas(EndMicronsY)` | `(maxY - EndY) * zoom` |

### Новые конвертеры

| Конвертер | Назначение |
|-----------|-----------|
| `LeftEdgeMicronsConverter` | `min(StartMicronsX, EndMicronsX)` для Line |
| `TopEdgeMicronsConverter` | `max(StartMicronsY, EndMicronsY)` для Line |
| `LocalX1Converter` | `StartMicronsX - minX` → пиксели |
| `LocalY1Converter` | `maxY - StartMicronsY` → пиксели |
| `LocalX2Converter` | `EndMicronsX - minX` → пиксели |
| `LocalY2Converter` | `maxY - EndMicronsY` → пиксели |
