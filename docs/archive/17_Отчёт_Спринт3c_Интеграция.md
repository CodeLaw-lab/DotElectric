## 5. КЛЮЧЕВЫЕ АРХИТЕКТУРНЫЕ РЕШЕНИЯ

### 5.1. Preview-элементы в ViewModel

**Проблема:** Tools создавались заново при каждом событии мыши → Preview терялся.

**Решение:**
- PreviewLine, PreviewRectangle, PreviewText → свойства EditorViewModel
- Tools кэшируются через `GetOrCreateTool<T>()`
- Tools обновляют `EditorViewModel.PreviewXxx` вместо локальных полей

### 5.2. Маркеры выделения через DataTemplate

**Проблема:** Сложно реализовать маркеры в чистом XAML без code-behind.

**Решение:**
- Вычисляемые свойства в моделях (RightMicronsX, CenterMicronsX, и т.д.)
- ContentControl с DataTemplate по типу объекта
- MultiBinding через существующие конвертеры
- ~260 строк XAML для всех маркеров

### 5.3. Клавиатурные команды

**Решение:**
- InputBindings на EditorCanvas
- Отдельные команды для каждой стрелки (NudgeUp/Down/Left/Right)
- Буфер обмена внутри EditorViewModel (не системный Clipboard)

## 6. ИЗВЕСТНЫЕ ОГРАНИЧЕНИЯ

1. **Multi-selection маркеры** — не реализованы (только пунктирная подсветка объектов)
2. **Inline-редактирование текста** — двойной клик не открывает редактор
3. **PanTool** — средняя кнопка мыши / Space+drag не реализованы
4. **ResizeTool** — маркеры изменения размера не реализованы
5. **Selection Box** — рамка выделения не реализована
6. **LightTheme.xaml** — переключение тем не реализовано
7. **StatusBar binding** — статусная строка не привязана к ViewModel
8. **Покрытие 67.9%** — до цели 70% не хватает ~2% (невозможно покрыть без интеграционных тестов)
