## 1. Исправлено позиционирование листа

**Проблема:** CanvasOffsetX/Y возвращали PanOffsetX/Y вместо -PanOffsetX/Y, из-за чего RenderTransform сдвигал canvas в неправильном направлении.

**Решение:** `ZoomPanManager.cs`:
```csharp
// Было
public double CanvasOffsetX => IsCentered ? 0 : PanOffsetX;

// Стало
public double CanvasOffsetX => IsCentered ? 0 : -PanOffsetX;
```

**Файлы:** `ZoomPanManager.cs`

---

## 2. Исправлено обновление скроллбаров

**Проблема:** EditorViewModel не пробрасывал PropertyChanged для ScrollXRange/Y, ScrollXValue/Y, ViewportWidth/HeightPixels, IsCentered от ZoomPanManager.

**Решение:** Расширена подписка на PropertyChanged в конструкторе EditorViewModel.

**Файлы:** `EditorViewModel.cs`, `EditorCanvas.xaml`

## 3. Исправлено рисование объектов

### Корневая причина
`EditorViewModel` wrapper-свойства (`PreviewLine`, `PreviewRectangle`, `PreviewText`) не вызывали `OnPropertyChanged()`. `PreviewLineChangedBehavior` подписан на `EditorViewModel.PropertyChanged` и слушает изменения этих свойств, но уведомления не генерировались — preview-элементы в XAML никогда не обновлялись.

### Решение
Добавлен `OnPropertyChanged()` в setter'ы 8 wrapper-свойств:
- `PreviewLine`, `PreviewRectangle`, `PreviewText`
- `SelectionBoxLeft`, `SelectionBoxBottom`, `SelectionBoxWidth`, `SelectionBoxHeight`, `SelectionBoxDirection`

**Файлы:** `EditorViewModel.cs`, `EditorViewModelTests.cs` (+5 тестов)

---

## 4. Тулбар — только один активный инструмент

**Проблема:** `ToggleButton` в тулбаре позволяли выбирать несколько инструментов одновременно.

**Решение:**
- `ToggleButton` → `RadioButton` с `GroupName="DrawingTools"`
- Создан стиль `ToolBarRadioButtonStyle` в `DarkTheme.xaml` и `LightTheme.xaml`
- `SetActiveToolCommand` и setter `ActiveTool` теперь вызывают `ResetTool` и `ClearAll` при смене инструмента

**Файлы:** `MainWindow.xaml`, `DarkTheme.xaml`, `LightTheme.xaml`, `EditorViewModel.cs`

---

## 5. Escape — переключение на Select + обновление курсора

**Проблема:** При нажатии Escape инструмент переключался на Select, но курсор не обновлялся.

**Решение:** Добавлен метод `UpdateCursor` в `EditorCanvasBehavior.cs`, вызывается после смены инструмента по Escape.

**Файлы:** `EditorCanvasBehavior.cs`
