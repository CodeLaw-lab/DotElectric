## РЕАЛИЗОВАННЫЕ КОМПОНЕНТЫ

### 1. IApplicationLifecycle (Долг #4 — УСТРАНЁН ✅)

**Проблема:** `MainViewModel.Exit()` напрямую вызывал `Application.Current.Shutdown()`, что нарушало MVVM и делало код нетестируемым.

**Решение:** Создан интерфейс `IApplicationLifecycle` с WPF-реализацией, зарегистрирован в DI, внедрён в `MainViewModel`.

| Файл | Описание |
|------|----------|
| `Services/IApplicationLifecycle.cs` | **НОВЫЙ** — интерфейс с методом `Shutdown()` |
| `Services/WpfApplicationLifecycle.cs` | **НОВЫЙ** — WPF-реализация через `Application.Current.Shutdown()` |
| `ViewModels/MainViewModel.cs` | Добавлен `_applicationLifecycle`, DI, IDisposable |
| `App.xaml.cs` | Регистрация `IApplicationLifecycle → WpfApplicationLifecycle` |
| `ViewModels/MainViewModelTests.cs` | +моки, +тест `ExitCommand_CallsShutdown` |
| `Services/WpfApplicationLifecycleTests.cs` | **НОВЫЙ** — тест создания |

**Результат:** MainViewModel больше не зависит от WPF напрямую для завершения приложения.

### 2. WeakReferenceMessenger для закрытия вкладок (Долг #2 — УСТРАНЁН ✅)

**Проблема:** `EditorViewModel` имел свойство `MainVm: MainViewModel?`, через которое вызывал команды закрытия вкладок. Это создавало циклическую зависимость (child → parent), нарушало dependency direction и усложняло тестирование.

**Решение:** Заменено на `WeakReferenceMessenger` из CommunityToolkit.Mvvm.

| Файл | Описание |
|------|----------|
| `Messages/TabCloseMessages.cs` | **НОВЫЙ** — 3 record-класса: `CloseTabRequestMessage`, `CloseOtherTabsRequestMessage`, `CloseAllTabsRequestMessage` |
| `ViewModels/EditorViewModel.cs` | Убрано свойство `MainVm`, команды теперь посылают сообщения через `WeakReferenceMessenger.Default.Send()` |
| `ViewModels/MainViewModel.cs` | Подписка на 3 типа сообщений в конструкторе через `WeakReferenceMessenger.Default.Register()` |

**Изменения в EditorViewModel:**
```csharp
// БЫЛО:
public MainViewModel? MainVm { get; set; }
private async Task CloseTabAsync() {
    if (MainVm?.CloseTabCommand.CanExecute(this) == true)
        await MainVm.CloseTabCommand.ExecuteAsync(this);
}

// СТАЛО:
private void CloseTabAsync() {
    WeakReferenceMessenger.Default.Send(new CloseTabRequestMessage(this));
}
```

**Изменения в MainViewModel:**
```csharp
// Подписка в конструкторе:
WeakReferenceMessenger.Default.Register<CloseTabRequestMessage>(this, (r, m) => {
    _ = CloseTab(m.Tab);
});
```

**Результат:** Чистая dependency direction: EditorViewModel → Messages ← MainViewModel. Нет циклических ссылок.

### 3. Code-behind PropertiesPanelContent (Долг #1 — УСТРАНЁН ✅)

**Проблема:** `PropertiesPanelContent.xaml.cs` содержал **220 строк** code-behind — 16 обработчиков событий (11 TextBox LostFocus + 5 ComboBox SelectionChanged), которые парсили значения из UI и вызывали команды ViewModel.

**Решение:** Созданы Attached Behavior'ы + команды-обёртки в ViewModel.

| Файл | Описание |
|------|----------|
| `Behaviors/TextBoxLostFocusCommandBehavior.cs` | **НОВЫЙ** — attached property `LostFocusCommand`, вызывает ICommand при LostFocus, передаёт `TextBox.Text` |
| `Behaviors/ComboBoxSelectionChangedCommandBehavior.cs` | **НОВЫЙ** — attached property `SelectionChangedCommand`, вызывает ICommand при SelectionChanged, передаёт текст выбранного элемента |
| `ViewModels/PropertiesViewModel.cs` | +15 команд-обёрток (`*FromString(string?)`), using `System.Globalization`, `DotElectric.TemplateEditor.Models` |
| `Views/PropertiesPanelContent.xaml.cs` | **СОКРАЩЁН** с 220 строк → **13 строк** (только `InitializeComponent()`) |
| `Views/PropertiesPanelContent.xaml` | +xmlns:behaviors, все `LostFocus=` и `SelectionChanged=` заменены на attached behavior |

**Новые команды в PropertiesViewModel (15 штук):**

| Команда | Назначение |
|---------|-----------|
| `ChangeLineStartXFromStringCommand` | Парсит мм → микроны → ChangeLineStartX |
| `ChangeLineStartYFromStringCommand` | Парсит мм → микроны → ChangeLineStartY |
| `ChangeLineEndXFromStringCommand` | Парсит мм → микроны → ChangeLineEndX |
| `ChangeLineEndYFromStringCommand` | Парсит мм → микроны → ChangeLineEndY |
| `ChangeRectXFromStringCommand` | Парсит мм → микроны → ChangeRectX |
| `ChangeRectYFromStringCommand` | Парсит мм → микроны → ChangeRectY |
| `ChangeRectWidthFromStringCommand` | Парсит мм → микроны → ChangeRectWidth |
| `ChangeRectHeightFromStringCommand` | Парсит мм → микроны → ChangeRectHeight |
| `ChangeTextXFromStringCommand` | Парсит мм → микроны → ChangeTextX |
| `ChangeTextYFromStringCommand` | Парсит мм → микроны → ChangeTextY |
| `ChangeTextFontSizeFromStringCommand` | Парсит мм → микроны → ChangeTextFontSize |
| `ChangeTextRotationFromStringCommand` | Парсит int → ChangeTextRotation |
| `ChangeLineTypeFromStringCommand` | Парсит строку → LineType → ChangeLineType |
| `ChangeRectLineTypeFromStringCommand` | Парсит строку → LineType → ChangeRectLineType |
| `ChangeTextTypeFromStringCommand` | Парсит строку → TextType → ChangeTextType |

**Пример XAML (было → стало):**
```xml
<!-- БЫЛО: -->
<TextBox LostFocus="TextBox_LostFocus_LineStartX"
         Text="{Binding LineStartX, Converter={StaticResource MicronsToMmConverter}, ...}"/>

<!-- СТАЛО: -->
<TextBox behaviors:TextBoxLostFocusCommandBehavior.LostFocusCommand="{Binding ChangeLineStartXFromStringCommand}"
         Text="{Binding LineStartX, Converter={StaticResource MicronsToMmConverter}, ...}"/>
```

**Результат:** Code-behind сокращён на **94%** (220 → 13 строк). Вся логика в ViewModel, Undo/Redo работает через CommandHistory.
