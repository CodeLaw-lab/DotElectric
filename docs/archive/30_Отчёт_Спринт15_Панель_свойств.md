## 3. Описание проблем и решения

### 3.1. Проблема 1: Панель не отображает свойства выделенного объекта

**Симптом:**
- Пользователь выделяет линию/прямоугольник/текст на canvas
- Панель свойств отображает заглушку «Выберите объект для редактирования»
- Поля ввода (X, Y, ширина, высота и т.д.) пустые

**Корневая причина:**

В `PropertiesViewModel` вычисляемые свойства (`LineStartX`, `RectX`, `TextX` и др.) возвращают значения из `SelectedObject`, но **не уведомляют WPF Binding об изменении** при смене выделения.

Метод `UpdateSelection()`:
- Обновлял `SelectedObject` (вызывал уведомление только для этого свойства)
- Вызывал `OnPropertyChanged` для `IsLineSelected`, `IsRectangleSelected`, `IsTextSelected`
- **НЕ** вызывал `OnPropertyChanged` для вычисляемых свойств

WPF Binding не обновлялся, потому что не было явного уведомления об изменении `LineStartX`, `RectX` и т.д.

**Решение:**

Добавлены вызовы `OnPropertyChanged` для всех 19 вычисляемых свойств в конце `UpdateSelection()`:

```csharp
// Line properties (5)
OnPropertyChanged(nameof(LineStartX));
OnPropertyChanged(nameof(LineStartY));
OnPropertyChanged(nameof(LineEndX));
OnPropertyChanged(nameof(LineEndY));
OnPropertyChanged(nameof(LineTypeValue));

// Rectangle properties (5)
OnPropertyChanged(nameof(RectX));
OnPropertyChanged(nameof(RectY));
OnPropertyChanged(nameof(RectWidth));
OnPropertyChanged(nameof(RectHeight));
OnPropertyChanged(nameof(RectLineType));

// Text properties (7)
OnPropertyChanged(nameof(TextX));
OnPropertyChanged(nameof(TextY));
OnPropertyChanged(nameof(TextContent));
OnPropertyChanged(nameof(TextFontSize));
OnPropertyChanged(nameof(TextFontName));
OnPropertyChanged(nameof(TextTypeValue));
OnPropertyChanged(nameof(TextRotation));

// Common properties (2)
OnPropertyChanged(nameof(ObjectId));
OnPropertyChanged(nameof(ObjectTypeName));
```

**Результат:** При выделении объекта WPF Binding обновляется и отображает актуальные значения.

### 3.2. Проблема 2: Тип линии не отображается в ComboBox

**Симптом:**
- Пользователь выделяет линию с типом «Сплошная»
- ComboBox для типа линии пустой (не показывает «Сплошная»)
- При выборе другого типа — значение применяется, но не отображается

**Корневая причина:**

ComboBox использовал `SelectedItem="{Binding LineTypeValue, Mode=OneWay}"`, пытаясь сопоставить enum-значение (`LineType.Solid`) с объектами `ComboBoxItem` в ItemsSource. WPF не мог автоматически выполнить это сопоставление, т.к. типы несовместимы (`LineType` enum vs `ComboBoxItem`).

**Решение:**

Изменён подход к binding в ComboBox:

| Свойство | До | После |
|----------|-----|-------|
| Line LineType | `SelectedItem="{Binding LineTypeValue}"` | `Text="{Binding LineTypeValue, Converter={StaticResource LineTypeToStringConverter}}"` |
| Rectangle LineType | `SelectedItem="{Binding RectLineType}"` | `Text="{Binding RectLineType, Converter={StaticResource LineTypeToStringConverter}}"` |
| Text TextType | `SelectedItem="{Binding TextTypeValue}"` | `Text="{Binding TextTypeValue, Converter={StaticResource TextTypeToStringConverter}}"` |
| Text Rotation | `SelectedItem="{Binding TextRotation}"` | `Text="{Binding TextRotation}"` |

**Добавлены конвертеры в ресурсы:**
```xml
<converters:LineTypeToStringConverter x:Key="LineTypeToStringConverter"/>
<converters:TextTypeToStringConverter x:Key="TextTypeToStringConverter"/>
```

**Почему это работает:**
- `Text` binding принимает строковое значение
- Конвертер `LineTypeToStringConverter.Convert()` преобразует `LineType.Solid` → `"Сплошная"`
- WPF находит `ComboBoxItem` с `Content="Сплошная"` и отображает его
- При выборе другого элемента `SelectionChanged` handler вызывает команду с правильным enum-значением

**Результат:** Тип линии/текста корректно отображается при выделении объекта.

## 4. Архитектурные решения

### 4.1. Явные уведомления для вычисляемых свойств

Вместо того чтобы полагаться на автоматическое обновление Binding через `SelectedObject`, явно вызываются `OnPropertyChanged` для всех зависимых свойств. Это:
- Гарантирует обновление UI
- Не зависит от реализации binding-движка
- Явно документирует зависимости между свойствами
- Соответствует принципу «явное лучше неявного»

### 4.2. Text vs SelectedItem в ComboBox

Для binding enum-значений к ComboBox с `ComboBoxItem` элементами:
- **SelectedItem** требует совпадения типов (enum ↔ ComboBoxItem) — невозможно без конвертера
- **Text** с конвертером — строка ↔ enum, WPF находит нужный ComboBoxItem по тексту
- **Альтернатива:** `ItemsSource` с enum-значениями + DataTemplate, но требует больше XAML

Текущий подход (Text + конвертер) — минимальные изменения, максимальная совместимость.
