## Выполненные задачи

### 21.1 — Исправить позиционирование листа (P0) ✅

**Проблема:** `PanOffsetX/Y` инициализировались значением `10.0` (DIP), но не масштабировались по Zoom. При Zoom≠1.0 отступ был некорректным:
- Zoom=1.0: 10 DIP = 10 мм ✅
- Zoom=2.0: 10 DIP = 5 мм ❌
- Zoom=0.5: 10 DIP = 20 мм ❌

**Исправление:**

**Файл:** `ViewModels/EditorViewModel.cs`

**Конструктор (строка 695):**
```csharp
// Было:
var offsetPixels = DefaultSheetOffsetMm; // при Zoom=1: 1мм = 1 unit
PanOffsetX = offsetPixels;
PanOffsetY = offsetPixels;

// Стало:
PanOffsetX = DefaultSheetOffsetMm * Zoom;
PanOffsetY = DefaultSheetOffsetMm * Zoom;
```

**FitToScreen (строка 793):**
```csharp
// Было:
PanOffsetX = DefaultSheetOffsetMm;
PanOffsetY = DefaultSheetOffsetMm;

// Стало:
PanOffsetX = DefaultSheetOffsetMm * Zoom;
PanOffsetY = DefaultSheetOffsetMm * Zoom;
```

**Новые тесты:**
- `Constructor_PanOffset_ScalesWithZoom` — проверка корректного начального значения
- `FitToScreen_PanOffset_ScalesWithZoom` — проверка масштабирования после FitToScreen

**Результат:** 2 новых теста, баг исправлен.

### 21.2 — Добавить ILogger в TemplateLibraryViewModel (P1) ✅

**Проблема:** Ошибка загрузки шаблонов в `LoadTemplates()` не логировалась, только отображалась в UI.

**Исправление:**

**Файл:** `ViewModels/TemplateLibraryViewModel.cs`

- Добавлен `ILogger<TemplateLibraryViewModel> _logger`
- Конструктор принимает опциональный `logger?` (по умолчанию `NullLogger<T>.Instance`)
- `catch` блок в `LoadTemplates()` теперь вызывает `_logger.LogError(ex, "Ошибка загрузки списка шаблонов из библиотеки")`

**Файл:** `ViewModels/MainViewModel.cs`

- Передача `null` в конструктор `TemplateLibraryViewModel` (logger создастся по умолчанию)

**Результат:** Ошибки библиотеки шаблонов теперь логируются в Serilog.
