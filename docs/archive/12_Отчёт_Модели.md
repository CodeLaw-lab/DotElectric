## 3. КЛЮЧЕВЫЕ АРХИТЕКТУРНЫЕ РЕШЕНИЯ

### 3.1. Fixed-Point координаты

- Все координаты хранятся в **микронах** (`long`) — 1 мм = 1000 микрон
- Сериализация в XML — `xs:long` (целые числа, round-trip без потерь)
- Конвертация в мм только для UI (`Coordinate.ToMm()`)

### 3.2. Система координат

- Модель хранит **Декартовы** координаты (0,0 = левый нижний угол, Y↑)
- Конвертация в WPF-координаты (Y↓) — только на уровне `EditorCanvas`
- ViewModel, Model, Services — НЕ знают про WPF

### 3.3. Реализация INotifyPropertyChanged

- Все модели наследуют `ObservableObject` из CommunityToolkit.Mvvm
- Свойства с уведомлениями — через `[ObservableProperty]` (source generator)
- Валидация — через `partial void OnXxxChanged()` (например, `OnRotationAngleChanged`)

### 3.4. Clone()

- `Clone()` создаёт **новый Id** (через конструктор, `Guid.NewGuid()`)
- Остальные свойства копируются из оригинала
- Для Line: Clone() копирует Start/End координаты и LineType

### 3.5. Сетка

- `Template.DefaultGrid` — **константа** 5 мм (readonly, НЕ сериализуется)
- `EditorViewModel.GridSettings` — **настраивается** пользователем (НЕ сериализуется)
