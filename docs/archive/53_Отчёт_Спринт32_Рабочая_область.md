## Реализованные изменения

### 1. ZoomPanManager.cs — новые свойства

| Свойство | Тип | Назначение |
|----------|-----|------------|
| `IsCentered` | `bool` | true когда canvas ≤ viewport |
| `CanvasOffsetX` | `double` | Итоговое смещение Canvas по X |
| `CanvasOffsetY` | `double` | Итоговое смещение Canvas по Y |
| `ScrollXRange` | `double` | Максимальное значение горизонтального скролла |
| `ScrollYRange` | `double` | Максимальное значение вертикального скролла |
| `ScrollXValue` | `double` | Текущее значение горизонтального скролла |
| `ScrollYValue` | `double` | Текущее значение вертикального скролла |
| `SetScrollX()` | `void` | Установка горизонтального скролла |
| `SetScrollY()` | `void` | Установка вертикального скролла |
| `CenterCanvas()` | `void` | Центрирование при изменении viewport/zoom |

**Логика CanvasOffsetX/Y:**
- Когда `IsCentered = true`: `CanvasOffset = 0` (центрирование через Border alignment)
- Когда `IsCentered = false`: `CanvasOffset = PanOffset` (скролл/панорамирование)

**Логика ScrollXRange/Y:**
- Когда `IsCentered = true`: `0` (скролл не нужен)
- Когда `IsCentered = false`: `CanvasPx - ViewportPx`

### 2. EditorCanvas.xaml — новая структура

```
Grid (2x2)
├── [0,0] Border (Stretch) → Canvas (Center/Center)
├── [0,1] ScrollBar Vertical
├── [1,0] ScrollBar Horizontal
└── [1,1] Rectangle (corner)
```

**Ключевые изменения:**
- Border: `HorizontalAlignment="Stretch"`, `VerticalAlignment="Stretch"`
- Canvas: `HorizontalAlignment="Center"`, `VerticalAlignment="Center"`
- TranslateTransform привязан к `CanvasOffsetX/Y`
- ScrollBar привязаны к `ScrollXRange/Y`, `ScrollXValue/Y`, `ViewportWidth/HeightPixels`
- Visibility через `NonZeroToVisibilityConverter`

### 3. EditorCanvas.xaml.cs — обработчики скроллбаров

## Архитектурные решения

### Центрирование через Border alignment

Вместо использования TranslateTransform для центрирования (что ломало координаты), используется `HorizontalAlignment="Center"` / `VerticalAlignment="Center"` на Canvas внутри Border. Это работает надёжнее и не влияет на координаты объектов.

### Разделение CanvasOffset и PanOffset

- `PanOffsetX/Y` — управляется скроллбарами и панорамированием мышью
- `CanvasOffsetX/Y` — итоговое смещение (0 когда центрирован, PanOffset когда скролл)

Это позволяет скроллбарам работать корректно при любом уровне zoom.

### Скроллбары привязаны к ViewModel

Кастомные ScrollBar привязаны к свойствам ViewModel через binding, а не используют встроенный ScrollViewer. Это обеспечивает полную совместимость с существующим механизмом панорамирования через TranslateTransform.
