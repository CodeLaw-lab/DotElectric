## 2. РЕАЛИЗОВАННЫЕ КОМПОНЕНТЫ

### 2.1. Attached Behaviors (2 класса)

#### MouseBehavior

**Файл:** `Behaviors/MouseBehavior.cs`

Универсальные attached properties для привязки ICommand к событиям мыши любого UIElement:

| Attached Property | Тип | Описание |
|-------------------|-----|----------|
| `OnMouseDown` | ICommand | Левая кнопка мыши (MouseLeftButtonDown) |
| `OnMouseMove` | ICommand | Перемещение мыши (MouseMove) |
| `OnMouseUp` | ICommand | Отпускание кнопки (MouseLeftButtonUp) |
| `OnMouseWheel` | ICommand | Прокрутка (PreviewMouseWheel, Delta в int) |
| `OnMouseDoubleClick` | ICommand | Двойной клик (ручная детекция 500мс) |
| `OnContextMenu` | ICommand | Правая кнопка (MouseRightButtonDown) |
| `OnContextMenuUp` | ICommand | Отпускание правой кнопки |

**Принцип работы:**
- DependencyProperty.RegisterAttached → PropertyMetadata → PropertyChanged callback
- В callback подписка/отписка от события UIElement
- В обработчике: `e.GetPosition(sender)` → `command.Execute(wpfPoint)`

#### EditorCanvasBehavior

**Файл:** `Behaviors/EditorCanvasBehavior.cs`

Attached behavior, привязываемый к Canvas через `EditorCanvasBehavior.Editor="{Binding}"`:

| Что делает | Как |
|------------|-----|
| Подписка на Canvas.MouseDown/Move/Up/PreviewMouseWheel | При установке свойства Editor |
| Конвертация WPF Point → PointMicrons | Учитывает zoom, ScrollViewer offset, инверсию Y |
| Создание текущего ITool | По Editor.ActiveTool (Select/Line/Rectangle/Text) |
| Делегирование в ITool.OnMouseDown/Move/Up/MouseWheel | С правильными параметрами |
| Зум через колесо | EditorViewModel.SetZoom(Zoom * 1.1 или * 0.9) |
| Поиск ScrollViewer | VisualTreeHelper.GetParent — рекурсивно |

**Координатная конвертация:**
```csharp
mmX = (wpfX + ScrollOffsetX) / zoom
mmY = sheetHeightMm - (wpfY + ScrollOffsetY) / zoom  // инверсия Y
return PointMicrons.FromMm(mmX, mmY);
```

**State-класс:** EditorCanvasState хранит Editor, LastButtonRaw, ScrollViewer

### 2.2. WPF Commands (4 класса)

**Файл:** `WpfCommands/ToolCommands.cs`

Обёртки System.Windows.Input.ICommand для биндинга в XAML:

| Класс | Назначение |
|-------|-----------|
| `ToolCommandBase` | Базовый: CanExecuteChanged, ToModelPoint(), GetModifiers() |
| `ToolMouseDownCommand` | Left button → ITool.OnMouseDown |
| `ToolMouseMoveCommand` | MouseMove → ITool.OnMouseMove (с отслеживанием кнопки) |
| `ToolMouseUpCommand` | Left button → ITool.OnMouseUp |
| `ToolMouseWheelCommand` | Delta → EditorViewModel.SetZoom (×1.1 или ×0.9) |

**Tool Factory:** Editor.ActiveTool string → new SelectTool/DrawingLineTool/DrawingRectangleTool/TextTool

### 2.3. EditorCanvas Converters (5 конвертеров)

**Файл:** `Converters/EditorCanvasConverters.cs`

| Конвертер | Тип | Формула |
|-----------|-----|---------|
| `ModelXToCanvasLeftConverter` | IMultiValueConverter | x_wpf = ToMm(micronsX) × zoom |
| `ModelYToCanvasTopConverter` | IMultiValueConverter | y_wpf = (sheetHeightMm - ToMm(micronsY)) × zoom |
| `MicronsToPixelConverter` | IMultiValueConverter | length_wpf = ToMm(microns) × zoom |
| `LineTypeToDashArrayConverter` | IValueConverter | Solid→null, Dashed→{10,5}, DashDot→{10,5,2,5} |
| `NotConverter` | IValueConverter | bool → !bool |

## 3. КЛЮЧЕВЫЕ АРХИТЕКТУРНЫЕ РЕШЕНИЯ

### 3.1. No Code-Behind подход

Вся логика реализована через:
1. **Attached Behaviors** — подписка на события, конвертация координат
2. **WPF Commands** (System.Windows.Input.ICommand) — делегирование в ITool
3. **DataTemplates** — рендеринг объектов через MultiBinding-конвертеры
4. **LayoutTransform** — зум через ScaleTransform

**Почему:** UI/UX специалист может модифицировать XAML без знания C#. Вся бизнес-логика инкапсулирована в behaviors и converters.

### 3.2. Разделение ответственности

```
Behaviors/      → Обработка событий мыши, конвертация координат
WpfCommands/    → ICommand-обёртки для XAML binding
Converters/     → Модельные координаты → WPF пиксели
Views/          → XAML разметка, DataTemplates
Tools/          → ITool логика (SelectTool, DrawingLineTool, etc.)
ViewModels/     → Состояние (EditorViewModel, MainViewModel)
```

### 3.3. Конвертация координат в двух местах

| Место | Направление | Где |
|-------|------------|-----|
| **EditorCanvasBehavior** | WPF Point → PointMicrons | При событиях мыши → ITool |
| **EditorCanvasConverters** | Микроны → WPF пиксели | При рендеринге объектов в XAML |

Это необходимо, потому что:
- ITool работает с модельными координатами (не знает про WPF)
- XAML работает с WPF-координатами (Canvas.Left, Canvas.Top)
- Конвертеры не могут зависеть от behaviors (чистые функции)

### 3.4. Tool Factory

Каждый WpfCommand создаёт новый экземпляр ITool при каждом вызове Execute:
```csharp
return Editor.ActiveTool switch {
    "Select" => new SelectTool(Editor),
    "Line" => new DrawingLineTool(Editor),
    ...
};
```

**Почему:** State Pattern — каждый инструмент хранит своё состояние (начальная точка, preview). Новый экземпляр = чистое состояние.

## 7. ИЗВЕСТНЫЕ ОГРАНИЧЕНИЯ

1. **Tool Factory создаёт новые экземпляры** при каждом вызове → потери состояния между MouseDown/MouseMove/MouseUp
2. **MouseBehavior не поддерживает** правую кнопку для SelectTool (контекстное меню)
3. **EditorCanvasBehavior** не обрабатывает клавиатуру (Delete, стрелки, Ctrl+C/V/Z/Y)
4. **DataTemplates** для Line/Rectangle/Text используют MultiBinding на каждое свойство → потенциальные проблемы с производительностью при большом количестве объектов
5. **Preview-элементы** не привязаны к XAML (TODO)
