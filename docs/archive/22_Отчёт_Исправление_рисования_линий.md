# Отчёт — Исправление рисования линий

**Дата:** 07.04.2026
**Задача:** Исправить рисование линий — линии рисуются некорректно, preview не отображается

---

## ПРОБЛЕМЫ

### Проблема 1: Двойное масштабирование Canvas
**Файл:** `Views/EditorCanvas.xaml`

Canvas имел **двойное** масштабирование:
1. `Width/Height` через `MicronsToPixelConverter` → `sheetWidthMm * zoom`
2. `LayoutTransform.ScaleTransform` → ещё раз `* zoom`

Итоговый визуальный размер: `sheetWidthMm × zoom²` вместо `sheetWidthMm × zoom`.

### Проблема 2: Лишний учёт прокрутки в координатах
**Файл:** `Behaviors/EditorCanvasBehavior.cs`, метод `ToModelPoint()`

```csharp
// БЫЛО — прибавлял scrollOffset, хотя e.GetPosition(canvas) уже относительно Canvas
var mmX = (wpfPoint.X + scrollOffsetX) / zoom;
var mmY = (wpfPoint.Y + scrollOffsetY) / zoom;
```

`e.GetPosition(canvas)` возвращает координаты **относительно Canvas**, а не ScrollViewer. Прокрутка сдвигает viewport, но координаты внутри Canvas не меняются. Прибавление `scrollOffsetX/Y` было избыточным.

### Проблема 3: MultiBinding не обновлялся для preview-элементов
**Файл:** `Views/EditorCanvas.xaml`

Binding `Path="PreviewLine.StartMicronsX"` (цепочка свойств через DataContext) не обновлялся при изменении свойств `Line`. WPF отслеживает `PropertyChanged` только от первого объекта в цепочке (`EditorViewModel.PreviewLine`), но НЕ от вложенного `Line.StartMicronsX`.

При `_editor.PreviewLine = new Line(...)` генерируется `PropertyChanged` для `PreviewLine`, но конвертер MultiBinding **не перечитывает** значения `StartMicronsX`/`EndMicronsX` нового объекта — binding "залипает" на старых значениях.

---

## РЕШЕНИЯ

### Решение 1: Убран LayoutTransform
**Файл:** `EditorCanvas.xaml`

Убран `<Canvas.LayoutTransform><ScaleTransform ScaleX="{Binding Zoom}" ScaleY="{Binding Zoom}"/></Canvas.LayoutTransform>`.

Zoom уже учтён в `Canvas.Width/Height` через `MicronsToPixelConverter`.

### Решение 2: Убран scrollOffset из конвертации
**Файл:** `Behaviors/EditorCanvasBehavior.cs`

```csharp
// СТАЛО — без scrollOffset
var mmX = wpfPoint.X / zoom;
var mmY = wpfPoint.Y / zoom;
```

### Решение 3: PreviewLineChangedBehavior — прямое обновление WPF-элементов
**Новый файл:** `Behaviors/PreviewLineChangedBehavior.cs`

Подписывается на `PropertyChanged` EditorViewModel и обновляет WPF-элементы (`PreviewLineElement`, `PreviewRectangleElement`, `PreviewTextElement`) напрямую через конвертеры координат:

```csharp
public static void RegisterCanvas(Canvas canvas, EditorViewModel vm)
{
    vm.PropertyChanged += OnVmPropertyChanged;
}

private static void OnVmPropertyChanged(...)
{
    if (e.PropertyName == nameof(EditorViewModel.PreviewLine))
        UpdatePreviewLine(canvas, vm, zoom, sheetHeightMm, converterX, converterY);
    // ...
}
```

Регистрация в `EditorCanvas.xaml.cs`:
```csharp
public EditorCanvas()
{
    InitializeComponent();
    DataContextChanged += OnDataContextChanged;
    Loaded += OnLoaded;
}

private void OnDataContextChanged(...)
{
    if (e.NewValue is EditorViewModel vm)
        PreviewLineChangedBehavior.RegisterCanvas(DrawingCanvas, vm);
}
```

**Preview-элементы в XAML** — простые элементы без binding:
```xml
<Line x:Name="PreviewLineElement" Stroke="Red" StrokeThickness="1.5"
      StrokeDashArray="4,2" Visibility="Collapsed"/>
```
