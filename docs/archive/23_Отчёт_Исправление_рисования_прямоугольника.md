# Отчёт — Исправление рисования прямоугольника

**Дата:** 07.04.2026
**Задача:** Исправить рисование прямоугольника — финальный прямоугольник перескакивает в левый верхний угол

---

## ПРОБЛЕМЫ

### Проблема 1: Canvas.Left/Top задавались на Content внутри ContentPresenter
**Файл:** `Views/EditorCanvas.xaml` — DataTemplate для Rectangle и Text

`ItemsControl` обёртывает каждый объект в `ContentPresenter`. `Canvas.Left/Top`, заданные внутри `DataTemplate` (на самом `Rectangle` или `TextBlock`), **игнорировались** WPF, т.к. они были не на прямом потомке Canvas, а на элементе **внутри** `ContentPresenter`.

В результате все прямоугольники и тексты появлялись в `(0, 0)` Canvas — левом верхнем углу.

### Проблема 2: Canvas.Top использовал MicronsY вместо BottomMicronsY
**Файлы:** `Views/EditorCanvas.xaml`, `Behaviors/PreviewLineChangedBehavior.cs`

`MicronsY` у Rectangle = **нижний** левый угол. `Canvas.Top` в WPF = позиция **верхнего** края элемента. Конвертер `ModelYToCanvasTopConverter` превращает модельный Y в Canvas.Top. При `Canvas.Top = ToCanvasTop(MicronsY)` прямоугольник сдвигался вниз на свою высоту.

### Проблема 3: Нет Canvas.Left/Top на ItemContainer
**Файл:** `Views/EditorCanvas.xaml` — ItemContainerStyle

ItemContainerStyle был пустым — координаты не передавались на ContentPresenter.

---

## РЕШЕНИЯ

### Решение 1: Canvas.Left/Top перенесены в ItemContainerStyle
**Файл:** `EditorCanvas.xaml`

```xml
<!-- БЫЛО: ContentPresenter с пустым стилем, координаты внутри DataTemplate -->
<ItemsControl.ItemContainerStyle>
    <Style TargetType="ContentPresenter">
        <!-- Пусто -->
    </Style>
</ItemsControl.ItemContainerStyle>

<!-- СТАЛО: Canvas.Left/Top на ContentPresenter -->
<ItemsControl.ItemContainerStyle>
    <Style TargetType="ContentPresenter">
        <Setter Property="Canvas.Left">
            <Setter.Value>
                <MultiBinding Converter="{StaticResource ModelXToCanvasLeftConverter}">
                    <Binding Path="MicronsX"/>
                    <Binding Path="DataContext.Zoom" RelativeSource="{RelativeSource AncestorType=UserControl}"/>
                </MultiBinding>
            </Setter.Value>
        </Setter>
        <Setter Property="Canvas.Top">
            <Setter.Value>
                <MultiBinding Converter="{StaticResource ModelYToCanvasTopConverter}">
                    <Binding Path="." Converter="{StaticResource TopEdgeMicronsConverter}"/>
                    <Binding Path="DataContext.Template.Sheet.HeightMm" RelativeSource="{RelativeSource AncestorType=UserControl}"/>
                    <Binding Path="DataContext.Zoom" RelativeSource="{RelativeSource AncestorType=UserControl}"/>
                </MultiBinding>
            </Setter.Value>
        </Setter>
    </Style>
</ItemsControl.ItemContainerStyle>
```

### Решение 2: Убраны Canvas.Left/Top из DataTemplate
**Файл:** `EditorCanvas.xaml`

Из DataTemplate для Rectangle убраны `Canvas.Left` и `Canvas.Top` — координаты задаются на ContentPresenter через ItemContainerStyle. DataTemplate содержит только Width/Height.

Аналогично для Text — убраны Canvas.Left/Top.

### Решение 3: Новый конвертер TopEdgeMicronsConverter
**Новый файл:** `Converters/CommonConverters.cs`

```csharp
public class TopEdgeMicronsConverter : IValueConverter
{
    public object Convert(object? value, ...) => value switch
    {
        Rectangle rect => rect.MicronsY + rect.HeightMicrons,  // BottomMicronsY
        Line line => Math.Max(line.StartMicronsY, line.EndMicronsY),
        Text text => text.MicronsY,
        _ => 0L
    };
}
```

### Решение 4: PreviewRectangle исправлен
**Файл:** `Behaviors/PreviewLineChangedBehavior.cs`

```csharp
// БЫЛО: MicronsY
Canvas.SetTop(rect, convY.Convert({ preview.MicronsY, ... }));

// СТАЛО: MicronsY + HeightMicrons (верхний край)
Canvas.SetTop(rect, convY.Convert({ preview.MicronsY + preview.HeightMicrons, ... }));
```
