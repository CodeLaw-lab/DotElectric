# Отчёт — Исправление двойного индикатора несохранённых изменений

**Дата:** 07.04.2026
**Задача:** Исправить дублирование `*` в заголовке вкладки при несохранённых изменениях

---

## ПРОБЛЕМА

При изменении шаблона в заголовке вкладки отображались **две** звёздочки:
1. Одна `*` — добавлялась в строку `DisplayName` методом `UpdateDisplayName()` в `EditorViewModel.cs`
2. Вторая `*` (оранжевая) — отрисовывалась через `TextBlock` в `ControlTemplate` TabItem (DarkTheme.xaml / LightTheme.xaml)

Результат: заголовок выглядел как `my_template.tdel **` вместо ожидаемого `my_template.tdel *`.

---

## ПРИЧИНА

Два независимых механизма отображали индикатор несохранённых изменений:

| Механизм | Расположение | Как работал |
|----------|--------------|-------------|
| `UpdateDisplayName()` | `EditorViewModel.cs` | Добавлял ` *` к строке DisplayName |
| `DirtyIndicator` | `ControlTemplate` в `DarkTheme.xaml` / `LightTheme.xaml` | Оранжевый `TextBlock` с `Visibility` через `DataTrigger` на `IsDirty` |

---

## РЕШЕНИЕ

Удалить добавление `*` из `UpdateDisplayName()`, оставив только ControlTemplate-индикатор.

### Изменённые файлы

| Файл | Изменение |
|------|-----------|
| `ViewModels/EditorViewModel.cs` | `UpdateDisplayName()` больше не добавляет ` *` |
| `ViewModels/EditorViewModelTests.cs` | Обновлены 2 теста |

### Детали изменений

**EditorViewModel.cs:**
```csharp
// БЫЛО:
DisplayName = IsDirty ? $"{baseName} *" : baseName;

// СТАЛО:
DisplayName = baseName;
```

Комментарий к методу обновлён: «Обновить DisplayName (без * — индикатор отображается в ControlTemplate TabItem)».

### Обновлённые тесты

| Тест | Было | Стало |
|------|------|-------|
| `MarkDirty_AddsAsteriskToDisplayName` | `Assert.EndsWith(" *", vm.DisplayName)` | `MarkDirty_DisplayNameUnchanged_IsDirtyTrue` — проверяет `DisplayName` неизменность + `IsDirty == true` |
| `DisplayName_AfterClearDirty_RemovesAsterisk` | `Assert.EndsWith(" *", ...)` + `Assert.False(...EndsWith(" *"))` | `DisplayName_AfterClearDirty_IsDirtyFalse` — проверяет `IsDirty == false` после `ClearDirty()` |
