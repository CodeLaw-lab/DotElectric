## 3. Исправленные баги

### Баг #1: Панель свойств не применяла изменения к линии

**Описание:** При вводе координаты X1 = 50 в панели свойств, линия не двигалась.

**Причина:** `long.TryParse(tb.Text, out long microns)` парсил «50» как 50 микрон, но TextBox отображал значение в мм (через `MicronsToMmConverter`). Ввод «50» означал 50мм = 50 000 микрон.

**Исправление:**
```csharp
// Было:
if (long.TryParse(tb.Text, out long microns))
    vm.ChangeLineStartXCommand.Execute(microns);

// Стало:
if (double.TryParse(tb.Text, NumberStyles.Float, CultureInfo.InvariantCulture, out double mm))
    vm.ChangeLineStartXCommand.Execute(Coordinate.ToMicrons(mm));
```

**Затронутые поля:** Line (X1, Y1, X2, Y2), Rectangle (X, Y, Width, Height), Text (X, Y, FontSize).

---

### Баг #2: Линия не обновлялась при редактировании свойств

**Описание:** При изменении X1 в панели свойств маркеры двигались, но линия оставалась на месте.

**Причина:** DataTemplate линии использовал `<Binding Path="."/>` — WPF не подписывается на PropertyChanged при binding к целому объекту. Конвертеры LocalX1/Y1/X2/Y2 не пересчитывались.

**Исправление:**
```xml
<!-- Было (НЕ РАБОТАЛО): -->
<Binding Path="."/>

<!-- Стало (РАБОТАЕТ): -->
<Binding Path="StartMicronsX"/>
<Binding Path="EndMicronsX"/>
```

Конвертеры обновлены на приём отдельных значений:
```csharp
// Было:
if (values[0] is not Line line) return 0.0;

// Стало:
if (values[0] is not long startX) return 0.0;
if (values[1] is not long endX) return 0.0;
```

---

### Баг #3: ContentPresenter не перемещался при редактировании

**Описание:** Линия/прямоугольник/текст не перемещались при изменении координат, хотя маркеры двигались.

**Причина:** ItemContainerStyle использовал `<Binding Path="." Converter="{StaticResource LeftEdgeMicronsConverter}"/>` — binding к `.` не подписывается на PropertyChanged.

**Исправление:** Созданы `LeftEdgeMicronsMultiConverter` и `TopEdgeMicronsMultiConverter`, которые принимают конкретные свойства:

```xml
<!-- Line: -->
<Binding Path="StartMicronsX"/>
<Binding Path="EndMicronsX"/>
<Binding Path="MicronsX"/>  <!-- Rectangle, Text -->
<Binding Path="DataContext.Zoom" .../>

<!-- Top (Y): -->
<Binding Path="StartMicronsY"/>     <!-- Line -->
<Binding Path="EndMicronsY"/>       <!-- Line -->
<Binding Path="MicronsY"/>          <!-- Rectangle -->
<Binding Path="HeightMicrons"/>     <!-- Rectangle -->
<Binding Path="MicronsY"/>          <!-- Text -->
<Binding Path="FontSizeMicrons"/>   <!-- Text -->
```

---

### Баг #4: Rectangle определялся как Text

**Описание:** При использовании TopEdgeMicronsMultiConverter Rectangle ошибочно определялся как Text.

**Причина:** Проверка `values[4] is long` (MicronsY) срабатывала и для Rectangle, и для Text.

**Исправление:** Проверка Text теперь использует `values[5]` (FontSizeMicrons), который есть только у Text:
```csharp
var isText = values[5] != UnsetValue && values[5] is long;
```

---

### Баг #5: Сетка пропадала при zoom < 25%

**Описание:** При уменьшении масштаба сетка полностью исчезала.

**Причина:** Добавленная проверка `MinPixelSpacing` в `GenerateGridNodes` отключала генерацию узлов при spacing < 5px.

**Исправление:** Убрана проверка `MinPixelSpacing` из `GenerateGridNodes` — узлы генерируются при любом масштабе.

---

### Баг #6: Сетка смещена относительно листа

**Описание:** Узлы сетки не совпадали с краями листа при разных форматах и ориентациях.

**Причина:** Двойное смещение — Canvas уже имел `RenderTransform` с PanOffset, но координаты узлов тоже рассчитывались с `- PanOffsetX/Y`.

**Исправление:**
```csharp
// Было:
XPixels = Coordinate.ToMm(node.XMicrons) * Zoom - PanOffsetX;

// Стало:
XPixels = Coordinate.ToMm(node.XMicrons) * Zoom;
```

---

### Баг #7: Сетка не обновлялась при ресайзе окна

**Описание:** При изменении размера окна сетка не пересчитывалась.

**Причина:** `OnSizeChanged` обновлял `ViewportWidthMm/HeightMm`, но не триггерил `RefreshGridNodes()`.

**Исправление:** Добавлены partial-методы:
```csharp
partial void OnViewportWidthMmChanged(double value)
{
    if (value > 0 && _viewportHeightMm > 0)
        RefreshGridNodes();
}
```
