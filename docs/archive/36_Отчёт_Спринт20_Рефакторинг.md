### P0 — Критические исправления (4/4)

#### 1. Утечка памяти в EditorViewModel.Dispose() 🔴

**Проблема:** `EditorViewModel` подписывался на `SelectedObjects.CollectionChanged` в конструкторе, но **не отписывался** в `Dispose()`. Каждый закрытый таб вёл к утечке памяти — объект никогда не собирался GC.

**Также:** `PropertiesViewModel` использовал lambda-подписку `(s, e) => UpdateSelection()` — невозможно отписаться.

**Решение:**
```csharp
// EditorViewModel.Dispose()
SelectedObjects.CollectionChanged -= SelectedObjectsChanged;
PropertiesVm.Dispose();

// PropertiesViewModel — реализован IDisposable
private void OnSelectedObjectsChanged(object? sender, ...) { ... }
public void Dispose() => _selectedObjects.CollectionChanged -= OnSelectedObjectsChanged;
```

**Файлы:**
- `ViewModels/EditorViewModel.cs` — добавлена отписка и вызов `PropertiesVm.Dispose()`
- `ViewModels/PropertiesViewModel.cs` — реализован `IDisposable`, lambda заменена на именованный метод

#### 2. Text.WidthMicrons не обновлял UI 🔴

**Проблема:** Вычисляемые свойства `WidthMicrons`, `RightMicronsX`, `BottomMicronsY`, `CenterMicronsX`, `CenterMicronsY` зависят от `Content` и `FontSizeMicrons`, но при их изменении UI **не обновлялся** — отсутствовали `[NotifyPropertyChangedFor]`.

**Последствие:** При редактировании текста через панель свойств маркеры выделения и bounding box **не перерисовывались**.

**Решение:**
```csharp
[ObservableProperty]
[NotifyPropertyChangedFor(nameof(WidthMicrons))]
[NotifyPropertyChangedFor(nameof(RightMicronsX))]
[NotifyPropertyChangedFor(nameof(CenterMicronsX))]
private string _content = string.Empty;

[ObservableProperty]
[NotifyPropertyChangedFor(nameof(WidthMicrons))]
[NotifyPropertyChangedFor(nameof(RightMicronsX))]
[NotifyPropertyChangedFor(nameof(CenterMicronsX))]
[NotifyPropertyChangedFor(nameof(BottomMicronsY))]
[NotifyPropertyChangedFor(nameof(CenterMicronsY))]
private long _fontSizeMicrons;
```

**Файлы:**
- `Models/Objects/Text.cs` — добавлены 9 атрибутов `[NotifyPropertyChangedFor]`

#### 3. Кнопки Undo/Redo не обновляли состояние 🔴

**Проблема:** После `_commandHistory.Undo()` `CanUndo` меняется, но RelayCommand **не уведомлял** UI о изменении `CanExecute`. Кнопки оставались активными/неактивными некорректно.

**Решение:** Добавлен `NotifyCanExecuteChanged()` после каждого Undo/Redo, а также в `MarkDirty()`:
```csharp
private void Undo()
{
    _commandHistory.Undo();
    MarkDirty();
    UndoCommand.NotifyCanExecuteChanged();
    RedoCommand.NotifyCanExecuteChanged();
    OnPropertyChanged(nameof(UndoDisplayName));
    OnPropertyChanged(nameof(RedoDisplayName));
}
```

**Файлы:**
- `ViewModels/EditorViewModel.cs` — `Undo()`, `Redo()`, `MarkDirty()`

#### 4. Template наследовал ObservableObject без уведомлений 🔴

**Проблема:** `Template : ObservableObject` но `Version`, `Metadata`, `Sheet` — обычные auto-properties без `[ObservableProperty]`. Ложное ожидание, лишнее наследование.

**Решение:** Убрано наследование `ObservableObject` — `Template` создаётся один раз и не заменяется после инициализации.

**Файлы:**
- `Models/Template.cs` — `partial class Template : ObservableObject` → `class Template`

### P1 — Серьёзные исправления (4/5 выполнено)

#### 5. Отсутствие логирования в MainViewModel 🟡

**Проблема:** `MainViewModel` перехватывал исключения, но **НЕ логировал** их — stack trace терялся, невозможна диагностика проблем у пользователя.

**Решение:**
- Добавлен `ILogger<MainViewModel>` в DI-контейнер и конструктор
- Все 4 catch-блока теперь логируют через `_logger.LogError(ex, ...)`

**Файлы:**
- `ViewModels/MainViewModel.cs` — +ILogger, логирование
- `App.xaml.cs` — комментарий о настроенном Serilog
- `ViewModels/MainViewModelTests.cs` — добавлен `CreateMockLogger()` во все 12 созданий VM

#### 6. Async без await в AutosaveService 🟡

**Проблема:** `AutosaveTabAsync` и `SaveSessionAsync` были помечены `async Task`, но не использовали `await` — лишняя аллокация state machine.

**Решение:**
- `AutosaveTabAsync` → `AutosaveTab` (синхронный)
- `SaveSessionAsync` → `SaveSession` (синхронный, `File.WriteAllText` вместо `File.WriteAllTextAsync`)

**Файлы:**
- `Services/AutosaveService.cs`

---

#### 7. Избыточный OnPropertyChanged в PropertiesViewModel 🟡

**Проблема:** `UpdateSelection()` вызывал `OnPropertyChanged` для ВСЕХ тип-специфичных свойств (Line*, Rect*, Text*) даже когда выделен объект только одного типа.

**Решение:** Теперь уведомляются только свойства актуального типа:
```csharp
if (IsLineSelected) { /* Line* */ }
else if (IsRectangleSelected) { /* Rect* */ }
else if (IsTextSelected) { /* Text* */ }
// Общие — всегда
OnPropertyChanged(nameof(ObjectId));
OnPropertyChanged(nameof(ObjectTypeName));
```

**Файлы:**
- `ViewModels/PropertiesViewModel.cs`
- `ViewModels/PropertiesViewModelExtendedTests.cs` — обновлён тест

## АРХИТЕКТУРНЫЕ РЕШЕНИЯ

| Решение | Обоснование |
|---------|-------------|
| **Template без ObservableObject** | Template создаётся один раз, не заменяется — уведомления не нужны |
| **IDisposable в PropertiesViewModel** | Обязательная отписка от CollectionChanged для предотвращения утечек |
| **NotifyPropertyChangedFor в Text** | Вычисляемые свойства зависят от ObservableProperty — source generator обеспечит уведомления |
| **Синхронные методы в AutosaveService** | `_templateService.Save` — синхронный, нет смысла в async без реального IO |
| **Тип-специфичный OnPropertyChanged** | Меньше лишних уведомлений — лучше производительность UI |
