### 9. Print Service — детали реализации [РЕШЕНО]

**Статус:** ✅ Реализовано в Sprint 39-41

**Контекст:**
Требование FR-050: «Печатать шаблон через стандартный диалог печати Windows».

**Вопросы:**
1. `PrintDialog` из `System.Printing` или `System.Windows.Controls.PrintDialog`?
2. Как обрабатывается масштаб 1:1 при разных DPI принтера?
3. Как обрабатываются поля принтера (printable area)?
4. Как генерируется PrintVisual — весь Canvas или отдельный `FixedDocument`?
5. Предпросмотр печати — отдельное окно или встроенный диалог?

**Рекомендация:**

| Параметр | Решение |
|----------|---------|
| **PrintDialog** | `System.Windows.Controls.PrintDialog` (стандартный WPF, не требует `System.Printing`) |
| **Печать** | `PrintVisual(canvas, "DotElectric Template")` — печать всего Canvas |
| **DPI** | WPF автоматически обрабатывает DPI через `PrintVisual`. Canvas уже в device-independent units (1/96 inch). |
| **Printable area** | `PrintDialog.PrintableAreaWidth/Height` — использовать для проверки, но не обрезать автоматически |
| **Предпросмотр** | Отложить на Этап 2. На Этапе 1 — сразу печать через диалог. |
| **PDF** | Через системный PDF-принтер (Microsoft Print to PDF, встроен в Windows 10/11) |

**Пример кода (для реализации):**

```csharp
public class PrintService : IPrintService
{
    public bool Print(Visual visual, string description)
    {
        var dialog = new System.Windows.Controls.PrintDialog();
        
        if (dialog.ShowDialog() != true)
            return false;
        
        // WPF автоматически масштабирует под DPI принтера
        dialog.PrintVisual(visual, description);
        return true;
    }
}
```

**Риски:**
- ⚠️ Если принтер не поддерживает размер листа шаблона — может обрезаться
- ⚠️ `PrintableAreaWidth/Height` может быть null для некоторых принтеров

**Когда решать:** Во время реализации FR-050 (середина Этапа 1).

### 10. Темы — механизм переключения [РЕШЕНО]

**Статус:** ✅ РЕШЕНО — реализовано

**Контекст:**
Требование ТЗ (раздел 3.2): «Светлая и тёмная тема оформления».

**Вопросы:**
1. DynamicResource или ресурсные словари, переключаемые через `Application.Current.Resources.MergedDictionaries`?
2. Где хранятся ResourceDictionary — отдельные файлы или App.xaml?
3. Как переключается тема — через `ISettingsService` при запуске?
4. Холст остаётся белым в любой теме?

**Рекомендация:**

| Параметр | Решение |
|----------|---------|
| **Механизм** | `Application.Current.Resources.MergedDictionaries` — очистка и загрузка нового ResourceDictionary |
| **Хранение** | Отдельные файлы: `Resources/Styles/LightTheme.xaml`, `Resources/Styles/DarkTheme.xaml` |
| **Переключение** | Через `ISettingsService.Theme` → перезагрузка MergedDictionaries |
| **Ключи ресурсов** | DynamicResource для всех цветов/кистей (StaticResource не обновляется при смене) |
| **Холст** | Всегда белый (как в спецификации). Тема влияет только на панели, меню, фон окна |
| **Тема по умолчанию** | Светлая |

**Пример кода (для реализации):**

```csharp
// В App.xaml.cs или ThemeService.cs
public void SetTheme(ThemeType theme)
{
    var dict = new ResourceDictionary { Source = new Uri(
        theme == ThemeType.Light
            ? "Resources/Styles/LightTheme.xaml"
            : "Resources/Styles/DarkTheme.xaml",
        UriKind.Relative) };

    // Находим и заменяем тему в MergedDictionaries
    var dictionaries = Application.Current.Resources.MergedDictionaries;
    var oldTheme = dictionaries.FirstOrDefault(
        d => d.Source?.OriginalString.Contains("Theme.xaml") == true);
    
    if (oldTheme != null)
        dictionaries.Remove(oldTheme);
    dictionaries.Add(dict);

    _settingsService.Set("Theme", theme);
}
```

**Пример LightTheme.xaml:**

```xml
<ResourceDictionary xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
                    xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml">
    <SolidColorBrush x:Key="WindowBackground" Color="#F5F5F5"/>
    <SolidColorBrush x:Key="PanelBackground" Color="#FFFFFF"/>
    <SolidColorBrush x:Key="TextPrimary" Color="#212121"/>
    <SolidColorBrush x:Key="AccentColor" Color="#0078D4"/>
    <!-- ... -->
</ResourceDictionary>
```

**Риски:**
- ⚠️ DynamicResource медленнее StaticResource (но для desktop это незаметно)
- ⚠️ При переключении может быть «мигание» (решается через BeginInit/EndInit)

**Когда решать:** Во время создания UI (начало Этапа 1, после моделей и ViewModel).

### 12. Валидация — UI-интеграция [РЕШЕНО]

**Статус:** ✅ РЕШЕНО — все правила реализованы

**Контекст:**
Правила V-001 — V-007 (валидация XML):
- V-001: Уникальные ID объектов
- V-002: Уникальные ключи изменяемых текстовых полей
- V-003: Координаты в пределах листа
- V-004: Положительные размеры
- V-005: Корректные HEX-цвета
- V-006: Корректные имена форматов
- V-007: Корректные типы линий

**Вопросы:**
1. Где происходит валидация — при сохранении, при каждом изменении, или при загрузке?
2. Как валидация связана с UI — `INotifyDataErrorInfo`, или отдельный сервис?
3. Что происходит при загрузке невалидного файла — отклоняется полностью или игнорируются битые объекты?

**Рекомендация:**

| Параметр | Решение |
|----------|---------|
| **Валидация при сохранении** | Sync, через `ValidationService.Validate(template)` → `IEnumerable<ValidationError>` |
| **Валидация при загрузке** | Async, битые объекты игнорируются + показывается список ошибок |
| **Валидация при изменении** | НЕ в реальном времени (слишком дорого). Только при Save и Load. |
| **UI-интеграция** | `ValidationResult` dialog при сохранении. Показывает список ошибок, блокирует сохранение до исправления. |
| **INotifyDataErrorInfo** | НЕ используем (слишком сложно для Этапа 1). Простой dialog с списком ошибок. |
| **ValidationService** | Отдельный метод `Validate(Template)` → `IEnumerable<ValidationError>` |

**Пример кода (для реализации):**

```csharp
public class ValidationError
{
    public string RuleId { get; init; }  // V-001, V-002, ...
    public string Message { get; init; }
    public string? ObjectId { get; init; }
    public ValidationSeverity Severity { get; init; }
}

public enum ValidationSeverity
{
    Error,    // Блокирует сохранение
    Warning   // Предупреждение, но можно сохранить
}

public class ValidationService
{
    public IEnumerable<ValidationError> Validate(Template template)
    {
        var errors = new List<ValidationError>();

        // V-001: Уникальные ID
        var ids = template.Objects.Select(o => o.Id).ToList();
        if (ids.Count != ids.Distinct().Count())
            errors.Add(new ValidationError
            {
                RuleId = "V-001",
                Message = "Обнаружены неуникальные ID объектов",
                Severity = ValidationSeverity.Error
            });

        // V-002: Уникальные ключи текстовых полей
        var texts = template.Objects.OfType<Text>()
            .Where(t => t.IsEditable)
            .ToList();
        var keys = texts.Select(t => t.Key).ToList();
        if (keys.Count != keys.Distinct().Count())
            errors.Add(new ValidationError
            {
                RuleId = "V-002",
                Message = "Обнаружены неуникальные ключи текстовых полей",
                Severity = ValidationSeverity.Error
            });

        // V-003: Координаты в пределах листа
        foreach (var obj in template.Objects)
        {
            if (obj.MicronsX < 0 || obj.MicronsY < 0 ||
                obj.MicronsX > template.Sheet.WidthMicrons ||
                obj.MicronsY > template.Sheet.HeightMicrons)
            {
                errors.Add(new ValidationError
                {
                    RuleId = "V-003",
                    Message = $"Объект {obj.Id} за пределами листа",
                    ObjectId = obj.Id,
                    Severity = ValidationSeverity.Error
                });
            }
        }

        // V-004: Положительные размеры (для Rectangle, Text)
        // V-005: Корректные HEX-цвета
        // V-006: Корректные имена форматов
        // V-007: Корректные типы линий

        return errors;
    }
}
```

**UI dialog при сохранении:**

```
┌────────────────────────────────────────┐
│  Ошибки валидации                      │
├────────────────────────────────────────┤
│  ✖ V-001: Неуникальные ID объектов     │
│  ✖ V-003: Объект line5 за пределами    │
│    листа                               │
│                                        │
│  ⚠ V-005: Неизвестный цвет #XYZ (будет │
│    использован #000000)                │
│                                        │
│  [OK] [Отмена]                         │
└────────────────────────────────────────┘
```

**Риски:**
- ⚠️ При загрузке битого файла — молчаливое игнорирование объектов (пользователь может не заметить)
- ⚠️ Валидация при Save — блокирует работу, если ошибки не исправить быстро

**Когда решать:** Во время реализации FileService (середина Этапа 1).

### 13. Шрифты ГОСТ — доставка в приложение [РЕШЕНО]

**Статус:** ✅ РЕШЕНО — GOST шрифты внедрены

**Контекст:**
Требование SEC-005: «Шрифты (ГОСТ) должны поставляться в комплекте с приложением».
Шрифты: ГОСТ А (для заголовков), ГОСТ Б (для основного текста).

**Вопросы:**
1. Как шрифты доставляются — встраиваются как Resource в сборку, или копируются в папку приложения?
2. Как WPF их находит — `pack://application:` URI?
3. Лицензирование шрифтов — есть ли ограничения на распространение?

**Рекомендация:**

| Параметр | Решение |
|----------|---------|
| **Встраивание** | Resource в сборку (Build Action = Resource) |
| **XAML использование** | `FontFamily="pack://application:,,,/Resources/Fonts/#ГОСТ А"` |
| **C# использование** | `new FontFamily("pack://application:,,,/Resources/Fonts/#ГОСТ А")` |
| **Файлы** | `Resources/Fonts/GostA.ttf`, `Resources/Fonts/GostB.ttf` |
| **Лицензирование** | Требуется проверка. Шрифты ГОСТ — свободные, но нужно убедиться. |
| **Fallback** | Если шрифт не найден → Arial (default WPF fallback) |

**Пример (для реализации):**

```xml
<!-- Text объект на Canvas -->
<TextBlock
    FontFamily="pack://application:,,,/DotElectric.TemplateEditor;
        component/Resources/Fonts/#ГОСТ Б"
    FontSize="14"
    Text="Наименование документа"/>
```

**Структура файлов:**

```
Resources/
└── Fonts/
    ├── GostA.ttf        # Build Action: Resource
    └── GostB.ttf        # Build Action: Resource
```

**В .csproj:**

```xml
<ItemGroup>
    <Resource Include="Resources\Fonts\*.ttf" />
</ItemGroup>
```

**Риски:**
- ⚠️ **Критично:** нужно проверить лицензию шрифтов ГОСТ до распространения
- ⚠️ Шрифт ГОСТ может не поддерживать все символы Unicode (кириллица — ОК, спецсимволы —?)
- ⚠️ Resource увеличивает размер сборки (~100-300 KB на шрифт) — но это ничтожно

**Когда решать:** При создании первых текстовых объектов (середина Этапа 1).

```csharp
public interface ITemplateLibraryService
{
    string TemplatesFolder { get; }  // %APPDATA%\DotElectric\Templates
    IReadOnlyList<TemplateInfo> LoadTemplateInfos();  // *.tdel, сортировка по DisplayName
}

public record TemplateInfo(string FileName, string DisplayName, string FullPath);
```

**Рекомендация:**

| Параметр | Решение |
|----------|---------|
| **SingleLine** | `TextBlock` с `TextTrimming="CharacterEllipsis"` (обрезка с `...`) |
| **MultiLine** | `TextBlock` с `TextWrapping="Wrap"` |
| **Ширина/высота** | Фиксированные (задаются при создании/ресайзе через маркеры) |
| **Перенос слов** | Стандартный WPF `TextWrapping` (по пробелам, дефисам) |
| **Режим редактирования** | `TextBox` с `AcceptsReturn="True"` (для MultiLine) |
| **Переполнение** | SingleLine: `TextTrimming`. MultiLine: `TextBlock` обрезается по высоте. |

**Пример XAML (для реализации):**

```xml
<!-- SingleLine Text -->
<TextBlock
    Text="{Binding Content}"
    TextTrimming="CharacterEllipsis"
    FontFamily="..."
    FontSize="{Binding FontSizePt}"/>

<!-- MultiLine Text -->
<TextBlock
    Text="{Binding Content}"
    TextWrapping="Wrap"
    TextTrimming="None"
    FontFamily="..."
    FontSize="{Binding FontSizePt}"/>
```

**Риски:**
- ⚠️ `TextBlock` не поддерживает вертикальное выравнивание (но для шаблонов — не критично)
- ⚠️ MultiLine с переполнением — текст обрезается без индикатора (можно добавить позже)

**Когда решать:** При создании TextTool (середина Этапа 1).
