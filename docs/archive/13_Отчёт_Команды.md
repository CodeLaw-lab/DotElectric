## 2. РЕАЛИЗОВАННЫЕ КОМПОНЕНТЫ

### 2.1. Интерфейс

### 2.2. CommandHistory

| Класс | Файл | Описание |
|-------|------|----------|
| **CommandHistory** | `Commands/CommandHistory.cs` | Undo/Redo стеки, 50 уровней, Push(), Undo(), Redo(), Clear(), Trim() |

**Свойства:**
| Свойство | Тип | Описание |
|----------|-----|----------|
| `MaxLevels` | int | Максимальное количество уровней Undo (по умолчанию 50) |
| `UndoCount` | int | Количество команд в undo-стеке |
| `RedoCount` | int | Количество команд в redo-стеке |
| `CanUndo` | bool | Есть ли команды для отмены |
| `CanRedo` | bool | Есть ли команды для повтора |
| `LastUndoName` | string? | Имя последней undo-команды (для UI) |
| `LastRedoName` | string? | Имя последней redo-команды (для UI) |

**Методы:**
| Метод | Описание |
|-------|----------|
| `Push(ICommand)` | Выполняет команду, добавляет в undo-стек, очищает redo-стек |
| `Undo()` | Отменяет последнюю команду, перемещает в redo-стек |
| `Redo()` | Повторяет отменённую команду, перемещает в undo-стек |
| `Clear()` | Очищает оба стека |

**Ключевые особенности:**
- `markDirty` callback — вызывается при каждом `Push()` (автоматическая пометка шаблона как изменённого)
- Trim — автоматически удаляет oldest команды при превышении `MaxLevels`
- Rollback — при ошибке в `Undo()` команда возвращается в undo-стек, выбрасывается `InvalidOperationException`
- Redo при `Redo()` — аналогичный rollback

### 2.3. Команды

|---------|------|-------------|---------|------|

## 3. АРХИТЕКТУРНЫЕ РЕШЕНИЯ

### 3.1. Минимальные зависимости

Все команды имеют **минимальные зависимости** — не требуют `EditorViewModel` или `Template`:

```
Команда получает:
├── ObservableCollection<ITemplateObject> — для работы с коллекцией
├── ITemplateObject / Rectangle / Text — для работы с конкретным объектом
├── Func<T> / Action<T> — для generic изменения свойства
└── Action? markDirty — callback для пометки как изменённого
```

Это позволяет:
- Тестировать команды без моков `EditorViewModel`
- Использовать команды в любом контексте (не только в редакторе)
- Легко интегрировать с будущим `EditorViewModel`

### 3.2. MarkDirty callback

Каждая команда принимает опциональный `Action? markDirty`:
```csharp
var cmd = new AddObjectCommand(objects, obj, () => editor.MarkDirty());
```

- Вызывается **только при `Execute()`**, НЕ при `Undo()`
- `CommandHistory.Push()` также вызывает `_markDirty?.Invoke()` после добавления
- Это гарантирует, что `IsDirty` устанавливается только при реальных изменениях

### 3.3. Undo/Redo потоки

```
User action
    │
    ▼
Command.Execute()          → markDirty() → шаблон изменён
    │
    ▼
CommandHistory.Push()      → добавляет в undo-стек, очищает redo, Trim
```

```
User: Ctrl+Z
    │
    ▼
CommandHistory.Undo()      → command.Undo() → перемещает в redo-стек
                              (rollback при ошибке)
```

```
User: Ctrl+Y
    │
    ▼
CommandHistory.Redo()      → command.Execute() → перемещает в undo-стек
                              (rollback при ошибке, Trim)
```

### 3.4. BatchCommand

Составные операции (например, «переместить + изменить размер»):

```csharp
var moveCmd = new MoveObjectCommand(obj, 10000, 20000);
var resizeCmd = new ResizeObjectCommand(rect, 50000, 30000);
var batch = new BatchCommand(new[] { moveCmd, resizeCmd }, "Переместить и изменить");

history.Push(batch);  // Выполняет обе команды
history.Undo();        // Отменяет в обратном порядке: resize → move
```

### 3.5. ChangePropertyCommand<T>

Generic-команда для изменения любого свойства:

```csharp
var cmd = new ChangePropertyCommand<string>(
    getter: () => textObject.Content,
    setter: v => textObject.Content = v,
    newValue: "Новый текст",
    propertyName: "Содержимое текста",
    markDirty: () => editor.MarkDirty()
);
```

### 3.6. RotateObjectCommand

Нормализует угол к допустимым значениям (0/90/180/270):

```csharp
// Текст с углом 270°, вращаем +90°
// 270 + 90 = 360 → нормализуется → 0°
var cmd = new RotateObjectCommand(textObject, deltaDegrees: +90);
```

### 3.7. DeleteObjectCommand

Запоминает индекс для восстановления при Undo:

```csharp
// Удаление объекта на позиции 5
var cmd = new DeleteObjectCommand(objects, obj);
cmd.Undo();  // Вставляет на позицию 5 (или в конец, если коллекция изменилась)
```

## 6. ИЗВЕСТНЫЕ ОГРАНИЧЕНИЯ

1. **Нет сериализации истории** — при закрытии приложения undo/redo теряются
2. **`ResizeObjectCommand` только для Rectangle** — для Line нужен отдельный подход (изменение EndMicrons)
3. **`RotateObjectCommand` только для Text** — Line и Rectangle не поддерживают вращение в модели
4. **`ClearSelectionCommand` НЕ помечает шаблон как изменённый** — выделение не влияет на данные шаблона
