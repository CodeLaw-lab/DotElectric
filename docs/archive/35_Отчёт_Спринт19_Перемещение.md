## ПРОБЛЕМЫ ДО ИСПРАВЛЕНИЯ

### Диагноз

Перемещение объектов работало некорректно по следующим причинам:

1. **Undo drag не работал** — `MoveObjectCommand` создавался в `OnMouseUp` когда объект УЖЕ перемещён, поэтому `_oldX = obj.MicronsX` захватывал новую позицию, а не старую. Undo возвращал объект туда же.

2. **Multi-selection drag без Undo** — для 2+ выделенных объектов никакая Undo-команда не создавалась.

3. **Snap to grid не работал при drag** — `SelectTool.OnMouseMove` вызывал `obj.Move()` напрямую без проверки `GridSettings.SnapEnabled`.

4. **Нет hover-эффекта** — линия не реагировала на наведение курсора.

5. **Маркеры не интерактивны** — `IsHitTestVisible="False"`, невозможно кликнуть по маркеру для resize.

6. **Нет hit-test маркеров** — невозможно отличить клик по телу объекта от клика по маркеру.

7. **SelectTool не различал режимы** — drag тела vs drag маркера обрабатывались одинаково.

8. **ResizeTool не активировался автоматически** — `ActiveResizeHandle` нигде не устанавливался.

## РЕАЛИЗОВАННЫЕ ИЗМЕНЕНИЯ

### 1. Hit-test маркеров (`HitTestHelper.GetHitHandle`)

**Файл:** `Helpers/HitTestHelper.cs`

Новый метод `GetHitHandle(PointMicrons, ITemplateObject)` определяет, попал ли курсор на маркер изменения размера:

- **Line:** 2 маркера — `TopLeft` (начало), `BottomRight` (конец). Зона захвата: 8мм (`HandleHitTolerance = 8000`).
- **Rectangle:** 8 маркеров — углы + середины сторон.
- **Text:** 4 угловых маркера.

Возвращает `ResizeHandle?` — `Some(handle)` если попали, `null` если нет.

### 2. Hover-эффект для объектов

**Файлы:** `ViewModels/EditorViewModel.cs`, `Tools/SelectTool.cs`, `Views/EditorCanvas.xaml`

- **`HoveredObject`** (ObservableProperty) — объект, на который наведён курсор.
- **`HoveredHandle`** (ObservableProperty) — маркер, на который наведён курсор.
- **`IsHoveredConverter`** — MultiBinding converter для проверки `ReferenceEquals(obj, hoveredObject)`.
- **`SelectTool.UpdateHoverState()`** — вызывается в `OnMouseMove` когда кнопка НЕ зажата. Определяет hover по маркерам (приоритет) и по объектам.
- **XAML DataTrigger** — при `IsHovered=true` линия получает `StrokeThickness=2.5` и `Stroke="#333"`.

### 3. Курсор из `GetCursor()` в EditorCanvasBehavior

**Файл:** `Behaviors/EditorCanvasBehavior.cs`

`State_MouseMove` теперь вызывает `tool.GetCursor()` и устанавливает `Canvas.Cursor`:
- `Hand` — hover на объекте / drag
- `Cross` — hover на маркере линии
- `SizeNWSE/SizeNESW/SizeNS/SizeWE` — resize маркеры rectangle
- `SizeAll` — панорамирование
- `Arrow` — по умолчанию

### 4. PushTool/PopTool — временное переключение инструментов

**Файл:** `ViewModels/EditorViewModel.cs`

- **`_toolStack: Stack<string>`** — стек предыдущих инструментов.
- **`PushTool(string)`** — сохраняет текущий `ActiveTool` в стек и переключает на новый.
- **`PopTool()`** — восстанавливает предыдущий инструмент из стека.

### 5. SelectTool — корректный drag + маркерный hit-test

**Файл:** `Tools/SelectTool.cs`

**OnMouseDown:**
1. **Приоритет 1:** Hit по маркеру single-selected объекта → `PushTool("Resize")` → делегирование ResizeTool.
2. **Приоритет 2:** Hit по объекту → выделение + начало drag тела.
3. **Приоритет 3:** Клик в пустое место → Selection Box.

**OnMouseMove:**
- Hover detection (когда НЕ зажата кнопка).
- Drag: delta рассчитывается от `_dragStartModelPoint`, применяется ко ВСЕМ выделенным объектам.
- **Snap to grid** — если `GridSettings.SnapEnabled`, координаты привязываются к сетке через `SnapHelper.SnapX/Y`.
- **Threshold** — `Math.Max(3000, 3000 / Zoom)` — учитывает масштаб.

**OnMouseUp:**
- **Single selection:** `MoveObjectCommand(obj, oldX, oldY, newX, newY, MarkDirty)` — корректный Undo.
- **Multi-selection:** `BatchCommand` с `MoveObjectCommand` для каждого объекта.

**GetCursor:**
- `Cross` если hover на маркере
- `Hand` если drag или hover на объекте
- `Arrow` по умолчанию

### 6. ResizeTool — resize линии по одной точке

**Файл:** `Tools/ResizeTool.cs`

- **`ResizeLine`** — меняет ТОЛЬКО одну точку: `TopLeft` → начало, `BottomRight` → конец. Вторая точка остаётся на месте.
- **`GetCursor`** — для линии всегда возвращает `Cross`.
- **`OnMouseUp`** — вызывает `_editor.PopTool()` для возврата к SelectTool.
- **`OnMouseUp`** — сбрасывает `ActiveResizeHandle = null`.

### 7. MoveObjectCommand — явный конструктор + markDirty в Undo

**Файл:** `Commands/MoveObjectCommand.cs`

- **Новый конструктор** `(obj, oldX, oldY, newX, newY, markDirty)` — для создания команды когда объект уже перемещён.
- **`Undo()`** — теперь вызывает `_markDirty?.Invoke()`.

### 8. BatchCommand — markDirty

**Файл:** `Commands/BatchCommand.cs`

- **Новый конструктор** `(commands, name, markDirty)`.
- **`Execute()`** и **`Undo()`** — вызывают `_markDirty?.Invoke()`.

## АРХИТЕКТУРНЫЕ РЕШЕНИЯ

### 1. Модельные координаты для hit-test, а не WPF

Hit-test маркеров идёт через `HitTestHelper.GetHitHandle(modelPoint, obj)` — используются модельные координаты (микроны), а не WPF hit-test. Это означает:
- Маркеры в XAML могут оставаться `IsHitTestVisible="False"` (визуальные элементы).
- Вся логика на уровне ViewModel/Tools — тестируется без UI.

### 2. PushTool/PopTool вместо прямой активации

Вместо прямой смены `ActiveTool` используется стек:
- `SelectTool.OnMouseDown` → `PushTool("Resize")` → ResizeTool.
- `ResizeTool.OnMouseUp` → `PopTool()` → возврат к SelectTool.

Это позволяет временно переключаться без потери контекста.

### 3. Delta-based drag

Перемещение рассчитывается как delta от начальной точки (`_dragStartModelPoint`), а не как абсолютная позиция. Это обеспечивает:
- Корректное перемещение всех выделенных объектов на одинаковую дельту.
- Независимость от того, за какую точку объекта схватили.

### 4. Snap применяется к каждой позиции

При включённом snap, каждая промежуточная позиция при drag привязывается к сетке. Это обеспечивает визуальную стабильность — объект не «прыгает» между шагами сетки.

## ИЗВЕСТНЫЕ ОГРАНИЧЕНИЯ

1. **Hover для Rectangle/Text** — hover-эффект реализован только для линии (StrokeThickness). Для Rectangle/Text можно добавить аналогично (например, более яркий stroke или highlight маркеров).

2. **Маркеры не интерактивны в XAML** — hit-test идёт через модельные координаты, не через WPF. Визуально маркеры не реагируют на hover (нет подсветки). Можно добавить в будущем.

3. **Drag без визуальной обратной связи при snap** — объект «прыгает» к ближайшему шагу сетки. Можно добавить preview-линию/прямоугольник для плавного drag с финальным snap в OnMouseUp.
