## 1. Описание проблемы

При создании нового шаблона (Ctrl+N / меню «Файл → Новый шаблон» / кнопка на Toolbar) **всегда создаётся шаблон формата A3**. Пользователь не имеет возможности выбрать формат листа (A0–A4 или пользовательский).

### Текущий поток (захардкожен на A3)

```
User: Ctrl+N
  ↓
MainWindow.xaml: CommandParameter="A3"  ← ЖЁСТКО
  ↓
MainViewModel.NewTab(format = "A3")
  ↓
TemplateService.CreateNew(format = "A3")
  ↓
Sheet.FromFormat("A3") → A3 (420×297 мм)
```

### Где захаркожено «A3» (8 мест)

| # | Файл | Строка | Код |
|---|------|--------|-----|
| 1 | `MainWindow.xaml` | ~24 | `<KeyBinding ... CommandParameter="A3"/>` |
| 2 | `MainWindow.xaml` | ~94 | `<MenuItem ... CommandParameter="A3"/>` |
| 3 | `MainWindow.xaml` | ~213 | `<Button ... CommandParameter="A3"/>` |
| 4 | `MainViewModel.cs` | ~102 | `NewTab(string format = "A3")` |

## 3. Требования к реализации

### 3.1. Функциональные требования

**FR-FMT-001:** При вызове команды «Новый шаблон» пользователь должен видеть диалог (или подменю) для выбора формата листа.

**FR-FMT-002:** Доступные варианты: A0, A1, A2, A3, A4, «Пользовательский».

**FR-FMT-003:** По умолчанию должен выбираться формат из настройки `DefaultSheetFormat` (текущее значение пользователя).

**FR-FMT-004:** Если пользователь выбирает «Пользовательский» — открыть диалог ввода ширины и высоты (в мм).

**FR-FMT-005:** Формат должен отображаться в DisplayName вкладки: `«A3 — Без имени»`.

### 3.2. Нефункциональные требования

**NFR-FMT-001:** Решение должно быть тестируемым (мокируемый диалог через `IDialogService` или отдельный интерфейс).

**NFR-FMT-002:** Сохранить обратную совместимость — Ctrl+N без выбора должен использовать `DefaultSheetFormat` из настроек.

**NFR-FMT-003:** Все новые/изменённые методы должны иметь unit-тесты (покрытие ≥ 80%).

## 4. Рекомендуемый подход

### Вариант A: Подменю «Новый шаблон ▸ A0/A1/A2/A3/A4» (Рекомендуется)

**Плюсы:**
- Быстрый выбор формата (1 клик)
- Соответствует паттерну «последний использованный = выбранный»
- Минимальные изменения в архитектуре

**Изменения:**
1. `MainWindow.xaml`: заменить `<MenuItem Header="_Новый шаблон">` на подменю с 6 пунктами (A0–A4 + Пользовательский)
2. `MainWindow.xaml`: KeyBinding Ctrl+N → вызывает команду с `LastUsedSheetFormat`
3. `MainViewModel.cs`: `NewTab(string format)` — уже принимает параметр, оставить
4. `SettingsService`: запоминать `LastUsedFormat` при каждом создании
5. Toolbar-кнопка «Новый» → использовать `LastUsedFormat`

### Вариант B: Диалог выбора формата

**Плюсы:**
- Явный выбор каждый раз
- Можно добавить превью формата

**Минусы:**
- Лишний клик
- Требует нового диалога

### Вариант C: ComboBox на Toolbar + Ctrl+N = DefaultSheetFormat

**Минусы:**
- Toolbar перегружен
- Ctrl+N = последний выбранный (неочевидно)

---

### Рекомендация: **Вариант A** (Подменю)

Наиболее соответствует CAD-приложениям и ГОСТ ЕСКД. Пользователь видит все варианты сразу, 1 клик = выбор. Ctrl+N = последний использованный формат.

## 5. Детальный план реализации (Вариант A)

### Шаг 1: Расширить AppSettings

```csharp
// AppSettings.cs
public string DefaultSheetFormat { get; set; } = "A3";
public string LastUsedSheetFormat { get; set; } = "A3";  // NEW
```

### Шаг 2: Обновить MainViewModel

```csharp
// MainViewModel.cs

[RelayCommand]
private void NewTab(string format = null)  // Убрать дефолт "A3"
{
    var fmt = format ?? _settingsService.GetSetting("LastUsedSheetFormat", "A3");
    _settingsService.SetSetting("LastUsedSheetFormat", fmt);
    
    var template = _templateService.CreateNew(fmt);
    var editor = new EditorViewModel(template, _templateService, printService: _printService) { MainVm = this };
    OpenedTabs.Add(editor);
    SelectedTab = editor;
}
```

### Шаг 3: Обновить MainWindow.xaml

**Было:**
```xml
<MenuItem Header="_Новый шаблон" Command="{Binding NewTabCommand}" 
          CommandParameter="A3" InputGestureText="Ctrl+N"/>
```

**Стало:**
```xml
<MenuItem Header="_Новый шаблон">
    <MenuItem Header="A0 (1189×841)" Command="{Binding NewTabCommand}" CommandParameter="A0"/>
    <MenuItem Header="A1 (841×594)"  Command="{Binding NewTabCommand}" CommandParameter="A1"/>
    <MenuItem Header="A2 (594×420)"  Command="{Binding NewTabCommand}" CommandParameter="A2"/>
    <MenuItem Header="A3 (420×297)"  Command="{Binding NewTabCommand}" CommandParameter="A3"/>
    <MenuItem Header="A4 (297×210)"  Command="{Binding NewTabCommand}" CommandParameter="A4"/>
    <Separator/>
    <MenuItem Header="Пользовательский..." Command="{Binding NewCustomTabCommand}"/>
</MenuItem>
```

**KeyBinding Ctrl+N:**
```xml
<!-- Ctrl+N = последний использованный формат -->
<KeyBinding Key="N" Modifiers="Control" Command="{Binding NewTabWithLastFormatCommand}"/>
```

**Toolbar:**
```xml
<!-- Кнопка «Новый» = последний использованный формат -->
<Button Command="{Binding NewTabWithLastFormatCommand}" ToolTip="Новый шаблон (Ctrl+N)"/>
```

### Шаг 4: Новая команда в MainViewModel

```csharp
[RelayCommand]
private void NewTabWithLastFormat()
{
    var fmt = _settingsService.GetSetting("LastUsedSheetFormat", "A3");
    NewTab(fmt);
}
```

### Шаг 5: Команда «Пользовательский» в MainViewModel

```csharp
[RelayCommand]
private async Task NewCustomTabAsync()
{
    // Показать диалог ввода размеров (через IDialogService)
    var result = await _dialogService.ShowCustomSheetDialog();
    if (result != null)
    {
        var sheet = Sheet.Custom(result.WidthMm, result.HeightMm);
        var template = _templateService.CreateFromSheet(sheet);  // NEW метод в TemplateService
        var editor = new EditorViewModel(template, ...) { MainVm = this };
        OpenedTabs.Add(editor);
        SelectedTab = editor;
    }
}
```

### Шаг 6: Добавить метод в TemplateService

```csharp
// ITemplateService.cs
Template CreateFromSheet(Sheet sheet);

// TemplateService.cs
public Template CreateFromSheet(Sheet sheet)
{
    var metadata = new Metadata { Name = "Без имени — Custom", ... };
    return new Template(metadata, sheet);
}
```
