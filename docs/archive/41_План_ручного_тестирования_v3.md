## 1. Введение

### 1.1. Цель документа

Настоящий план определяет стратегию ручного тестирования CAD-системы DotElectric (Этап 1 — Редактор шаблонов листов) после завершения Sprint 23.

### 1.2. Область применения

Документ охватывает **всю реализованную функциональность** включая:
- Управление шаблонами (multi-tab, ориентация листа)
- Рисование объектов (линии, прямоугольники, текст)
- Выделение и манипуляции (drag, resize, rotate, delete)
- Undo/Redo (50 уровней)
- Сетка и привязка (GridNodesLayer, шаг сетки, SnapHelper)
- Панель свойств (16 RelayCommands + 15 *FromString обёрток)
- Сохранение и загрузка (.tdel, XML в ZIP)
- Масштабирование и навигация (Zoom, Pan, Fit to Screen)
- Печать
- Темы оформления (Light/Dark)
- Горячие клавиши (24 штуки)
- Inline editing текста
- WeakReferenceMessenger (закрытие вкладок)
- IApplicationLifecycle
- **DI ITextToolSettings в TextTool** (Sprint 23)
- **Единый SnapHelper для инструментов** (Sprint 23)
- **ILogger в TemplateService/FileService** (Sprint 23)

### 1.3. Критерии приёмки

| Уровень | Критерий |
|---------|----------|
| ✅ **PASS** | Все P0 и P1 тесты пройдены, ≤ 3 P2 провалены |
| ⚠️ **MARGINAL** | Все P0 пройдены, ≤ 2 P1 провалены |
| ❌ **FAIL** | Любой P0 провален, или > 2 P1 провалены |

### 1.4. Известные проблемы (НЕ тестируются)

**Все Known Issues Sprint 23 устранены.** Нет известных проблем, требующих отдельной пометки при тестировании.

## 3. Тест-кейсы

### MT-001: Запуск приложения (P0)

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Запустить `dotnet run --project src/DotElectric.TemplateEditor` | Приложение запускается без ошибок |
| 2 | Проверить главное окно | Окно отображается с меню, Toolbar, StatusBar |
| 3 | Проверить вкладку | Вкладок нет, отображается заглушка "Нет открытых шаблонов" |
| 4 | Проверить StatusBar | StatusBar пуст или показывает версию |
| 5 | Проверить второй запуск | Второй экземпляр НЕ запускается (Mutex) |

### MT-002: Глобальные обработчики ошибок (P1)

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Проверить логирование | Файл лога создан в `%APPDATA%\DotElectric\logs\` |
| 2 | Проверить формат лога | Формат: `dotelectric-{yyyyMMdd}.txt`, daily ротация, 30 файлов |
| 3 | Проверить минимальный уровень | Уровень логирования: Debug (не Information) |

#### MT-002a: Логирование — TemplateService (P1)

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Создать новый шаблон | В логах есть запись: `TemplateService.CreateNew` с параметрами формата |
| 2 | Загрузить .tdel файл | В логах есть запись: `TemplateService.Load` с именем файла |
| 3 | Сохранить .tdel файл (Ctrl+S) | В логах есть запись: `TemplateService.Save` с именем файла |
| 4 | Попробовать загрузить corrupted файл | В логах есть запись: `ERROR` с деталями исключения |

#### MT-002b: Логирование — FileService (P1)

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Открыть файл (Ctrl+O) | В логах есть запись: `FileService.OpenDialog` с выбранным путём |
| 2 | Сохранить файл (Ctrl+S) | В логах есть запись: `FileService.SaveDialog` с выбранным путём |
| 3 | Дождаться автосохранения | В логах есть запись: `FileService.Backup` с путём к бэкапу |
| 4 | Проверить логи ошибок | Ошибки файловой системы логируются (например, доступ запрещён) |

### MT-003: Темы оформления (P1)

| Шаг | Действие | Ожидаемый результат |
