## 3. Описание проблем и решений

### 3.1. Проблема: Бесконечный холст вместо ScrollViewer

**Симптом:** Полосы прокрутки мешают работе, навигация должна быть через Pan (средняя кнопка мыши / Space+Drag) и зум.

**Решение:**
- Убран `ScrollViewer` из EditorCanvas.xaml
- Добавлен `Border` с `ClipToBounds` для фона
- Canvas имеет `RenderTransform → TranslateTransform` с binding к `PanOffsetX/Y`
- `PanCanvas()` обновляет PanOffsetX/Y вместо вызова события
- `ToModelPoint()` учитывает PanOffset при конвертации координат мыши

### 3.2. Проблема: Катастрофические тормоза при отрисовке сетки

**Симптом:** При зумировании на А3 (5000+ узлов) и особенно А0 (40 000+ узлов) приложение зависает. Каждый узел — отдельный `Ellipse` элемент WPF, пересоздаваемый при каждом изменении Zoom/Pan.

| Формат | Узлов | Операций при зуме (30/сек) |
|--------|-------|---------------------------|
| А4 | ~2 500 | 75 000 |
| А3 | ~5 000 | 150 000 |
| А2 | ~10 000 | 300 000 |
| А1 | ~20 000 | 600 000 |
| А0 | ~40 000 | 1 200 000 |

**Корневая причина:**
1. `ObservableCollection<GridNodeVm>` с 5000+ элементов → 5000× `CollectionChanged` при каждом изменении
2. Каждый `GridNodeVm` создаёт отдельный `Ellipse` (400 байт × 5000 = 2МБ)
3. WPF ItemsControl не поддерживает binding `Canvas.Left/Top` в ItemContainerStyle
4. Генерация для всего листа вместо видимой области

**Решение (4 уровня оптимизации):**

| Уровень | Было | После | Эффект |
|---------|------|-------|--------|
| **1. ObservableCollection → List** | 5000× CollectionChanged | 1× GridInvalidated callback | ~5000× |
| **2. Ellipse × 5000 → OnRender** | 5000 UI-элементов | 1 UIElement | ~100× |
| **3. Весь лист → видимая область** | 40 000 узлов | ~500 узлов | ~80× |
| **Итого** | 1 200 000 операций/сек | ~500 операций/сек | **~2000×** |

### 3.3. Проблема: Нельзя изменить шаг сетки

**Симптом:** Шаг сетки зафиксирован на 5мм, нет UI для изменения.

**Решение:**
- Добавлен ComboBox на Toolbar (1 мм, 2 мм, 5 мм, 10 мм)
- Свойство `StatusBarGridStepMm` в EditorViewModel
- Конвертер `GridStepToStringConverter` для преобразования double ↔ "X мм"

## 4. Архитектурные решения

### 4.1. GridNodesLayer через OnRender

```csharp
public class GridNodesLayer : UIElement
{
    public IList<GridNodeVm> Nodes { get; set; } = new List<GridNodeVm>();
    
    protected override void OnRender(DrawingContext dc)
    {
        foreach (var node in Nodes)
            dc.DrawEllipse(_nodeBrush, null, new Point(node.XPixels + 1, node.YPixels + 1), 1, 1);
    }
}
```

**Преимущества:**
- Один UIElement вместо 40 000
- Direct2D через DrawingContext — рендер на GPU
- Freeze кисти для оптимизации
- InvalidateVisual() — мгновенное обновление

### 4.2. Генерация только видимой области

```csharp
var viewportLeftMm = PanOffsetX / Zoom;
var viewportTopMm = PanOffsetY / Zoom;
var widthMm = ViewportWidthMm > 0 ? ViewportWidthMm : Template.Sheet.WidthMm;
```

Viewport вычисляется из `SizeChanged` EditorCanvas — размеры экрана в мм при текущем зуме.

### 4.3. Pan через RenderTransform

`TranslateTransform` работает на уровне композиции WPF — не вызывает layout pass, что обеспечивает плавное панорамирование даже при большом количестве элементов на Canvas.
